## 4. 连续外部工序合并周期：separate/merged

仍在零件详情页：

### 4.1 合并设置（merged）
- 将外部组周期模式选为“合并设置”
- 合计周期（天）填 `3`
- 点击“保存外部组设置”

预期（存储规则）：
- 外部组 `merge_mode = merged`，`total_days = 3`
- 组内外部工序 `ext_days` 显示为 `-`（不单独计时）

### 4.2 分别设置（separate）
- 将模式切回“分别设置”
- 为 35/40/45 各工序填写周期（如都填 `1`）
- 点击“保存外部组设置”

预期（存储规则）：
- 外部组 `merge_mode = separate`，`total_days = NULL`
- 组内外部工序 `ext_days` 各自有值

建议截图：

- separate 模式截图 1 张

## 6. Excel 导入导出（留痕必核对）

### 6.1 工种 Excel
- 打开：`/process/excel/op-types`
- 下载模板、导入预览、确认导入、导出

### 6.2 供应商 Excel
- 打开：`/process/excel/suppliers`

### 6.3 零件工艺路线 Excel
- 打开：`/process/excel/routes`
- 导入确认后应触发解析并生成模板

### 6.4 零件工序模板导出
- 打开：`/process/excel/part-operations`

预期：
- 预览页面显示 NEW/UPDATE/ERROR 状态
- 所有提示信息中文清晰
- 导入/导出后 `OperationLogs` 有记录（action=import/export）

建议 SQL（用 DB Browser for SQLite 执行）：

```sql
SELECT id, log_time, log_level, module, action, target_type, target_id, error_code, error_message, detail
FROM OperationLogs
ORDER BY id DESC
LIMIT 50;
```
