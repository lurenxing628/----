# 枚举映射表（裸字符串 → `core/models/enums.py`）

- 时间：2026-02-28 20:25
- 目的：把常见裸字符串值统一映射到 `core/models/enums.py` 的枚举，作为后续 Phase2~6 批次“最小一致性收敛”的事实依据。

## 总原则（落地口径）

- **来源唯一**：以 `core/models/enums.py` 为事实来源；业务代码只引用枚举成员（如 `BatchStatus.PENDING.value`），禁止裸字符串。
- **存储与比较**：
  - DB/JSON/Excel 中仍以字符串落库/落档：写入用 `Enum.value`
  - 业务判断统一用 `Enum.value` 比较（或先 normalize 再映射到 Enum）
- **normalize 口径**：统一 `strip().lower()`（现有 `normalize_text/to_yes_no` 等工具优先复用），避免大小写/空白导致的“隐性枚举漂移”。

## 模板/Jinja2 + 静态资源（static/js）策略与引用点登记

> 说明：现有 Python 体检工具不覆盖 `templates/` 与 `static/js/` 的字符串比较，本章节用于补齐“前端契约盲区”的事实登记与执行策略。

### 扫描范围与方法（只读）

- 范围：
  - `templates/**/*.html`
  - `static/js/**/*.js`（仅自研脚本；若存在第三方/压缩库，原则上排除）
- 关注点：`==/!=/===/!==` 等比较表达式中出现的值域（`yes/no/partial`、`internal/external`、`workday/weekend/holiday`、`pending/...`、`merged/separate` 等）。

### 执行策略（登记 + 复扫，避免一次性全量改模板）

- **默认策略**：先登记引用点 + 明确映射口径；不在本阶段对 `templates/` 做大规模替换，避免 UI 行为漂移。
- **Phase4（Route + 模板契约）**：若批次改到某个模块的 Route/ViewModel/Template，则在**该批次范围内**按本映射表做最小一致性收敛：
  - 优先把“判断逻辑”下沉到 ViewModel（输出 `label/class/flags`），模板只负责渲染；
  - 必须在模板内判断时，统一使用本表值域，不新增“同义字符串”。
- **Phase6（收口）**：复扫 `templates/` + `static/js`，核对枚举值域与 `core/models/enums.py` 是否一致，确保没有“前端枚举漂移”。
