- [ ] 版本语义变更已打穿既有兼容合同，但这轮修复没有显式处理取舍
  - 证据：
    - `web/routes/normalizers.py:85-113`
    - `web/routes/reports.py:35-47`
    - `web/routes/domains/scheduler/scheduler_week_plan.py:120-178`
    - `tests/regression_week_plan_filename_uses_normalized_version.py:47-67`
    - `tests/regression_reports_export_version_default_latest.py:76-124`
    - 实测：两个回归文件当前失败
  - 判断：
    - 这不是“测试没跟上”这么简单，而是同一仓库内旧契约与新契约并存且互相冲突。
    - 如果这是有意破坏兼容，必须同步修订旧回归、迁移说明、链接/脚本兼容策略；否则就是直接回归。

- [ ] “quality gate / full-selftest 已形成强绑定证明链”的自我结论当前证据不成立
  - 证据：
    - `scripts/run_quality_gate.py:467-515`

    - `tests/regression_week_plan_filename_uses_normalized_version.py` 已在 required-tests registry 中，但当前代码实测失败

    - 只要 required test 在当前代码上失败，就不能接受“quality gate 已通过且证明链有效”的结论。
    - 当前最多只能说“实现了 verifier 代码”，不能说“已经拿到可验证证据”。

### 建议改（P2）

- [ ] 路由注册仍未真正摆脱 import side effect，当前更像“显式函数 + 双重导入兜底”而不是“显式注册收口”
  - 证据：
    - `web/routes/domains/scheduler/scheduler_pages.py:3-9`
    - `web/routes/scheduler.py:5-9`
    - `web/routes/domains/scheduler/scheduler_route_registrar.py:23-29`
  - 判断：
    - 现在的实现确实把 side-effect registrar 收进了函数，但 root/pages import 仍会立即触发注册。
    - 如果目标是去掉导入时注册，这一轮并未真正完成，只是把 side effect 换了位置。

- [ ] manifest+verifier 的绑定项与 `HEAD + clean worktree` 高度重叠，维护成本上升但新增证明力有限
  - 证据：
    - `tools/quality_gate_shared.py:649-680`
  - 判断：
    - `required_tests_hash / commands_hash / collection_proof_hash / gate_sources_hash` 会显著增加演进摩擦。
    - 这些字段对“当前源码与 manifest 一致”有帮助，但对“这些命令确实执行过且结果可信”的增量证明有限。
