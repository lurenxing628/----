# 根因倒查法

当错误出现在很深的位置时，不要在报错点直接打补丁。正确做法是：**从症状一路倒着追到源头。**

## 步骤

1. 记下报错点：文件、行号、异常、坏值
2. 问：这个坏值是谁传进来的？
3. 再问：上一层为什么会传这个值？
4. 一直往上追，直到找到**第一个做错决定的位置**
5. 修那个源头，而不是最底层症状

## 在 APS 仓库里的常见链路

- Route 参数 → Service 归一化 → Repository 查询 → 模板显示
- Excel 导入行 → 预览校验 → 确认写入 → 汇总页展示
- 调度参数 → 输入构建 → 算法层 → 结果汇总

## 什么时候算追到头

至少满足一条：

- 找到了第一个错误默认值
- 找到了第一个错误分支判断
- 找到了第一个错误类型转换
- 找到了第一个错用旧字段 / 旧枚举的位置

如果你只是停在“这里报错了”，那还没追到根因。
