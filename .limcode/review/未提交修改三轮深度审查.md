  ### 安全视角
  - `run_quality_gate.py` 使用 `subprocess.run` **不带** `shell=True`，避免命令注入 ✓
  - CI 工作流中 `actions/checkout` 和 `actions/setup-python` 使用 SHA 锁定，降低供应链攻击面 ✓
  - `check_quickref_vs_routes.py` 的临时目录由 `tempfile.TemporaryDirectory` 管理，上下文退出后自动清理 ✓
  - 公共探针函数均为**只读**，不修改任何系统状态或文件 ✓

  ### 异常路径与边界场景
  1. **`_load_app_for_scan` 加载失败**：如果 `app.py` 不导出 `app` 也不导出 `create_app`，显式抛 `RuntimeError` ✓；如果 `spec` 为 `None`，显式抛出 ✓；加载过程中的异常不会泄漏模块到 `sys.modules`（`except` 分支里做了 `pop`） ✓
  2. **`_close_temp_log_handlers` 防重复关闭**：使用 `handled: Set[int]` 集合追踪已关闭的 handler id，避免同一 handler 被多个 logger 引用时重复 `close()` ✓
  3. **`_pid_signal` 对 None payload**：显式返回 `("absent", None, None, "")` ✓
  4. **`_health_signal` 对 port ≤ 0**：返回 `"unknown"` 而不是尝试探测无效端口 ✓
  5. **`_coerce_int` 对非法值**：调用方 `_pid_signal`/`_health_signal` 都用 `try/except` 包裹了 ✓
  6. **`test_no_silent_exception_swallow` 对 `web/ui_mode.py` 的特殊 scope_tag 检查**：只有该文件需要区分 `startup_guard` 和 `render_bridge` 两种 scope，逻辑正确且文档化 ✓

  ### Python 3.8 兼容性
  - 所有新增文件均使用 `from __future__ import annotations` ✓
  - `launcher.py` 中的 `tuple[str, str, str, str]`、`str | None` 等新语法仅出现在类型标注位置，在 `from __future__ import annotations` 下作为惰性字符串处理，不会在 3.8 下报错 ✓
  - CI 工作流显式指定 `python-version: '3.8'` ✓
