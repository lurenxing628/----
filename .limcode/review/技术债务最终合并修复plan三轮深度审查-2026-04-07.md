### Python 3.8 runtime_checkable 协议在算法热路径的性能风险未提及

- ID: F-performance-8
- 严重级别: 低
- 分类: 性能
- 跟踪状态: 开放
- 相关里程碑: round3-risk-and-gaps
- 说明:

  plan 第三节「关键结构决策」第 7 条声明算法层类型约束优先使用 Python 3.8 标准库 `typing.Protocol` 与 `typing.runtime_checkable`。但 `typing.Protocol` 在 Python 3.8 中借助 `from __future__ import annotations` 可以定义，而 `runtime_checkable` 装饰后的 `isinstance()` 检查在 Python 3.8 的标准库实现中比 3.9+ 性能更差（每次调用都会遍历协议成员）。如果在算法热路径（如每次调度循环）中使用 `isinstance(op, SomeProtocol)`，可能引入不必要的性能开销。plan 未显式要求“只在边界层做协议检查，热路径内不用 isinstance”。
- 建议:

  在任务 6 的实施过程中，显式约束 `runtime_checkable` 协议只用于算法入口的边界检查，不在调度主循环内部使用 `isinstance(op, Protocol)` 做运行时判定。
