## 变更摘要

本次变更共涉及 **17 个文件**（10 个业务/框架文件、5 个新增测试、2 个工具文件），目标是消除 `web/bootstrap/*` 和 `web/ui_mode.py` 中的 `except: pass` 静默回退，统一替换为可观测的降级日志（logger 优先，stderr 兜底）。

### 核心设计

| 组件 | 位置 | 职责 |
|------|------|------|
| `safe_log()` | `core/infrastructure/logging.py` | 两级容错日志：先尝试 logger 方法，失败则降级 stderr |
| `_format_log_message()` | 同上 | `%` 格式化安全封装 |
| `_app_log_once()` | `web/bootstrap/factory.py` | 每应用生命周期内每 key 仅记录一次（防刷屏） |
| `_write_launch_error_with_observability()` | `web/bootstrap/entrypoint.py` | 写启动错误文件 + 写入失败时降级日志 |
| `_log_startup_warning()` | `web/ui_mode.py` | 启动期 warning 封装，委托 `safe_log` |

## 第三轮：安全、性能与集成

### 3.1 性能

- `_app_log_once` 每次请求一次 `set` 查找（O(1)），可忽略
- `safe_log` 一次 `getattr` + 一次函数调用，可忽略
- 无内存泄漏风险：`_app_log_once` 的 flags set 生命周期跟随 Flask app，key 数量有限（< 10）

### 3.2 安全

- 日志消息只写文件/stderr，不暴露给前端用户，无信息泄露
- 异常对象通过 `%s` 格式化，不会泄露堆栈

### 3.3 线程安全

- `_app_log_once` 的 `set` 读写无锁保护，最坏情况是同一消息记录两次（可接受）
- `request_runtime_server_shutdown` 的 `_RUNTIME_SERVER_LOCK` 已有互斥，`safe_log` 在锁外调用，不会死锁

### 3.4 Win7 / Python 3.8 兼容

- `str | None` 类型注解在 `entrypoint.py` 第 106 行使用，但有 `from __future__ import annotations`，Python 3.8 安全
- 无新依赖引入

## 评审发现

### safe_log 的 getattr 未做防御

- ID: F1
- 严重级别: 低
- 分类: 可维护性
- 跟踪状态: 接受风险
- 相关里程碑: M2
- 说明:

  当 logger 对象有自定义 `__getattr__` 且抛出非 AttributeError 时，`getattr(logger, level, None)` 会把异常传播出 `safe_log`，违反「永不失败」的语义契约。实际所有调用方传入的都是标准 logging.Logger 或 None，此场景在生产中不会触发。
- 建议:

  可将整行用 try/except 包裹（作为后续加固项），当前风险极低，不阻断提交。
- 证据:
  - `core/infrastructure/logging.py:31-35`
  - `core/infrastructure/logging.py`

### safe_log 降级路径的 _format_log_message 未包裹在 _invoke_safely 内

- ID: F2
- 严重级别: 中
- 分类: 可维护性
- 跟踪状态: 接受风险
- 相关里程碑: M2
- 说明:

  `safe_log` 第 35 行，`_format_log_message(message, *args)` 在传给 `_invoke_safely(print, ...)` 之前先行求值。若 `str(message)` 抛异常（极端情况：message 对象的 `__str__` 爆炸），整个 `safe_log` 调用会向上抛。当前所有调用方传入的 message 都是字面量字符串，实际不会触发。
- 建议:

  可在 `safe_log` 末尾加一层 `try: ... except Exception: pass` 作为终极兜底。作为后续改进项。
- 证据:
  - `core/infrastructure/logging.py:35`
  - `core/infrastructure/logging.py`
