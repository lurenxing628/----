# SP09：页面流程与前端协议

## 背景

`SP05` 完成并通过修订后的兼容矩阵后，才视为目录骨架、scheduler 域 route 新路径、`navigation_utils.py` 唯一策略实现与根层兼容薄门面口径已经固定。`SP09` 负责在这条路径口径上继续完成页面域治理：`process/personnel/equipment/system` 域 route 迁移、`batch/dispatch/calendar` 服务分组物理归位，以及页面流程、模板协议、前端启动与静态资源契约的收口。

## 前置条件

- `SP05` 已完成并通过修订后的兼容矩阵；后续页面与 route 改动统一只改迁移后真实实现路径。
- `web/routes/domains/scheduler/` 已存在且通过 `SP05` 的 route 兼容 / patch-through 验证，scheduler 域 route 物理位置不在 `SP09` 内重新决策。
- `web/routes/navigation_utils.py` 已作为 `_safe_next_url` 唯一策略实现就位；若 `web.routes.system_utils._safe_next_url` 仍存在，它只能是保留旧路径观测 seam 的薄包装，不代表 `_safe_next_url` 回退到 `system_utils.py` 真实实现。
- `system_ui_mode.py` 的 referrer 回跳必须显式覆盖真实浏览器 absolute referrer：仅允许 same-origin absolute referrer 先折算为相对 `path?query`，不得把 absolute URL 直接喂给 `_safe_next_url`。

## 只做什么

1. 完成 `process/`、`personnel/`、`equipment/`、`system/` 域 route 实现向 `web/routes/domains/*/` 的物理迁移。
2. 完成 `core/services/scheduler/batch/`、`dispatch/`、`calendar/` 延后分组的物理归位与后续业务收口。
3. 收口页面流程、模板协议、`page_boot` 启动链、静态资源版本化与 `web/ui_mode.py` 渲染期桥接边界。
4. 修复 `system_ui_mode.py`：`next` 非法时优先回 same-origin absolute referrer，其次回相对 referrer，跨域或解析失败才回 `dashboard.index`。
5. 在旧模块精确路径导入方全部迁净后，按附录 C 回收符合条件的根层 route / 服务兼容薄门面。

## 不做什么

1. 不重新定义 `scheduler.py` 一级门面与 `domains/scheduler/scheduler_pages.py` 二级聚合链。
2. 不回退 `_safe_next_url` 到 `system_utils.py` 真实实现；允许 `system_utils.py` 保留经 `SP05` 登记的旧路径薄包装和 `url_for` monkeypatch seam。
3. 不让根层兼容薄门面重新承载页面业务逻辑或前端协议逻辑。

## 承接自 SP05 的硬约束

1. scheduler 域 route 路径必须已在 `SP05` 完成并通过修订后的兼容矩阵后才可被 `SP09` 消费；`SP09` 不再把 `scheduler_gantt.py`、`scheduler_week_plan.py`、`scheduler_excel_batches.py`、`scheduler_excel_calendar.py` 改回旧路径。
2. 根层 `process.py`、`personnel.py`、`equipment.py`、`system.py` 如在本批切到新域目录，必须同提交更新根层门面、模板引用、门禁文件与路由合同回归。
3. 根层兼容薄门面只作过渡；若 `SP09` 删除它们，必须同步清理所有旧模块精确路径导入方并更新附录 C。
4. `calendar_admin.py` 迁入 `calendar/` 时，必须同步把 `ConfigService` 改成 `from ..config.config_service import ConfigService`；`resource_pool_builder.py` 迁入 `dispatch/` 时，必须同步修改根层 `schedule_service.py` 对其导入路径，并检查 `core/services/scheduler/__init__.py` 的稳定导出是否已直指新真实实现。
