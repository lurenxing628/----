# 同步前端说明书以反映最新代码变更

## 一、变更来源分析

从提交 `dbd31d24` 到最新提交，经过多轮审查和修复计划（`20260404_未提交修改核实后收口修复plan.md`、`补充两条针对性回归测试消除假覆盖.plan.md`），已实施的**用户可感知行为变更**包括：

### 变更 A：批次 Excel 导入告警限流策略统一

- **位置**：`web/routes/scheduler_excel_batches.py` 第 387 行
- **变更前**：批次 Excel 导入成功且自动补建工序时，告警逐条全部 `flash` 到页面，无上限
- **变更后**：复用 `_surface_schedule_warnings()`，最多显示前 3 条明细告警；多于 3 条时页面会出现"另有 N 条告警，请到系统历史查看"的提示
- **影响说明书位置**：1.5.10 批次信息、6.1 新增批次、6.3 执行排产（告警显示描述）

### 变更 B：人员专属工作日历"可用工时/效率"严格数值校验

- **位置**：`core/services/common/excel_validators.py` 第 194、216 行
- **变更前**：使用裸 `float()` 解析，布尔值 `TRUE` 会被静默转成 `1.0`、`FALSE` 转成 `0.0`
- **变更后**：使用 `parse_finite_float()` 严格解析，布尔值会被明确拒绝（报错"必须是数字"），`NaN/Inf` 同样被拒绝
- **影响说明书位置**：1.5.9 人员专属工作日历、1.3.1 格式注意事项

### 变更 C：工作日历 Excel "可用工时/效率"严格数值校验

- **位置**：`web/routes/scheduler_excel_calendar.py` 第 204、216 行
- **变更前**：使用裸 `float()` + 宽泛 `except Exception` 解析
- **变更后**：使用 `parse_finite_float()` + `except ValidationError` 严格解析，拒绝布尔值、NaN、Inf
- **影响说明书位置**：1.5.8 工作日历

### 变更 D：供应商默认周期 Excel 导入严格数值校验

- **位置**：`web/routes/process_excel_suppliers.py` 第 94 行
- **变更前**：`TRUE` 会被 `float()` 静默转成 `1.0`
- **变更后**：使用 `parse_required_float()` 严格解析，拒绝布尔值、NaN、Inf
- **影响说明书位置**：1.5.2 供应商配置

### 变更 E：工序号解析严格化（零件工序工时导入）

- **位置**：`web/routes/process_excel_part_operation_hours.py` 第 47、58 行
- **变更前**：布尔值和科学计数法字符串（如 `5e0`）不被拒绝
- **变更后**：布尔值和科学计数法字符串被拒绝
- **影响说明书位置**：1.5.4 零件工序工时
