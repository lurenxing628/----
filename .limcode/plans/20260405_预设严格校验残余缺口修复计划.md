### 任务 2：在 `config_validator.py` 中做最小修复

**目标**
- 用显式存在性判别替代 `data.get(...) or base.xxx`，让三个选择字段与 `dispatch_mode` / `dispatch_rule` 的严格模式语义对齐。

**文件**
- 修改：`core/services/scheduler/config_validator.py`

- [ ] **步骤 1：拆出三个字段的原始值与是否出现**

  分别为 `sort_strategy`、`algo_mode`、`objective` 引入：
  - 原始输入值 `raw_* = data.get(key)`
  - 是否显式提供 `*_present = key in data`
  - 基准值 `base_* = str(base.xxx).strip().lower()`

- [ ] **步骤 2：按“缺字段 / 显式空白 / 非法非空 / 合法值”四段处理**

  三个字段统一改成与 `dispatch_mode` / `dispatch_rule` 同样的判别顺序：
  1. 字段未出现：直接使用 `base_*`
  2. 字段出现且规整后为空：
     - 严格模式 → `ValidationError(field=...)`
     - 非严格模式 → 回退 `base_*`
  3. 字段出现且非空，但不在合法集合里：
     - 严格模式 → `ValidationError(field=...)`
     - 非严格模式 → 回退 `base_*`
  4. 字段出现且合法：保留当前小写归一化结果

  关键点：继续保留对合法大小写混输输入的 `.strip().lower()` 归一化，不要把现有大小写兼容能力一并删掉。

- [ ] **步骤 3：不扩大改动边界**

  本步只修改上述三个字段，不改：
  - `_get_float()` / `_get_int()`
  - `_yesno()`
  - `dispatch_mode` / `dispatch_rule`
  - `config_presets.py` 的封装入口

- [ ] **步骤 4：回跑校验器用例**

  运行：`python -m pytest tests/regression_config_validator_preset_degradation.py -q`

  预期：新增空白断言通过，原有“坏数值仍拒绝”“缺字段仍允许回退”断言继续通过。
