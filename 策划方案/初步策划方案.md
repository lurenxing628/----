## 三、工艺路线解析器

### 3.1 解析逻辑

```
┌─────────────────────────────────────────────────────────────┐
│                    工艺路线解析流程                          │
└─────────────────────────────────────────────────────────────┘

输入: "5数铣10钳20数车25数铣30钳35标印40总检45表处理"

Step 1: 正则匹配
─────────────────────────────────────────────────
  模式: (\d+)([^\d]+)
  
  匹配结果:
  [(5, "数铣"), (10, "钳"), (20, "数车"), (25, "数铣"), 
   (30, "钳"), (35, "标印"), (40, "总检"), (45, "表处理")]

Step 2: 工种识别
─────────────────────────────────────────────────
  查询 OpTypes 表，判断每个工种是否为本单元能力
  
  数铣 → is_internal=true → internal
  钳   → is_internal=true → internal
  数车 → is_internal=true → internal
  标印 → is_internal=false → external
  总检 → is_internal=false → external
  表处理 → is_internal=false → external

Step 3: 生成工序列表
─────────────────────────────────────────────────
  [
    {seq: 5,  op_type: "数铣", source: "internal"},
    {seq: 10, op_type: "钳",   source: "internal"},
    {seq: 20, op_type: "数车", source: "internal"},
    {seq: 25, op_type: "数铣", source: "internal"},
    {seq: 30, op_type: "钳",   source: "internal"},
    {seq: 35, op_type: "标印", source: "external", supplier: "标印室", days: 0.5},
    {seq: 40, op_type: "总检", source: "external", supplier: "总检站", days: 1},
    {seq: 45, op_type: "表处理", source: "external", supplier: "热表中心", days: 2},
  ]
```

### 3.2 解析器代码示意

```python
# core/services/route_parser.py

import re
from typing import List, Tuple
from dataclasses import dataclass

@dataclass
class ParsedOperation:
    seq: int                    # 工序号
    op_type_name: str          # 工种名称
    source: str                 # internal/external
    supplier_id: str = None    # 外部供应商ID
    default_days: float = None # 默认周期

class RouteParser:
    """工艺路线解析器"""
    
    def __init__(self, op_types: dict, suppliers: dict):
        """
        op_types: {工种名: is_internal}
        suppliers: {工种名: (supplier_id, default_days)}
        """
        self.op_types = op_types
        self.suppliers = suppliers
    
    def parse(self, route_string: str) -> List[ParsedOperation]:
        """解析工艺路线字符串"""
        
        # 正则匹配: 数字 + 非数字
        pattern = r'(\d+)([^\d]+)'
        matches = re.findall(pattern, route_string)
        
        operations = []
        for seq_str, op_type_name in matches:
            seq = int(seq_str)
            op_type_name = op_type_name.strip()
            
            # 判断内部/外部
            is_internal = self.op_types.get(op_type_name, False)
            
            if is_internal:
                op = ParsedOperation(
                    seq=seq,
                    op_type_name=op_type_name,
                    source='internal'
                )
            else:
                # 外部工序，查找默认供应商
                supplier_info = self.suppliers.get(op_type_name)
                op = ParsedOperation(
                    seq=seq,
                    op_type_name=op_type_name,
                    source='external',
                    supplier_id=supplier_info[0] if supplier_info else None,
                    default_days=supplier_info[1] if supplier_info else 1.0
                )
            
            operations.append(op)
        
        return operations
```

## 六、排产算法更新

### 6.1 双重资源约束（设备+人员）

```
┌─────────────────────────────────────────────────────────────┐
│                    排产核心逻辑                              │
└─────────────────────────────────────────────────────────────┘

for 每个工序 in 优先级排序后的列表:
    
    if 工序.source == "external":
        # ─────────────────────────────────────────
        # 外部供应商：不占资源，只占时间
        # ─────────────────────────────────────────
        start = 上道工序完成时间
        end = start + ext_days (转换为小时)
        
        写入 Schedule (无 machine_id, 无 operator_id)
        continue
    
    # ─────────────────────────────────────────────
    # 内部工序：双重资源约束
    # ─────────────────────────────────────────────
    
    machine = 工序.machine_id
    operator = 工序.operator_id
    
    # 1. 前道工序约束
    prev_end = get_previous_op_end_time(工序)
    
    # 2. 设备可用时间
    machine_available = get_resource_available(machine)
    
    # 3. 人员可用时间
    operator_available = get_resource_available(operator)
    
    # 4. 取三者最晚时间
    earliest = max(prev_end, machine_available, operator_available)
    
    # 5. 工作日历约束
    earliest = adjust_for_calendar(earliest, 工序所属批次.priority)
    
    # 6. 计算工时
    total_hours = setup_hours + unit_hours × quantity
    
    # 7. 考虑效率系数 (周末)
    if is_weekend(earliest):
        total_hours = total_hours / efficiency
    
    # 8. 计算结束时间 (跳过非工作时段)
    end = calculate_end_time(earliest, total_hours, calendar)
    
    # 9. 更新资源占用
    occupy_resource(machine, earliest, end)
    occupy_resource(operator, earliest, end)
    
    # 10. 写入排程结果
    save_schedule(工序, machine, operator, earliest, end)
```

```
  原: [5外][10外][15内][20内]...
  删: [15内][20内]...
  ✅ 允许，不影响内部工序连续性

规则2: 尾部连续外部工序 → 可删除
─────────────────────────────────────────────────
  原: ...[20内][25内][30外][35外][40外]
  删: ...[20内][25内]
  ✅ 允许，不影响内部工序连续性

规则3: 中间孤立外部工序 → 不可删除
─────────────────────────────────────────────────
  原: [5内][10内][15外][20内][25内]
  删: [5内][10内][???][20内][25内]
  ❌ 不允许，工序10和工序20之间断档

规则4: 中间连续外部工序 → 整组删除或都不删
─────────────────────────────────────────────────
  原: [5内][10外][15外][20外][25内]
  删整组: [5内][25内]
  ❌ 不允许，工序5和25之间断档
  
  删部分: [5内][10外][20外][25内]
  ❌ 也不允许，还是断档


特殊情况: 通过调整让中间外部变成边缘
─────────────────────────────────────────────────
  原: [5内][10外][15内][20内]
  
  如果把工序5改成外部:
  调整后: [5外][10外][15内][20内]
  现在 [5外][10外] 变成首部连续外部
  → 可以删除

  这需要修改单元能力配置或工序归属
```

### 3.2 删除逻辑代码

```python
def can_delete_external_ops(operations: List[Operation], 
                            to_delete: List[int]) -> Tuple[bool, str]:
    """
    检查是否可以删除指定的外部工序
    
    Args:
        operations: 完整工序列表（按seq排序）
        to_delete: 要删除的工序seq列表
    
    Returns:
        (是否可删除, 原因说明)
    """
    
    # 标记删除后的工序列表
    remaining = [op for op in operations if op.seq not in to_delete]
    
    # 检查：要删除的必须都是外部工序
    for seq in to_delete:
        op = next((o for o in operations if o.seq == seq), None)
        if op and op.source == 'internal':
            return False, f"工序{seq}是内部工序，不能删除"
    
    # 检查：删除后内部工序是否连续
    internal_ops = [op for op in remaining if op.source == 'internal']
    
    if len(internal_ops) <= 1:
        return True, "删除后内部工序不足2道，允许"
    
    # 检查内部工序之间是否有断档
    for i in range(len(internal_ops) - 1):
        current = internal_ops[i]
        next_op = internal_ops[i + 1]
        
        # 检查两个内部工序之间是否只有被删除的外部工序
        between_ops = [op for op in remaining 
                       if current.seq < op.seq < next_op.seq]
        
        # 如果中间还有外部工序，说明断档了
        external_between = [op for op in between_ops 
                           if op.source == 'external']
        if external_between:
            return False, f"工序{current.seq}和{next_op.seq}之间存在未删除的外部工序，会导致断档"
    
    return True, "可以删除"
```
