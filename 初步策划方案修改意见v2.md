## 2) “留好 pandas 接口”的关键：定义清晰的边界

### 2.1 Excel 处理做成可替换后端（Backend）

定义一个抽象接口（核心点：**业务层只认 dict 列表**）：

```python
# core/services/common/tabular_backend.py
from abc import ABC, abstractmethod
from typing import List, Dict, Any, Optional

class TabularBackend(ABC):
    @abstractmethod
    def read(self, file_path: str, sheet: Optional[str] = None) -> List[Dict[str, Any]]:
        ...

    @abstractmethod
    def write(self, rows: List[Dict[str, Any]], file_path: str, sheet: str = "Sheet1"):
        ...
```

V1实现：
- `OpenpyxlBackend(TabularBackend)`

未来加 pandas 时：
- `PandasBackend(TabularBackend)`（内部用 DataFrame，但对外仍吐出/吃进 `List[Dict]`）

这样未来换 pandas，不动：
- 导入预览逻辑
- 校验逻辑
- repo/service 逻辑
- 排产逻辑

只替换 backend + 少量转换。

### 2.2 ExcelService 只做“业务级功能”，不关心实现库

```python
class ExcelService:
    def __init__(self, backend: TabularBackend, logger, op_logger):
        self.backend = backend
```

## 3) “注意做好记录”：四类关键留痕（你现有表足够承载）

你方案里已经有 `OperationLogs` 和 `ScheduleHistory`，非常好。我建议把下面这些“记录字段”固定下来，保证未来报表统计有数据来源（不增加新功能，只是把 detail 记录完整）。

### 3.1 Excel 导入留痕（OperationLogs）

- action: `import`
- target_type: `operator/machine/part/batch/calendar/...`
- detail（JSON建议包含）：
 - `filename`
 - `file_hash`（可选）
 - `mode`（append/overwrite/replace）
 - `total_rows/new_count/update_count/skip_count/error_count`
 - `errors_sample`（取前10条）
 - `time_cost_ms`

### 3.2 Excel 导出留痕（OperationLogs）

- action: `export`
- detail：
 - `template_or_export_type`
 - `filters`（导出筛选条件）
 - `row_count`
 - `time_range`（例如导出周计划的起止）

### 3.3 排产留痕（ScheduleHistory + OperationLogs）

- `ScheduleHistory`：已经有 strategy/version/result_summary，非常适合做未来统计的“事实表”
- 建议 `result_summary` 至少包含：
 - `strategy_params`（权重策略的权重）
 - `calendar_snapshot_id`（可选）
 - `overdue_batches`（数量/列表）
 - `resource_utilization`（可先不算，未来报表再算）

同时 `OperationLogs` 再记一条 action=`schedule`，用于操作审计。

### 3.4 报表留痕（未来）

即便未来做报表页面，我也建议每次生成报表都记 `OperationLogs`：
- action: `report`
- target_type: `report`
- target_id: `utilization_weekly` / `on_time_rate_monthly` 等
- detail：参数、时间范围、耗时、行数

这能解决你说的“注意做好记录”（可追溯、可审计、可复盘）。

## 4) 未来报表统计如何接入 pandas（不返工路线）
