## 1. 最终结论与必须落地的变更

### 1.1 总结结论

**方案整体可行，核心设计无需推翻。**

在“Win7 单机共享数据、仅允许单活用户”的交付前提下，V1 的关键点是：

1. **锁定 Python 3.8**（避免 Win7 兼容问题）
2. **Excel 采用 openpyxl-only（V1）**，并通过接口边界为未来引入 pandas 留后门
3. **甘特图库选型写死：Frappe Gantt 0.6.1（MIT）**，静态资源本地化
4. **删除业务级并发控制（资源锁）相关代码与表**，改为运行时单活用户保护
5. **备份机制简化为退出时自动备份**（避免后台线程/定时任务复杂度）
6. **交付方式明确：PyInstaller 4.10 打包为 exe，并必须在 Win7 环境打包**
7. **“注意做好记录”落地：导入/导出/排产/报表四类操作必须可追溯**（日志 + 数据库留痕字段规范化；报表中心 V1.1 已提供基础页面/导出，留痕以 export 为主）

## 2. 运行环境与交付形态（Win7 共享安装版）

### 2.1 运行环境

- 操作系统：Windows 7（目标机）
- 运行形态：**单机共享数据、仅允许单活用户**（不同账户可轮流使用同一套数据，但不允许并发访问）
- Python：**3.8.x**（仅开发/打包环境需要；目标机不需要安装 Python）
- 数据库：SQLite（Python 内置）
- Web 框架：Flask（本地离线运行）

### 2.2 交付形态

- 交付物：PyInstaller 打包后的 `--onedir` 目录，以及 Win7 双包安装器 `APS_Main_Setup.exe` + `APS_Chrome109_Runtime.exe`
- 运行方式：
  - 最小直拷口径：双击 `排产系统.exe`
  - 安装包口径：使用开始菜单/桌面快捷方式 `排产系统`
- 最小直拷目录：包含 `db/ logs/ backups/ templates_excel/ static/ templates/` 等（仅适用于目录自包含调试口径）
- 正式安装目录：主程序默认 `C:\Program Files\APS\SchedulerApp`，浏览器运行时默认 `C:\Program Files\APS\Chrome109`
- 共享数据目录：默认 `C:\ProgramData\APS\shared-data`，其中包含 `db/ logs/ backups/ templates_excel/`
- 浏览器运行时包采用 APS 专用保守裁剪：
  - 仅保留 `locales\zh-CN.pak`、`locales\en-US.pak`
  - 移除 `chrome_proxy.exe`、`chrome_pwa_launcher.exe`、`notification_helper.exe`、`elevation_service.exe`
  - 保留 `chrome_wer.dll` 与 `First Run`

### 2.3 启动端口策略（Win7 现场排障）

> 背景：Win7 现场经常遇到“端口被占用/受限（WinError 10013/10048）导致无法启动或打开错端口”。因此启动策略必须可自愈，并且可被脚本/运维准确感知实际端口。

- **host/port 来源**：
  - `APS_HOST`：默认 `127.0.0.1`
  - `APS_PORT`：首选端口（默认 5000）
- **默认环境**（未设置 `APS_ENV` 时）：源码运行使用 `default`；PyInstaller 冻结运行使用 `production`（见 `app.py`）。
- **自动回退策略**：若端口不可绑定，会按候选序列尝试并选择可用端口：`APS_PORT` → `5000` → `5705/5706/5707/5710/5711/5712/5713/5714/5715` → 最后由 OS 分配临时端口（bind 0）。
- **端口文件（关键）**：应用启动后会写入运行时状态目录中的 `aps_port.txt`（仅数字 + 换行），表示**实际监听端口**。
  - 现场脚本/启动器应优先读取该文件来打开 URL，避免“应用回退到新端口，但仍按 5000 打开”。
  - 该文件属于运行时 scratch，建议被 `.gitignore` 忽略。
- **Host 文件（关键）**：应用启动后会写入运行时状态目录中的 `aps_host.txt`（一行 host + 换行），表示**启动器/脚本应访问的实际 host**。
  - 当 `APS_HOST` 不可绑定回退到 `127.0.0.1`，或监听 `0.0.0.0` 时，会写入 `127.0.0.1`，避免打开不可访问地址。
  - 该文件属于运行时 scratch，建议被 `.gitignore` 忽略。
- **冻结环境（PyInstaller）**：禁用 Flask reloader，避免多进程导致端口与副作用不一致（例如父子进程各自选端口）。
- **启动器联动（交付目录 / 安装目录）**：`assets/启动_排产系统_Chrome.bat` 会先启动 exe，再优先读取运行时状态目录中的 `aps_host.txt` + `aps_port.txt` 打开 `http://{HOST}:{PORT}/`。
  - 安装包口径下，启动器会先解析共享数据根（`APS_SHARED_DATA_ROOT` 或 `HKLM\SOFTWARE\APS\SharedDataRoot`），并把 `APS_DB_PATH` / `APS_LOG_DIR` / `APS_BACKUP_DIR` / `APS_EXCEL_TEMPLATE_DIR` 注入当前进程
  - 浏览器查找顺序：`APS_CHROME_EXE` → 当前进程 `APS_CHROME_DIR\chrome.exe` / `App\chrome.exe` → 机器级注册表 `HKLM\SOFTWARE\APS\ChromeDir\chrome.exe` / `App\chrome.exe` → 兼容旧版注册表 `HKCU\Environment\APS_CHROME_DIR\chrome.exe` / `App\chrome.exe` → 默认 `C:\Program Files\APS\Chrome109\chrome.exe` / `%LOCALAPPDATA%\APS\Chrome109\chrome.exe` → `tools\chrome109\chrome.exe` / `App\chrome.exe`
  - 浏览器运行时包会把 `ChromeDir` 写入机器级注册表；`APS_CHROME_EXE` 仅用于人工覆盖
  - 启动器日志位于共享日志目录 `C:\ProgramData\APS\shared-data\logs\launcher.log`，会记录 `env_APS_CHROME_DIR`、`reg_HKLM_ChromeDir`、`reg_HKCU_APS_CHROME_DIR`、`chrome_source`、`chrome_exe` 等定位信息
  - 启动器会检查 `aps_runtime.lock`；若检测到其他账户正在使用共享数据目录，会明确阻止第二个账户进入
  - 主程序包不再内置浏览器运行时；最小直拷目录也不承诺自带浏览器运行时
  - 正式运行时包只保证本项目所需的 `chrome.exe --app=http://{HOST}:{PORT}/` 打开链路，不承诺完整桌面 Chrome / PWA / Windows 原生通知激活能力
  - `validate_dist_exe.py` 只覆盖主程序 `exe` 冷启动与 HTTP 页面可访问性，不覆盖快捷方式、环境变量刷新时序或 Chrome 启动链路

## 3. 技术选型与依赖锁定

### 3.1 V1 最终技术选择

| 领域 | 选型 | 备注 |
|---|---|---|
| Web | Flask 2.3.3 + Werkzeug 2.3.7 | 本地离线 Web UI |
| DB | SQLite | 单机易部署 |
| Excel | **openpyxl-only** | V1 交付成功率最高 |

### 3.2 关键边界（为未来 pandas/报表预留）

1. Excel 读写封装为可替换 Backend：V1 `OpenpyxlBackend`，未来可增 `PandasBackend`
2. 报表统计通过 `ReportEngine` 接口接入：V1 可先留空或纯 Python 简单实现
3. 所有对前端/接口返回的数据结构，统一 `List[Dict]`，避免绑定 pandas

## 4. 系统概述

### 4.1 系统目标

为回转壳体单元计调员提供智能排产工具，实现：

- 工艺路线自动解析与内外部工序识别
- 双重资源约束（设备 + 人员）排产
- 多策略排序算法（优先级/交期/权重/先进先出）
- 一周滚动甘特图可视化
- 插单模拟与超期预警
- 完善的错误处理与问题追溯

### 4.2 核心设计原则

```
┌─────────────────────────────────────────────────────────────┐
│                    设计原则                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 模块解耦                                                │
│     → 人员管理、设备管理、工艺管理、物料管理 独立模块       │
```

## 6. 数据模型（SQLite）

### 6.1 设计说明（唯一标识 / 字符串枚举 / 预留字段）

#### 6.1.1 唯一标识策略

- **零件（Part）**：以 `part_no`（图号）为主键。
- **批次（Batch）**：以 `batch_id`（批次号）为主键。
- **工序模板（PartOperations）**：使用自增 `id` 作为技术主键，同时使用 `UNIQUE(part_no, seq)` 约束业务唯一。
- **批次工序（BatchOperations）**：
  - `id` 为自增技术主键（用于关联排程表）。
  - `op_code` 为可读工序编码（`batch_id + seq` 生成，如 `B001_05`），用于展示/检索。
  - 业务唯一性：`UNIQUE(batch_id, seq, piece_id)`。
    - V1 默认 `piece_id` 为空（表示“批次级工序”），此时等价于 `batch_id + seq` 唯一。
    - 未来若引入“单件追溯”，`piece_id` 填充后，可支持同一批次同一工序号的多件拆分。

#### 6.1.2 字段统一为字符串枚举

原方案中 `is_active/is_internal/...` 等布尔概念字段统一替换为字符串枚举，便于扩展与 Excel 导入导出。

### 6.2.1 SchemaVersion（用于轻量迁移/回滚）

> 目的：记录数据库结构版本号，支撑“启动时轻量迁移/回滚”。  
> 当前实现会在启动 `ensure_schema()` 时读取/修正该版本号，并在需要迁移时先执行 `before_migrate_*` 备份；迁移失败时再用该备份做文件级回滚（见 `core/infrastructure/database.py`）。

```sql
CREATE TABLE SchemaVersion (
    id              INTEGER PRIMARY KEY CHECK (id = 1),
    version         INTEGER NOT NULL,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);

INSERT OR IGNORE INTO SchemaVersion (id, version) VALUES (1, 0);
```

- 当前版本常量：`CURRENT_SCHEMA_VERSION = 6`
- 迁移脚本范围：`core/infrastructure/migrations/v1.py` ~ `v6.py`
- 迁移返回值：`MigrationOutcome`
  - `APPLIED`：迁移意图被完整执行（含幂等 no-op）
  - `SKIPPED`：前置表/列不存在，整个迁移跳过
  - `PARTIAL`：部分步骤成功、部分跳过或部分失败
- 升版规则：**只有 `APPLIED` 才推进 `SchemaVersion`**；`SKIPPED/PARTIAL` 必须停在最后真实完成版本，不能“没报错就直升当前版本”。
- 空库快进规则：只有“真正空库”且代表性结构已齐全时，允许直接标定到当前版本；非空 partial DB 一律走逐版判断，不允许快进。
- 非空旧库补齐规则：若只缺“整张表”，允许先按 `schema.sql` 受控补齐整表/索引，再继续迁移；**不会**自动修补“表已存在但列/索引不完整”的复杂残缺库。
- 复杂残缺库策略：迁移前会先做预检；若预检仍会得到 `SKIPPED/PARTIAL`，直接 fail-fast 并报明确诊断，不再静默重试或重复生成迁移前备份。
- 迁移职责摘要：
  - `v1`：补 `ready_date/category/scope_type/...` 等列，并清洗 `Batches.due_date/ready_date`
  - `v2`：把 `WorkCalendar.day_type='weekend'` 统一为 `holiday`
  - `v3`：把 `OperatorCalendar.day_type='weekend'` 统一为 `holiday`
  - `v4`：统一清洗枚举/状态类文本字段（`TRIM + LOWER + default`）
  - `v5`：规范化 `OperatorMachine.skill_level/is_primary`
  - `v6`：创建 `ResourceTeams`，补 `Operators.team_id / Machines.team_id` 与对应索引

#### 6.6.1 内部工序：人员-设备匹配约束（OperatorMachine）

> 目的：避免在排产前置“工序补充”阶段出现“人员不会操作该设备”的脏数据，降低后续排产算法排障成本。

- **规则**：当批次工序（`BatchOperations.source='internal'`）同时填写 `machine_id` 与 `operator_id` 时，必须满足 `OperatorMachine` 中已维护的可操作关系（`(operator_id, machine_id)` 存在）。  
  - 任意一方为空：允许保存（表示“尚未补全”，或用户希望先填其一）。  
  - 两者都填写但不匹配：**禁止保存**，并给出**中文**错误提示，包含 `operator_id/machine_id/op_code/op_id`，明确指出是哪条工序的哪对资源不匹配。  
- **交互**：批次详情页的设备/人员下拉做**双向联动**（选设备过滤人员、选人员过滤设备），减少误选；最终以服务端强校验为准。  
- **引导**：若无可选项，提示用户先在【人员管理】或【设备管理】维护 `OperatorMachine` 关联。  

#### 6.6.2 人员专属工作日历（OperatorCalendar）

> 目的：支持“请假/加班/轮班”等人员差异日历，在排产推进工作时间时按人员维度生效。

- **覆盖优先级**：`OperatorCalendar(operator_id, date)` → `WorkCalendar(date)` → 默认规则（未配置日期时：工作日 8 小时；周末/节假日视为假期，默认不排产）。
- **口径**：`day_type` 统一视为 `workday/holiday` 两类；`weekend` 作为兼容输入/默认规则值，展示统一为“假期”，导入/落库会归一为 `holiday`。
- **算法使用**：日历服务接口支持传入 `operator_id`（例如 `adjust_to_working_time(..., operator_id=...)`），以便对同一天不同人员应用不同的班次/效率。
- **入口**：人员详情页提供“个人工作日历”入口；并提供 Excel 导入/导出（见第 11 章模板清单）。

#### 6.6.2A 工序补充输入校验（内部/外部）

> 目的：避免“看起来保存成功，但后续计算被静默回退”的隐式数据漂移，保证排产事实可追溯。

- **有限数值约束**：工序补充相关数值字段（如 `setup_hours` / `unit_hours` / `ext_days`）必须是**有限数字**，明确拒绝 `NaN/Inf`。
- **状态白名单**：批次工序状态仅允许 `pending/scheduled/processing/completed/skipped`；非法状态值禁止写入。
- **配置同口径**：`holiday_default_efficiency`、`time_budget_seconds`、`ortools_time_limit_seconds`、`freeze_window_days` 等配置键同样执行有限值校验（见 10.5.1）。

#### 6.6.3 系统使用说明（双模式：整本 + 页面级）

> 目的：给计划调度员/工艺员/管理员提供离线可用的“系统使用说明”，同时把说明入口升级为“当前页面只看当前说明 + 相关模块关键说明”的页面级手册，不新增第三方渲染依赖，且在模板-后端版本不一致时不导致整页 500。

## 7. 工艺路线解析器

### 7.1 解析逻辑

```
┌─────────────────────────────────────────────────────────────┐
│                    工艺路线解析流程                          │
└─────────────────────────────────────────────────────────────┘

输入: "5数铣 10钳-20数车、25数铣，30钳 35标印40总检45表处理"
      (包含各种分隔符和空格)

Step 0: 预处理
─────────────────────────────────────────────────────────────
  1. 去除首尾空格
  2. 统一分隔符（空格、逗号、顿号、破折号 → 移除）
  3. 保留数字和汉字

  结果: "5数铣10钳20数车25数铣30钳35标印40总检45表处理"

Step 1: 正则匹配
─────────────────────────────────────────────────────────────
  模式: (\d+)([^\d]+)

  匹配结果:
  [(5, "数铣"), (10, "钳"), (20, "数车"), (25, "数铣"),
   (30, "钳"), (35, "标印"), (40, "总检"), (45, "表处理")]

Step 2: 工种识别
─────────────────────────────────────────────────────────────
  查询 OpTypes 表，判断每个工种是否为本单元能力

  数铣 → category='internal' → internal
  钳   → category='internal' → internal
  数车 → category='internal' → internal
  标印 → category='external' → external
  总检 → category='external' → external
  表处理 → category='external' → external

  未识别工种 → 警告，默认标记为 external

Step 3: 识别连续外部工序组
─────────────────────────────────────────────────────────────
  遍历工序列表，将连续的外部工序归为一组

  [5数铣, 10钳, 20数车, 25数铣, 30钳] → 各自独立
  [35标印, 40总检, 45表处理] → 连续外部组 EXT_1

Step 4: 生成解析报告
─────────────────────────────────────────────────────────────
  {
    "success": true,
    "operations": [...],
    "external_groups": [...],
    "warnings": ["工种'表处理'未在系统中配置，已默认标记为外部工序"],
    "stats": {
      "total": 8,
      "internal": 5,
      "external": 3,
      "unknown": 1
    }
  }
```

### 7.2 解析器代码（方案原始代码，保留）

> 说明：以下代码来自方案 v2，体现“预处理 + 容错 + 报告”的完整逻辑，属于必须保留的核心实现。

```python
# core/services/process/route_parser.py

import re
from typing import List, Optional, Tuple, Dict, Any
from dataclasses import dataclass, field
from enum import Enum

class ParseStatus(Enum):
    SUCCESS = "success"
    PARTIAL = "partial"  # 有警告但可用
    FAILED = "failed"

@dataclass
class ParsedOperation:
    """解析后的工序"""
    seq: int                        # 工序号
    op_type_name: str              # 工种名称
    source: str                     # internal/external
    op_type_id: Optional[str] = None
    supplier_id: Optional[str] = None
    default_days: Optional[float] = None
    ext_group_id: Optional[str] = None
    is_recognized: bool = True      # 是否识别的工种

@dataclass
class ExternalGroup:
    """外部工序组"""
    group_id: str
    start_seq: int
    end_seq: int
    operations: List[ParsedOperation] = field(default_factory=list)

@dataclass
class ParseResult:
    """解析结果"""
    status: ParseStatus
    operations: List[ParsedOperation]
    external_groups: List[ExternalGroup]
    warnings: List[str]
    errors: List[str]
    stats: Dict[str, int]
    original_input: str
    normalized_input: str

class RouteParser:
    """工艺路线解析器"""

    # 需要移除的分隔符
    SEPARATORS = r'[\s,，、\-\—\→\->]+'

    def __init__(self, op_types_repo, suppliers_repo, logger=None):
        """
        Args:
            op_types_repo: 工种数据访问对象
            suppliers_repo: 供应商数据访问对象
            logger: 日志记录器
        """
        self.op_types_repo = op_types_repo
        self.suppliers_repo = suppliers_repo
        self.logger = logger

    def parse(self, route_string: str, part_no: str) -> ParseResult:
        """
        解析工艺路线字符串

        Args:
            route_string: 原始工艺路线字符串
            part_no: 图号 (用于生成组ID)

        Returns:
            ParseResult 解析结果
        """
        warnings = []
        errors = []
        original_input = route_string

        # Step 0: 预处理
        normalized = self._preprocess(route_string)

        if not normalized:
            return ParseResult(
                status=ParseStatus.FAILED,
                operations=[],
                external_groups=[],
                warnings=[],
                errors=["工艺路线为空或格式无效"],
                stats={"total": 0, "internal": 0, "external": 0, "unknown": 0},
                original_input=original_input,
                normalized_input=""
            )

        # 加载配置
        op_types = {ot.name: ot for ot in self.op_types_repo.get_all()}
        suppliers = self._build_supplier_map()

        # Step 1: 正则匹配
        pattern = r'(\d+)([^\d]+)'
        matches = re.findall(pattern, normalized)

        if not matches:
            return ParseResult(
                status=ParseStatus.FAILED,
                operations=[],
                external_groups=[],
                warnings=[],
                errors=["无法识别工艺路线格式，请使用'工序号+工种名'格式，如'5数铣10钳20数车'"],
                stats={"total": 0, "internal": 0, "external": 0, "unknown": 0},
                original_input=original_input,
                normalized_input=normalized
            )

        # Step 2: 工种识别
        operations = []
        stats = {"total": 0, "internal": 0, "external": 0, "unknown": 0}
        seen_seqs = set()

        for seq_str, op_type_name in matches:
            seq = int(seq_str)
            op_type_name = op_type_name.strip()

            # 检查工序号重复
            if seq in seen_seqs:
                warnings.append(f"工序号 {seq} 重复出现，将保留第一个")
                continue
            seen_seqs.add(seq)

            stats["total"] += 1

            op_type = op_types.get(op_type_name)

            if op_type:
                is_internal = op_type.category == 'internal'
                is_recognized = True
            else:
                # 未识别的工种，默认为外部
                is_internal = False
                is_recognized = False
                warnings.append(f"工种'{op_type_name}'未在系统中配置，已默认标记为外部工序")
                stats["unknown"] += 1

            source = 'internal' if is_internal else 'external'
            if is_internal:
                stats["internal"] += 1
            else:
                stats["external"] += 1

            op = ParsedOperation(
                seq=seq,
                op_type_name=op_type_name,
                op_type_id=op_type.op_type_id if op_type else None,
                source=source,
                is_recognized=is_recognized
            )

            # 外部工序：匹配供应商
            if not is_internal:
                supplier_info = suppliers.get(op_type_name)
                if supplier_info:
                    op.supplier_id = supplier_info[0]
                    op.default_days = supplier_info[1]
                else:
                    op.default_days = 1.0  # 默认1天

            operations.append(op)

        # 按工序号排序
        operations.sort(key=lambda x: x.seq)

        # Step 3: 识别连续外部工序组
        external_groups = self._identify_external_groups(operations, part_no)

        # 确定解析状态
        if errors:
            status = ParseStatus.FAILED
        elif warnings:
            status = ParseStatus.PARTIAL
        else:
            status = ParseStatus.SUCCESS

        result = ParseResult(
            status=status,
            operations=operations,
            external_groups=external_groups,
            warnings=warnings,
            errors=errors,
            stats=stats,
            original_input=original_input,
            normalized_input=normalized
        )

        # 记录日志
        if self.logger:
            self.logger.info(
                f"Route parsed for {part_no}: "
                f"{stats['total']} ops ({stats['internal']} internal, "
                f"{stats['external']} external, {stats['unknown']} unknown)"
            )

        return result

    def _preprocess(self, route_string: str) -> str:
        """预处理工艺路线字符串"""
        if not route_string:
            return ""

        # 1. 去除首尾空格
        result = route_string.strip()

        # 2. 移除常见分隔符
        result = re.sub(self.SEPARATORS, '', result)

        # 3. 全角数字转半角
        full_to_half = str.maketrans('０１２３４５６７８９', '0123456789')
        result = result.translate(full_to_half)

        return result

    def _build_supplier_map(self) -> dict:
        """构建 工种名 -> (supplier_id, default_days) 映射"""
        result = {}
        for supplier in self.suppliers_repo.get_all():
            if supplier.op_type_id:
                op_type = self.op_types_repo.get_by_id(supplier.op_type_id)
                if op_type:
                    result[op_type.name] = (supplier.supplier_id, supplier.default_days)
        return result

    def _identify_external_groups(self, operations: List[ParsedOperation],
                                   part_no: str) -> List[ExternalGroup]:
        """识别连续外部工序组"""
        groups = []
        current_group = None
        group_counter = 1

        for op in operations:
            if op.source == 'external':
                if current_group is None:
                    # 开始新组
                    group_id = f"{part_no}_EXT_{group_counter}"
                    current_group = ExternalGroup(
                        group_id=group_id,
                        start_seq=op.seq,
                        end_seq=op.seq,
                        operations=[op]
                    )
                    op.ext_group_id = group_id
                else:
                    # 继续当前组
                    current_group.end_seq = op.seq
                    current_group.operations.append(op)
                    op.ext_group_id = current_group.group_id
            else:
                if current_group is not None:
                    # 结束当前组
                    groups.append(current_group)
                    current_group = None
                    group_counter += 1

        # 处理最后一组
        if current_group is not None:
            groups.append(current_group)

        return groups

    def validate_format(self, route_string: str) -> Tuple[bool, str]:
        """
        快速验证格式是否可解析（不做完整解析）

        Returns:
            (是否有效, 提示信息)
        """
        if not route_string or not route_string.strip():
            return False, "工艺路线不能为空"

        normalized = self._preprocess(route_string)
        pattern = r'(\d+)([^\d]+)'
        matches = re.findall(pattern, normalized)

        if not matches:
            return False, "格式无效，请使用'工序号+工种名'格式"

        if len(matches) < 1:
            return False, "至少需要一道工序"

        return True, f"格式有效，识别到 {len(matches)} 道工序"
```

## 8. 外部工序删除规则

### 8.1 规则定义

```
┌─────────────────────────────────────────────────────────────┐
│                  外部工序删除规则                            │
└─────────────────────────────────────────────────────────────┘

规则1: 首部连续外部工序 → 可删除
─────────────────────────────────────────────────────────────
  原: [5外][10外][15内][20内]...
  删除 [5外][10外] → [15内][20内]...
  ✅ 允许，不影响内部工序连续性

规则2: 尾部连续外部工序 → 可删除
─────────────────────────────────────────────────────────────
  原: ...[20内][25内][30外][35外][40外]
  删除 [30外][35外][40外] → ...[20内][25内]
  ✅ 允许，不影响内部工序连续性

规则3: 中间外部工序 → 不可单独删除
─────────────────────────────────────────────────────────────
  原: [5内][10内][15外][20内][25内]
  删除 [15外] → 工序10和工序20之间断档
  ❌ 不允许

规则4: 中间连续外部工序 → 整组都不可删
─────────────────────────────────────────────────────────────
  原: [5内][10外][15外][20外][25内]
  删除任意外部 → 工序5和工序25之间断档
  ❌ 不允许

判定公式:
─────────────────────────────────────────────────────────────

  可删除条件：删除后，所有内部工序之间没有外部工序断点

  具体算法：
  1. 标记要删除的工序
  2. 遍历剩余工序
  3. 检查任意两个相邻内部工序之间是否有未删除的外部工序
  4. 如果有，则不允许删除
```

### 8.2 删除验证器代码（方案原始代码，保留）

```python
# core/services/process/deletion_validator.py

from typing import List, Tuple, Set
from dataclasses import dataclass
from enum import Enum

class ValidationResult(Enum):
    ALLOWED = "allowed"
    DENIED = "denied"
    WARNING = "warning"

@dataclass
class Operation:
    seq: int
    source: str  # internal/external
    status: str = 'active'

@dataclass
class DeletionCheckResult:
    """删除检查结果"""
    result: ValidationResult
    can_delete: bool
    message: str
    affected_ops: List[int] = None  # 受影响的工序

class DeletionValidator:
    """外部工序删除验证器"""

    def can_delete(self,
                   operations: List[Operation],
                   to_delete: List[int]) -> DeletionCheckResult:
        """
        检查是否可以删除指定的外部工序

        Args:
            operations: 完整工序列表（按seq排序）
            to_delete: 要删除的工序seq列表

        Returns:
            DeletionCheckResult 检查结果
        """
        to_delete_set = set(to_delete)

        # 过滤掉已删除的工序
        active_ops = [op for op in operations if op.status == 'active']

        # 检查：要删除的必须都是外部工序
        for seq in to_delete:
            op = self._find_op(active_ops, seq)
            if op is None:
                return DeletionCheckResult(
                    result=ValidationResult.DENIED,
                    can_delete=False,
                    message=f"工序 {seq} 不存在或已删除"
                )
            if op.source == 'internal':
                return DeletionCheckResult(
                    result=ValidationResult.DENIED,
                    can_delete=False,
                    message=f"工序 {seq} 是内部工序，不能删除。只能删除外部工序。"
                )

        # 构建删除后的工序列表
        remaining = [op for op in active_ops if op.seq not in to_delete_set]

        # 如果删除后没有工序了
        if not remaining:
            return DeletionCheckResult(
                result=ValidationResult.WARNING,
                can_delete=True,
                message="删除后将没有任何工序，确定继续？"
            )

        # 如果没有剩余内部工序，允许删除
        internal_ops = [op for op in remaining if op.source == 'internal']
        if len(internal_ops) == 0:
            return DeletionCheckResult(
                result=ValidationResult.WARNING,
                can_delete=True,
                message="删除后将没有内部工序，确定继续？"
            )

        if len(internal_ops) == 1:
            return DeletionCheckResult(
                result=ValidationResult.ALLOWED,
                can_delete=True,
                message="可以删除"
            )

        # 检查内部工序之间是否有外部工序断档
        internal_ops.sort(key=lambda x: x.seq)

        for i in range(len(internal_ops) - 1):
            current = internal_ops[i]
            next_op = internal_ops[i + 1]

            # 检查两个内部工序之间是否有未删除的外部工序
            between_ops = [
                op for op in remaining
                if current.seq < op.seq < next_op.seq and op.source == 'external'
            ]

            if between_ops:
                between_seqs = [op.seq for op in between_ops]
                return DeletionCheckResult(
                    result=ValidationResult.DENIED,
                    can_delete=False,
                    message=(
                        f"无法删除：工序 {current.seq} 和 {next_op.seq} 之间"
                        f"存在未删除的外部工序 {between_seqs}，"
                        f"删除后会导致工艺断档。"
                        f"如需删除，请一并删除这些工序。"
                    ),
                    affected_ops=between_seqs
                )

        return DeletionCheckResult(
            result=ValidationResult.ALLOWED,
            can_delete=True,
            message="可以删除"
        )

    def _find_op(self, operations: List[Operation], seq: int) -> Operation:
        """查找指定工序号的工序"""
        for op in operations:
            if op.seq == seq:
                return op
        return None

    def get_deletion_groups(self, operations: List[Operation]) -> List[List[int]]:
        """
        获取可删除的外部工序组
        （首部连续外部工序组、尾部连续外部工序组）

        Returns:
            可删除的工序组列表，每组是一个seq列表
        """
        active_ops = sorted(
            [op for op in operations if op.status == 'active'],
            key=lambda x: x.seq
        )

        if not active_ops:
            return []

        groups = []

        # 检查首部连续外部工序
        head_group = []
        for op in active_ops:
            if op.source == 'external':
                head_group.append(op.seq)
            else:
                break
        if head_group:
            groups.append(head_group)

        # 检查尾部连续外部工序
        tail_group = []
        for op in reversed(active_ops):
            if op.source == 'external':
                tail_group.insert(0, op.seq)
            else:
                break
        if tail_group and tail_group != head_group:
            groups.append(tail_group)

        return groups
```

## 10. 排产算法（多策略排序 + 双资源约束贪心）

### 10.1 多策略排序系统

```
┌─────────────────────────────────────────────────────────────┐
│                    排序策略系统                              │
└─────────────────────────────────────────────────────────────┘

策略1: 优先级优先 (priority_first)
─────────────────────────────────────────────────────────────
  排序规则:
    1. 优先级: critical > urgent > normal
    2. 同优先级按交期升序
    3. 同交期按批次号升序

  适用场景: 有明确紧急件需要优先处理

策略2: 交期优先 (due_date_first)
─────────────────────────────────────────────────────────────
  排序规则:
    1. 交期升序（越早越优先）
    2. 无交期的排最后
    3. 同交期按优先级
    4. 同优先级按批次号

  适用场景: 按期交付压力大

策略3: 权重混合 (weighted)
─────────────────────────────────────────────────────────────
  计算公式:
    score = priority_weight × priority_score
          + due_weight × due_score
          （V1.1 起不再包含齐套权重项；齐套约束可选：enforce_ready=yes 时作为“是否允许排产”的硬约束）

  默认权重:
    priority_weight = 0.4
    due_weight = 0.5
    ready_weight = 0.1（预留字段：前端不暴露、算法不使用）

  分数计算:
    priority_score: critical=100, urgent=60, normal=20
    due_score: 100 - min(max(days_left, 0), 100)  # 越紧迫分越高（逾期按 0 天处理；封顶 100）
    （V1.1：不再计算 ready_score；启用齐套约束（enforce_ready=yes）时，仅允许 ready_status=yes 的批次参与）

  适用场景: 综合考虑多因素

策略4: 先进先出 (fifo)
─────────────────────────────────────────────────────────────
  排序规则:
    1. 按批次创建时间升序
    2. 同时间按批次号升序

  适用场景: 简单公平调度


策略配置表:
```

### 10.2 策略实现代码（方案原始代码，保留）

```python
# core/algorithms/sort_strategies.py

from abc import ABC, abstractmethod
from datetime import datetime, date
from typing import List, Dict, Any
from dataclasses import dataclass
from enum import Enum

class SortStrategy(Enum):
    PRIORITY_FIRST = "priority_first"
    DUE_DATE_FIRST = "due_date_first"
    WEIGHTED = "weighted"
    FIFO = "fifo"

@dataclass
class BatchForSort:
    """用于排序的批次信息"""
    batch_id: str
    priority: str  # normal/urgent/critical
    due_date: date = None
    ready_status: str = 'no'  # yes/no/partial
    created_at: datetime = None

class BaseSortStrategy(ABC):
    """排序策略基类"""

    @abstractmethod
    def sort(self, batches: List[BatchForSort]) -> List[BatchForSort]:
        """执行排序"""
        pass

    @abstractmethod
    def get_name(self) -> str:
        """策略名称"""
        pass

class PriorityFirstStrategy(BaseSortStrategy):
    """优先级优先策略"""

    PRIORITY_ORDER = {'critical': 0, 'urgent': 1, 'normal': 2}

    def sort(self, batches: List[BatchForSort]) -> List[BatchForSort]:
        def sort_key(batch: BatchForSort):
            priority_rank = self.PRIORITY_ORDER.get(batch.priority, 99)
            due_rank = batch.due_date if batch.due_date else date.max
            return (priority_rank, due_rank, batch.batch_id)

        return sorted(batches, key=sort_key)

    def get_name(self) -> str:
        return "优先级优先"

class DueDateFirstStrategy(BaseSortStrategy):
    """交期优先策略"""

    PRIORITY_ORDER = {'critical': 0, 'urgent': 1, 'normal': 2}

    def sort(self, batches: List[BatchForSort]) -> List[BatchForSort]:
        def sort_key(batch: BatchForSort):
            # 无交期的排最后
            has_due = 0 if batch.due_date else 1
            due_rank = batch.due_date if batch.due_date else date.max
            priority_rank = self.PRIORITY_ORDER.get(batch.priority, 99)
            return (has_due, due_rank, priority_rank, batch.batch_id)

        return sorted(batches, key=sort_key)

    def get_name(self) -> str:
        return "交期优先"

class WeightedStrategy(BaseSortStrategy):
    """权重混合策略"""

    PRIORITY_SCORE = {'critical': 100, 'urgent': 60, 'normal': 20}

    def __init__(self, priority_weight=0.4, due_weight=0.5):
        self.priority_weight = priority_weight
        self.due_weight = due_weight

    def sort(self, batches: List[BatchForSort]) -> List[BatchForSort]:
        today = date.today()

        def calc_score(batch: BatchForSort) -> float:
            # 优先级分数
            priority_score = self.PRIORITY_SCORE.get(batch.priority, 0)

            # 交期分数 (越紧迫分越高)
            if batch.due_date:
                days_left = (batch.due_date - today).days
                due_score = 100 - min(max(days_left, 0), 100)
            else:
                due_score = 0  # 无交期不紧迫

            # 加权总分
            total = (
                self.priority_weight * priority_score +
                self.due_weight * due_score
            )

            return total

        # 按分数降序排序
        return sorted(batches, key=lambda b: (-calc_score(b), b.batch_id))

    def get_name(self) -> str:
        return "权重混合"

class FIFOStrategy(BaseSortStrategy):
    """先进先出策略"""

    def sort(self, batches: List[BatchForSort]) -> List[BatchForSort]:
        def sort_key(batch: BatchForSort):
            created = batch.created_at if batch.created_at else datetime.max
            return (created, batch.batch_id)

        return sorted(batches, key=sort_key)

    def get_name(self) -> str:
        return "先进先出"

class StrategyFactory:
    """策略工厂"""

    _strategies = {
        SortStrategy.PRIORITY_FIRST: PriorityFirstStrategy,
        SortStrategy.DUE_DATE_FIRST: DueDateFirstStrategy,
        SortStrategy.WEIGHTED: WeightedStrategy,
        SortStrategy.FIFO: FIFOStrategy,
    }

    @classmethod
    def create(cls, strategy: SortStrategy, **kwargs) -> BaseSortStrategy:
        """创建策略实例"""
        strategy_class = cls._strategies.get(strategy)
        if not strategy_class:
            raise ValueError(f"Unknown strategy: {strategy}")

        if strategy == SortStrategy.WEIGHTED:
            return strategy_class(**kwargs)
        return strategy_class()

    @classmethod
    def get_available_strategies(cls) -> List[Dict[str, str]]:
        """获取所有可用策略"""
        return [
            {"key": s.value, "name": cls.create(s).get_name()}
            for s in SortStrategy
        ]
```

### 10.3 双重资源约束算法（贪心）

> 关键点：
> - 内部工序：占用 **设备 + 人员** 双资源时间线
> - 外部工序：不占内部资源，只占用外协周期（天）
> - 前后约束：同一批次按工序顺序串行推进（前序完工后才可开始后序）
> - 工作日历：支持可用工时、效率系数、优先级是否允许排产

```python
# core/algorithms/greedy/  (package; main impl: core/algorithms/greedy/scheduler.py; entry: core.algorithms.greedy)

from datetime import datetime, timedelta
from typing import Dict, List, Optional
from dataclasses import dataclass
import logging

from .sort_strategies import (
    SortStrategy, StrategyFactory, BatchForSort
)

@dataclass
class ScheduleResult:
    """排程结果"""
    op_id: int                      # BatchOperations.id
    op_code: str                    # 工序编码
    batch_id: str
    seq: int
    machine_id: Optional[str] = None
    operator_id: Optional[str] = None
    start_time: Optional[datetime] = None
    end_time: Optional[datetime] = None
    source: str = 'internal'

@dataclass
class ScheduleSummary:
    """排程摘要"""
    success: bool
    total_ops: int
    scheduled_ops: int
    failed_ops: int
    warnings: List[str]
    errors: List[str]
    duration_seconds: float

class GreedyScheduler:
    """贪心排产算法"""

    def __init__(self, calendar_service, config_service, logger=None):
        self.calendar = calendar_service
        self.config = config_service
        self.logger = logger or logging.getLogger(__name__)

    def schedule(self,
                 operations: List,
                 batches: Dict,
                 strategy: SortStrategy = None,
                 strategy_params: Dict = None) -> tuple:
        """
        执行排产

        Args:
            operations: 待排工序列表
            batches: 批次信息字典 {batch_id: Batch}
            strategy: 排序策略 (可选，默认从配置读取)
            strategy_params: 策略参数 (用于权重策略)

        Returns:
            (排程结果列表, 排程摘要)
        """
        start_time = datetime.now()
        warnings = []
        errors = []

        # 获取排序策略
        if strategy is None:
            strategy_key = self.config.get('sort_strategy', 'priority_first')
            strategy = SortStrategy(strategy_key)

        if strategy_params is None and strategy == SortStrategy.WEIGHTED:
            strategy_params = {
                'priority_weight': float(self.config.get('priority_weight', 0.4)),
                'due_weight': float(self.config.get('due_weight', 0.5)),
            }

        # 创建策略并排序批次
        sorter = StrategyFactory.create(strategy, **(strategy_params or {}))
        batch_list = [
            BatchForSort(
                batch_id=b.batch_id,
                priority=b.priority,
                due_date=b.due_date,
                ready_status=b.ready_status,
                created_at=b.created_at
            )
            for b in batches.values()
        ]
        sorted_batches = sorter.sort(batch_list)
        batch_order = {b.batch_id: i for i, b in enumerate(sorted_batches)}

        # 按批次优先级和工序顺序排序工序
        sorted_ops = sorted(
            operations,
            key=lambda op: (batch_order.get(op.batch_id, 999), op.seq)
        )

        self.logger.info(
            f"Scheduling {len(sorted_ops)} operations from {len(batches)} batches "
            f"using strategy: {sorter.get_name()}"
        )

        # 资源占用追踪
        machine_timeline = {}   # machine_id -> [(start, end), ...]
        operator_timeline = {}  # operator_id -> [(start, end), ...]
        batch_progress = {}     # batch_id -> 最后完成时间

        results = []
        scheduled_count = 0
        failed_count = 0

        for op in sorted_ops:
            try:
                batch = batches[op.batch_id]

                if op.source == 'external':
                    # ─────────────────────────────────────────
                    # 外部供应商：不占资源，只占时间
                    # ─────────────────────────────────────────
                    result = self._schedule_external(op, batch, batch_progress)
                else:
                    # ─────────────────────────────────────────
                    # 内部工序：双重资源约束
                    # ─────────────────────────────────────────
                    result = self._schedule_internal(
                        op, batch, batch_progress,
                        machine_timeline, operator_timeline
                    )

                if result:
                    results.append(result)
                    batch_progress[op.batch_id] = result.end_time
                    scheduled_count += 1
                else:
                    failed_count += 1
                    errors.append(f"工序 {op.op_code} 排产失败")

            except Exception as e:
                failed_count += 1
                errors.append(f"工序 {op.op_code} 排产异常: {str(e)}")
                self.logger.error(f"Schedule error for {op.op_code}: {e}")

        # 生成摘要
        duration = (datetime.now() - start_time).total_seconds()
        summary = ScheduleSummary(
            success=failed_count == 0,
            total_ops=len(operations),
            scheduled_ops=scheduled_count,
            failed_ops=failed_count,
            warnings=warnings,
            errors=errors,
            duration_seconds=duration
        )

        self.logger.info(
            f"Scheduling completed: {scheduled_count}/{len(operations)} ops, "
            f"{failed_count} failed, took {duration:.2f}s"
        )

        return results, summary

    def _schedule_external(self, op, batch, batch_progress) -> ScheduleResult:
        """排产外部工序"""
        prev_end = batch_progress.get(op.batch_id, datetime.now())

        # 计算外部周期
        ext_days = op.ext_days or 1.0

        start = prev_end
        end = self.calendar.add_calendar_days(start, ext_days)

        return ScheduleResult(
            op_id=op.id,
            op_code=op.op_code,
            batch_id=op.batch_id,
            seq=op.seq,
            start_time=start,
            end_time=end,
            source='external'
        )

    def _schedule_internal(self, op, batch, batch_progress,
                           machine_timeline, operator_timeline) -> ScheduleResult:
        """排产内部工序（双重资源约束）"""

        # 1. 前道工序约束
        prev_end = batch_progress.get(op.batch_id, datetime.now())

        # 2. 设备可用时间
        machine_available = self._get_resource_available(
            machine_timeline, op.machine_id
        )

        # 3. 人员可用时间
        operator_available = self._get_resource_available(
            operator_timeline, op.operator_id
        )

        # 4. 取三者最晚时间
        earliest = max(prev_end, machine_available, operator_available)

        # 5. 工作日历约束（支持人员专属日历：传入 operator_id 后优先使用 OperatorCalendar）
        earliest = self.calendar.adjust_to_working_time(
            earliest,
            priority=batch.priority,
            operator_id=op.operator_id,
        )

        # 6. 计算工时
        total_hours = op.setup_hours + op.unit_hours * batch.quantity

        # 7. 考虑效率系数（支持人员专属日历）
        efficiency = self.calendar.get_efficiency(earliest, operator_id=op.operator_id)
        if efficiency and efficiency < 1.0:
            total_hours = total_hours / efficiency

        # 8. 计算结束时间
        end = self.calendar.add_working_hours(
            earliest,
            total_hours,
            priority=batch.priority,
            operator_id=op.operator_id,
        )

        # 9. 更新资源占用
        if op.machine_id:
            self._occupy_resource(machine_timeline, op.machine_id, earliest, end)
        if op.operator_id:
            self._occupy_resource(operator_timeline, op.operator_id, earliest, end)

        return ScheduleResult(
            op_id=op.id,
            op_code=op.op_code,
            batch_id=op.batch_id,
            seq=op.seq,
            machine_id=op.machine_id,
            operator_id=op.operator_id,
            start_time=earliest,
            end_time=end,
            source='internal'
        )

    def _get_resource_available(self, timeline: Dict, resource_id: str) -> datetime:
        """获取资源最早可用时间"""
        if not resource_id:
            return datetime.now()
        if resource_id not in timeline or not timeline[resource_id]:
            return datetime.now()

        # 返回最后一个占用时段的结束时间
        return max(end for _, end in timeline[resource_id])

    def _occupy_resource(self, timeline: Dict, resource_id: str,
                         start: datetime, end: datetime):
        """记录资源占用"""
        if not resource_id:
            return
        if resource_id not in timeline:
            timeline[resource_id] = []
        timeline[resource_id].append((start, end))
```

#### 10.5.2 派工模式（dispatch_mode=sgs）

- **batch_order（默认）**：先按批次策略得到 batch 顺序，再按工序顺序推进排产（10.1~10.4 描述的主流程）。
- **sgs（Serial SGS）**：维护“就绪集合（eligible set）”，每次选择一个最紧急/最优的就绪工序进行派工。
  - 规则实现见：`core/algorithms/dispatch_rules.py`（slack/cr/atc）。
  - 适用：希望提高利用率、减少局部空闲的场景；代价是排产结果更“动态”，对比/复盘时建议结合 `result_summary.algo.attempts`。
  - 健壮性约定：
    - 评分阶段会对关键 ID（如 `machine_id/operator_id/batch_id/ext_group_id`）做 `str(...).strip()` 规范化，避免上游误传 int/None/对象导致 `.strip()` 崩溃。
    - 若资源缺失/工时不可估算/评分异常：该候选会按惩罚项（penalty key）降级参与比较（而非静默跳过），避免出现 `best_pair=None -> candidates[0]` 的顺序依赖退化。
    - `dispatch_rules` 的 `cr/atc` 计算会过滤非有限值（NaN/Inf）：`proc_hours/avg_proc_hours` 非有限或 `<=0` 时按回退尺度处理，避免 key 被非有限值污染。
    - `sgs` 评分阶段会过滤非有限工时样本（`proc_samples`）并对非有限 `total_hours` 施加惩罚；效率值非有限或 `<=0` 时按 `1.0` 回退，避免产生零时长异常工序。

#### 10.5.3 自动资源分配（auto_assign_enabled=yes）

- 仅对 **内部工序** 生效：当 `BatchOperations.machine_id/operator_id` 缺省时，算法会在可用资源池中自动选择匹配的设备+人员组合。
- **安全约束**：只在“原本缺省资源”的工序上回写（避免覆盖人工已选）；回写发生在排产落库阶段。

#### 10.5.4 算法改进模式（algo_mode=improve）

- 在 `greedy` 的基础上，增加：
  - 多起点（multi-start）：尝试 `dispatch_mode × strategy × dispatch_rule` 组合（其中 `dispatch_rule` 仅在 `dispatch_mode=sgs` 时展开）
  - 目标函数择优（objective）
  - 局部搜索/轻量 ILS 重启（受 time_budget_seconds 限制）
- 输出：`ScheduleHistory.result_summary.algo.best_score/best_score_schema/comparison_metric/config_snapshot/metrics/improvement_trace` 便于复盘。

#### 10.5.5 OR-Tools 瓶颈工种 warm-start（ortools_enabled=yes）

- 仅作为“高质量起点”：对瓶颈工种抽取单机序列子问题（CP-SAT）求解一个批次顺序，供启发式排产作为起点。
- 可选依赖：`ortools` 不在 V1 默认依赖里；未安装时会自动返回 None，不影响主流程。
- 代码见：`core/algorithms/ortools_bottleneck.py`。
 - 健壮性约定：
   - warm-start 会过滤 NaN/Inf 等非有限工时；任何异常都会返回 None（失败即降级），不影响主流程。
   - 交期（due_date）允许早于 start_dt：用于更稳定地区分/截断“更逾期”的批次（仅影响 warm-start 顺序选择）。 

#### 10.5.6 冻结窗口（freeze_window_enabled=yes）

- 启用后会复用上一版本在窗口内的排程结果，落库到 `Schedule.lock_status='locked'`，其余工序继续由算法排产。
- 用途：减少反复重排导致的“已排程工序大幅变动”。

#### 11.3.2 V1 实现：OpenpyxlBackend

```python
# core/services/common/openpyxl_backend.py
from typing import List, Dict, Any, Optional
import openpyxl

class OpenpyxlBackend:
    """V1 Excel 后端：只依赖 openpyxl"""

    def read(self, file_path: str, sheet: Optional[str] = None) -> List[Dict[str, Any]]:
        wb = openpyxl.load_workbook(file_path, data_only=True)
        ws = wb[sheet] if sheet else wb.active

        rows = list(ws.iter_rows(values_only=True))
        if not rows:
            return []

        headers = [str(h).strip() if h is not None else "" for h in rows[0]]
        result: List[Dict[str, Any]] = []

        for raw in rows[1:]:
            # 跳过空行
            if raw is None or all(v is None or str(v).strip() == "" for v in raw):
                continue

            item: Dict[str, Any] = {}
            for idx, key in enumerate(headers):
                if not key:
                    continue
                val = raw[idx] if idx < len(raw) else None
                # 统一空字符串/空白
                if isinstance(val, str):
                    val = val.strip()
                    if val == "":
                        val = None
                item[key] = val

            result.append(item)

        return result

    def write(self, rows: List[Dict[str, Any]], file_path: str, sheet: str = "Sheet1"):
        wb = openpyxl.Workbook()
        ws = wb.active
        ws.title = sheet

        if not rows:
            wb.save(file_path)
            return

        # 以第一行 keys 作为列（也可由上层传入列定义）
        headers = list(rows[0].keys())
        ws.append(headers)

        for r in rows:
            ws.append([r.get(h) for h in headers])

        wb.save(file_path)
```

### 11.5 通用导入：预览/差异对比/模式（APPEND/OVERWRITE/REPLACE）

> 原方案 v2 提供了完整的“导入预览”数据结构（ImportMode/RowStatus/ImportPreviewRow/ImportResult）。
> V1 保留这些结构，但实现从 pandas DataFrame 改为 `List[Dict]`。

```python
# core/services/common/excel_service.py

from typing import List, Dict, Tuple, Any, Callable, Optional
from dataclasses import dataclass, field
from enum import Enum

class ImportMode(Enum):
    APPEND = "append"       # 追加，不影响已有
    OVERWRITE = "overwrite" # 相同ID覆盖
    REPLACE = "replace"     # 清空后导入

class RowStatus(Enum):
    NEW = "new"
    UPDATE = "update"
    UNCHANGED = "unchanged"
    ERROR = "error"
    SKIP = "skip"

@dataclass
class ImportPreviewRow:
    """导入预览行"""
    row_num: int
    status: RowStatus
    data: Dict[str, Any]
    message: str = ""
    changes: Dict[str, Tuple] = field(default_factory=dict)  # {field: (old, new)}

@dataclass
class ImportResult:
    """导入结果"""
    success: bool
    total: int
    new_count: int
    update_count: int
    skip_count: int
    error_count: int
    errors: List[Dict]  # [{row: int, field: str, message: str}]
    warnings: List[str]

class ExcelService:
    """通用Excel导入导出服务（V1：openpyxl-only + backend）"""

    def __init__(self, backend, logger=None, op_logger=None, template_dir: str = "templates_excel"):
        self.backend = backend
        self.logger = logger
        self.op_logger = op_logger
        self.template_dir = template_dir

    def preview_import(self,
                       rows: List[Dict[str, Any]],
                       id_column: str,
                       existing_data: Dict[str, Any],
                       validators: List[Callable[[Dict[str, Any]], Optional[str]]] = None,
                       mode: ImportMode = ImportMode.OVERWRITE) -> List[ImportPreviewRow]:
        preview: List[ImportPreviewRow] = []

        for idx, row_dict in enumerate(rows):
            row_num = idx + 2  # Excel行号 (1-based + 标题行)
            row_id = row_dict.get(id_column)

            if row_id is None or str(row_id).strip() == "":
                preview.append(ImportPreviewRow(
                    row_num=row_num,
                    status=RowStatus.ERROR,
                    data=row_dict,
                    message=f"{id_column}不能为空"
                ))
                continue

            row_id = str(row_id).strip()
            row_dict[id_column] = row_id

            # 执行验证
            validation_error = None
            if validators:
                for validator in validators:
                    err = validator(row_dict)
                    if err:
                        validation_error = err
                        break

            if validation_error:
                preview.append(ImportPreviewRow(
                    row_num=row_num,
                    status=RowStatus.ERROR,
                    data=row_dict,
                    message=validation_error
                ))
                continue

            if row_id in existing_data:
                if mode == ImportMode.APPEND:
                    preview.append(ImportPreviewRow(
                        row_num=row_num,
                        status=RowStatus.SKIP,
                        data=row_dict,
                        message='已存在，按“追加”模式将跳过'
                    ))
                else:
                    existing = existing_data[row_id]
                    changes = self._calc_changes(existing, row_dict)
                    if changes:
                        preview.append(ImportPreviewRow(
                            row_num=row_num,
                            status=RowStatus.UPDATE,
                            data=row_dict,
                            message='将更新现有记录',
                            changes=changes
                        ))
                    else:
                        preview.append(ImportPreviewRow(
                            row_num=row_num,
                            status=RowStatus.UNCHANGED,
                            data=row_dict,
                            message='无变更'
                        ))
            else:
                preview.append(ImportPreviewRow(
                    row_num=row_num,
                    status=RowStatus.NEW,
                    data=row_dict,
                    message='新增记录'
                ))

        return preview

    def _calc_changes(self, existing: Any, new_data: Dict[str, Any]) -> Dict[str, Tuple]:
        changes: Dict[str, Tuple] = {}
        existing_dict = existing if isinstance(existing, dict) else vars(existing)

        for key, new_val in new_data.items():
            if key in existing_dict:
                old_val = existing_dict[key]
                if old_val != new_val:
                    changes[key] = (old_val, new_val)
        return changes
```

```python
# core/infrastructure/transaction.py

from functools import wraps
from contextlib import contextmanager
import logging

logger = logging.getLogger(__name__)

class TransactionManager:
    """事务管理器（最外层 BEGIN/COMMIT，嵌套层 SAVEPOINT）"""

    def __init__(self, db_connection):
        self.conn = db_connection

    @contextmanager
    def transaction(self):  # 省略 _inc_depth/_dec_depth 等辅助函数
        conn = self.conn
        _inc_depth(conn)
        owns_tx = False
        uses_savepoint = False
        sp_name = None
        try:
            depth = int(_depth_map().get(id(conn), 0) or 0)
            if depth == 1:
                owns_tx = not bool(getattr(conn, "in_transaction", False))

            if depth == 1 and owns_tx:
                conn.execute("BEGIN")
            else:
                sp_name = f"aps_tx_{id(conn)}_{depth}"
                uses_savepoint = True
                conn.execute(f"SAVEPOINT {sp_name}")

            yield conn

            if uses_savepoint:
                conn.execute(f"RELEASE SAVEPOINT {sp_name}")
            elif depth == 1 and owns_tx:
                conn.commit()
        except Exception as e:
            if uses_savepoint:
                conn.execute(f"ROLLBACK TO SAVEPOINT {sp_name}")
                conn.execute(f"RELEASE SAVEPOINT {sp_name}")
            else:
                conn.rollback()
            logger.error(f"Transaction rolled back: {e}")
            raise
        finally:
            _dec_depth(conn)

def transactional(func):
    """事务装饰器"""
    @wraps(func)
    def wrapper(self, *args, **kwargs):
        with self.tx_manager.transaction():
            return func(self, *args, **kwargs)
    return wrapper
```

```python
# core/infrastructure/locking.py  （已废弃，不实现）

import sqlite3
from contextlib import contextmanager
from datetime import datetime, timedelta
import logging

logger = logging.getLogger(__name__)

class ResourceLock:
    """资源锁（基于数据库实现）"""

    def __init__(self, db_connection):
        self.conn = db_connection
        self._ensure_lock_table()

    def _ensure_lock_table(self):
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS ResourceLocks (
                resource_type TEXT NOT NULL,
                resource_id TEXT NOT NULL,
                locked_by TEXT NOT NULL,
                locked_at DATETIME DEFAULT CURRENT_TIMESTAMP,
                expires_at DATETIME NOT NULL,
                PRIMARY KEY (resource_type, resource_id)
            )
        """)
        self.conn.commit()

    def acquire(self, resource_type: str, resource_id: str,
                locked_by: str = 'system',
                timeout_seconds: int = 300) -> bool:
        expires_at = datetime.now() + timedelta(seconds=timeout_seconds)

        try:
            self.conn.execute(
                "DELETE FROM ResourceLocks WHERE expires_at < ?",
                (datetime.now(),)
            )
            self.conn.execute("""
                INSERT INTO ResourceLocks
                (resource_type, resource_id, locked_by, expires_at)
                VALUES (?, ?, ?, ?)
            """, (resource_type, resource_id, locked_by, expires_at))
            self.conn.commit()
            logger.debug(f"Lock acquired: {resource_type}/{resource_id} by {locked_by}")
            return True
        except sqlite3.IntegrityError:
            logger.warning(f"Lock failed: {resource_type}/{resource_id} already locked")
            return False

    def release(self, resource_type: str, resource_id: str,
                locked_by: str = 'system') -> bool:
        cursor = self.conn.execute("""
            DELETE FROM ResourceLocks
            WHERE resource_type = ? AND resource_id = ? AND locked_by = ?
        """, (resource_type, resource_id, locked_by))

        self.conn.commit()
        released = cursor.rowcount > 0

        if released:
            logger.debug(f"Lock released: {resource_type}/{resource_id}")

        return released

    @contextmanager
    def lock(self, resource_type: str, resource_id: str,
             locked_by: str = 'system', timeout_seconds: int = 300):
        if not self.acquire(resource_type, resource_id, locked_by, timeout_seconds):
            raise ResourceLockError(
                f"无法获取资源锁: {resource_type}/{resource_id}，"
                f"可能有其他操作正在进行"
            )

        try:
            yield
        finally:
            self.release(resource_type, resource_id, locked_by)

class ResourceLockError(Exception):
    pass
```

```python
@dataclass
class AppError(Exception):
    """应用异常基类"""
    code: ErrorCode
    message: str
    details: Optional[Dict[str, Any]] = None
    cause: Optional[Exception] = None

    def __str__(self):
        return f"[{self.code.value}] {self.message}"

    def to_dict(self) -> dict:
        result = {
            "success": False,
            "error": {
                "code": self.code.value,
                "message": self.message
            }
        }
        if self.details:
            result["error"]["details"] = self.details
        return result

class ValidationError(AppError):
    def __init__(self, message: str, field: str = None, **kwargs):
        details = {"field": field} if field else None
        super().__init__(
            code=ErrorCode.VALIDATION_ERROR,
            message=message,
            details=details,
            **kwargs
        )

class NotFoundError(AppError):
    def __init__(self, resource_type: str, resource_id: str):
        super().__init__(
            code=ErrorCode.NOT_FOUND,
            message=f"{resource_type} '{resource_id}' 不存在",
            details={"resource_type": resource_type, "resource_id": resource_id}
        )

class BusinessError(AppError):
    pass


@dataclass
class ApiResponse:
    """统一API响应"""
    success: bool
    data: Any = None
    error: Dict = None
    meta: Dict = None

    def to_dict(self) -> dict:
        result = {"success": self.success}
        if self.data is not None:
            result["data"] = self.data
        if self.error:
            result["error"] = self.error
        if self.meta:
            result["meta"] = self.meta
        return result

def success_response(data=None, meta=None) -> dict:
    return ApiResponse(success=True, data=data, meta=meta).to_dict()

def error_response(code: ErrorCode, message: str, details=None) -> dict:
    error = {"code": code.value, "message": message}
    if details:
        error["details"] = details
    return ApiResponse(success=False, error=error).to_dict()


# NOTE (2026-01-28): Flask error handlers moved to web/error_handlers.py.
def register_error_handlers(app):
    """注册Flask错误处理器"""

    @app.errorhandler(AppError)
    def handle_app_error(e: AppError):
        app.logger.warning(f"AppError: {e}")
        return e.to_dict(), 400

    @app.errorhandler(404)
    def handle_not_found(e):
        return error_response(ErrorCode.NOT_FOUND, "请求的资源不存在"), 404

    @app.errorhandler(500)
    def handle_internal_error(e):
        app.logger.error(f"Internal error: {e}\n{traceback.format_exc()}")
        return error_response(
            ErrorCode.UNKNOWN_ERROR,
            "服务器内部错误，请查看日志"
        ), 500
```

## 16. 数据备份机制（退出自动备份 + 手动备份/恢复）

### 16.1 最终策略

- V1 不使用后台定时线程，不引入 `schedule` 等常驻调度器。
- `BackupManager` 保留 4 个核心能力：`backup()` / `restore()` / `cleanup_old_backups()` / `list_backups()`。
- 退出自动备份：通过 `atexit.register` 触发，受 `auto_backup_enabled` 控制，生成 `*_exit.db`。
- 自动备份 / 自动清理：改为**按请求触发**的分钟级检查；配置落库 `SystemConfig`，上次执行信息落库 `SystemJobState`。
- 备份文件命名约定：
  - 手动备份：`aps_backup_<ts>_manual.db`
  - 自动备份：`aps_backup_<ts>_auto.db`
  - 退出备份：`aps_backup_<ts>_exit.db`
  - 恢复前自动备份：`aps_backup_<ts>_before_restore.db`
  - 迁移前备份：`aps_backup_<ts>_before_migrate_v<from>_to_v<to>.db`
- 恢复与迁移共享同一维护窗口；restore 完成后，`ensure_schema()` 必须在**同一窗口**内执行。

### 16.2 维护窗口与请求门禁

- 维护窗口实现位于 `core/infrastructure/backup.py`：
  - 进程内互斥：`_MAINT_MUTEX = threading.RLock()`
  - 线程内可重入上下文：`_MAINT_CONTEXT.state`
  - 跨请求/跨进程软锁文件：`{db_path}.maintenance.lock`
- 进入维护窗口的动作：
  - `BackupManager.restore()`
  - `core/infrastructure/database.py::_migrate_with_backup()`
  - `web/routes/system_backup.py::backup_restore()` 外层 `restore_flow`
- 普通 `backup()` 现在也直接进入 `maintenance_window(action="backup")`；这样手动备份、退出自动备份、restore、migrate 共用同一套维护互斥，不再保留“先检查允许、后真正开始备份”的竞态窗。
- stale lock 采用“保守自愈”：只有锁文件里的 `pid` 已不存在，且锁年龄超过阈值，才允许自动清理；否则继续按 active 处理，避免误删真实维护锁。
- `web/bootstrap/factory.py` 的 `before_request` 会检测 `is_maintenance_window_active(DATABASE_PATH)`：
  - 白名单：`/static*`、`/system/health`、`/system/runtime/shutdown`
  - 页面请求：返回 `error.html`，标题 `系统维护中`，HTTP 503
  - API / JSON 请求：返回 `{"success": false, "error": {"code": "503", "message": "系统正在维护中，请稍后重试。"}}`
- `_run_exit_backup()` 在维护窗口期间必须**跳过且快速返回**，不能长时间等待，更不能生成坏备份。

### 16.3 BackupManager 当前合同

```python
@dataclass(frozen=True)
class RestoreResult:
    ok: bool
    code: str
    message: str
    before_restore_path: Optional[str] = None
```

- `backup()`：
  - 先写 `*.tmp`，成功后 `os.replace()` 为最终文件，避免半成品备份。
  - 使用 `sqlite3.backup()` 复制数据库。
  - 对备份副本执行 `PRAGMA integrity_check`；检查失败则不生成最终备份。
- `restore()`：
  - 返回 `RestoreResult`，不再返回裸 `bool`。
  - 进入 `maintenance_window(action="restore")`。
  - 先做 `before_restore` 自动备份。
  - 再把备份文件复制回主库（带 `busy_timeout` 与重试）。
  - 若恢复失败，则尽最大努力回滚到 `before_restore` 备份。
- 当前主要返回码：
  - `success`
  - `busy`
  - `backup_missing`
  - `restore_failed`
  - `restore_failed_rolled_back`
  - `restore_failed_rollback_failed`

### 16.4 恢复成功门槛（路由口径）

- `POST /system/backup/restore` 的成功门槛已经收紧为：
  1. `BackupManager.restore()` 返回 `RestoreResult.ok = True`
  2. 随后的 `ensure_schema()` 也成功完成
  3. 只有满足 1 + 2，才写 restore success 的 `OperationLogs` 并给用户 success flash
- 若 `restore()` 成功但 `ensure_schema()` 失败：
  - 页面提示：`数据库文件已恢复，但后续结构检查失败，请查看日志后再继续使用。`
  - **不得**写 restore success 日志
  - 维护窗口在返回前才释放，避免刚恢复完就被并发请求打断

### 16.5 自动任务与 telemetry 口径

- 自动备份、自动清理备份、自动清理日志都采用“按请求触发 + 节流”策略。
- 三类任务都把**主动作**与**telemetry 写入**拆开：
  - 主动作：真正的备份/删文件/删旧日志
  - telemetry 1：`OperationLogs`
  - telemetry 2：`SystemJobState.last_run_time / last_run_detail`
- `OperationLogs` 与 `SystemJobState` 各自用独立 `TransactionManager(conn).transaction()` 提交：
  - `OperationLogs` 写入失败时，`SystemJobState` 仍然要继续尝试提交
  - `SystemJobState` 写入失败时，只记录 warning，不回滚主动作
- 自动清理日志的主删除动作本身先单独提交，再写 telemetry，避免“日志已删成功，但因为 telemetry 失败又整体回滚”。

### 18.1 UI 模式（经典/现代）与安全链接（safe_url_for）

> 目的：在不改变对外 URL 的前提下，支持“经典/现代”两套模板逐步演进，并在模板与后端版本不一致时避免整页 500。

- **模式**：`v1=经典`、`v2=现代`；默认优先启用 `v2`。
- **优先级**：Cookie（`aps_ui_mode`）→ DB（`SystemConfig.ui_mode`）→ default（`v2`）。
- **模板覆盖（overlay）**：当 `ui_mode=v2` 时，Jinja 模板优先从 `web_new_test/templates/` 加载，缺失则回退到 `templates/`。
- **静态资源**：`/static-v2` 映射到 `web_new_test/static/`（缺文件时返回 404，便于排障；打包时需把该目录随 onedir 一起带上）。
- **safe_url_for**：模板可用 `safe_url_for(endpoint, **values)` 安全生成链接：
  - endpoint 不存在时返回 `None`（入口直接隐藏，而不是抛异常 500）
  - 同时写入 warning 日志（每请求每 endpoint 仅一次），用于定位“运行旧版本/未重启/未重新打包”导致的模板-后端不一致。
  - **真实案例（系统使用说明入口）**：`get_manual_url(endpoint)` 内部使用 `safe_url_for('scheduler.config_manual_page', page=endpoint, src=...)`
    - route 存在且 endpoint 已进入 `ENDPOINT_TO_MANUAL_ID`：链接到页面级说明 `GET /scheduler/config/manual?page=<endpoint>&src=...`
    - route 缺失或 endpoint 未纳入 registry：隐藏入口（避免旧版本/未重启时在页面内渲染失效链接）

## 22. 确认清单（开工前逐条确认）

### 22.1 数据模型修订

- `piece_id` 字段预留，支持单件追溯扩展
- `BatchOperations.id` 使用自增主键
- `op_code` 用于显示与检索（`batch_id + seq`）
- 复合唯一约束：`(batch_id, seq, piece_id)`

### 22.2 字段统一为字符串枚举

- status: `active/inactive/maintain`
- source: `internal/external`
- priority: `normal/urgent/critical`
- ready_status: `yes/no/partial`

### 22.3 工艺解析器增强

- 预处理：去空格、统一分隔符、全角转半角
- 容错：未识别工种默认标记为外部
- 解析报告：success/partial/failed + 统计 + 警告
- 格式验证：快速检查方法 `validate_format`

### 22.4 多策略排序算法

- priority_first（优先级优先）
- due_date_first（交期优先）
- weighted（权重混合，权重可配置）
- fifo（先进先出）

### 22.5 技术规范

- 事务管理：装饰器 + 上下文管理器
- 错误处理：错误码体系 + 统一 API 响应
- 日志：文件滚动 + 操作日志入库
- 备份：退出自动备份 + 手动备份/恢复 + 清理

并发控制：

- 原方案包含资源锁（ResourceLocks），但 **V1 明确删除，不实现**

### 22.6 依赖与交付确认

- Python：3.8
- Excel：openpyxl-only（V1）
- 甘特图：Frappe Gantt 0.6.1（MIT），静态文件本地化
- 打包：PyInstaller 4.10，且必须在 Win7 环境打包

### 22.7 保持不变（复核通过）

| 模块 | 评价 |
|---|---|
| 数据模型 | 设计合理，枚举统一，预留字段恰当 |
| 工艺解析器 | 预处理 + 容错 + 报告，逻辑完整 |
| 排序策略 | 4种策略 + 工厂模式，可扩展 |
| 排产算法 | 贪心 + 双资源约束，V1 MVP 够用 |
| 事务管理 | 设计合理，保留 |
| 错误处理 | 错误码体系完整，保留 |
| 日志策略 | 分级 + 滚动，保留 |
| 模块结构 | 解耦清晰，保留 |
| 实施计划 | 12天合理，甘特图风险已识别 |

### 22.8 关键事务边界（必须按原子操作实现）

1. 批次创建 = 批次记录 + 工序记录
2. 批次删除 = 排程记录 + 工序记录 + 批次记录
3. 排产执行 = 多条排程记录
4. 工艺模板保存 = 零件 + 工序 + 外部组
5. Excel 批量导入 = 多条记录

### 22.9 原方案“资源锁保护操作”（已废弃，仅留档）

原方案建议锁保护：

1. 排产执行（全局锁，同时只能有一个排产任务）
2. 批次状态变更（批次锁）

最终结论：共享数据但仅允许单活用户的场景不需要业务级资源锁，V1 不实现；运行时只保留单活用户锁用于阻止第二个账户进入。
