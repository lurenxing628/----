## 1. 留痕分层（你需要留下哪些“证据”）

### 1.1 开发阶段留痕（本文件记录的核心）
- **提交记录（必须）**：每个步骤完成后至少 1 次提交，commit message 建议包含步骤 ID（如 `P1-03`）。  
- **验收记录（必须）**：在本文档里记录验收时间、负责人、版本号（commit hash）、验收结果、证据位置。
- **证据附件（推荐）**：截图/导出文件/关键日志片段，统一放到 `evidence/` 目录（建议按日期或步骤号分目录）。

### 1.1.1 你提出的“每完成功能就要测试+打点+留痕”（强烈建议按此执行）
- **最小测试（必须）**：每实现一个功能，至少做一次“冒烟测试”（主路径 + 1~2 个常见异常路径）。
- **打点日志（必须）**：关键链路要有清晰的调试日志（开始/结束/耗时/关键参数/异常原因），方便用户排障。
- **留痕核对（必须）**：涉及导入/导出/排产等关键操作时，核对 DB `OperationLogs` / `ScheduleHistory` 是否写入正确。
- **证据归档（必须）**：把本次测试结果与证据放进 `evidence/`，并在本文件的“阶段验收记录”里引用对应路径。

### 1.2 业务操作留痕（系统内置，必须打通）
按《开发文档.md》要求，关键操作必须写入：
- **数据库 `OperationLogs`**：当前已接入导入/导出/排产留痕（detail 为 JSON 字符串；报表中心页面/导出已存在，但报表导出 `OperationLogs` 仍待接入，`action=report` 保持预留）。  

  - 注意：detail 的**键名必须按文档固定**（例如 `filename/mode/time_cost_ms` 等），便于统一报表与审计。
- **数据库 `ScheduleHistory`**：排产历史（Phase7 才会用到，但从 Phase0 就要保留表结构）。

### 1.3 运行诊断留痕（用户排障必须）
- **文件日志**：`logs/aps.log` + `logs/aps_error.log`（滚动日志）。  
- **用户可见信息中文化（硬约束）**：页面提示、接口错误 message、预览行提示、系统日志页展示等必须中文，方便排查。

## 5. Phase 1（通用 Excel 能力）验收留痕要点

### 5.1 必须验证项
- `OpenpyxlBackend.read()` 能读取 `.xlsx` 并返回 `List[Dict]`（无 DataFrame）
- `ExcelService.preview_import()`：
  - 行号：`row_num = idx + 2`（标题行+1）
  - 状态正确：NEW/UPDATE/UNCHANGED/SKIP/ERROR
  - diff 正确：`changes[field] = (old, new)`
  - message 为中文（便于用户定位问题）
- 导入/导出留痕写入 `OperationLogs`：
  - import：detail 至少包含 `filename/mode/total_rows/.../errors_sample/time_cost_ms`
  - export：detail 至少包含 `template_or_export_type/filters/row_count/time_range`

### 5.2 推荐验收路径（本项目已提供 demo）
- 打开：`/excel-demo/`
- 下载模板并导入一次（产生 NEW）
- 再用同一文件导入一次（产生 UPDATE/UNCHANGED 或 SKIP，取决于 mode）
- 导入一个错误文件（如缺“工号/姓名/状态”）观察 ERROR 行提示是否中文清晰

### 5.3 建议证据（SQL）
```sql
-- 查看最近 20 条操作日志（导入/导出）
SELECT id, log_time, log_level, module, action, target_type, target_id, error_code, error_message, detail
FROM OperationLogs
ORDER BY id DESC
LIMIT 20;
```
