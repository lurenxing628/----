# 回转壳体单元智能排产系统（APS）— 系统速查表（V1 / Phase0~6）

> 目标：把**术语/枚举/HTTP接口/数据库表字段/留痕规范/Excel模板**一次性收敛到一份文档，后续开发无需反复研读长文档。
>
> 代码基线：以当前仓库实现为准（见 `app.py`、`schema.sql`、`web/routes/*.py`、`core/*`）。

---

## 全局硬约束（必须遵守）

- **目标环境**：Win7 单机单用户（目标机不装 Python）
- **Python**：3.8.x（打包机/开发机）
- **依赖锁定**：`requirements.txt`（openpyxl-only；不引入 pandas/numpy/schedule）
- **并发/资源锁**：V1 **不实现**（无 `ResourceLocks` 表、无 `core/infrastructure/locking.py`）
- **备份**：退出自动备份（`atexit.register`）+ 手动备份/恢复/清理（不启后台定时线程）
- **留痕**：导入/导出/排产/报表（预留）必须可追溯（文件日志 + DB 表 `OperationLogs` / `ScheduleHistory`）
- **甘特图资源**：Frappe Gantt **0.6.1** 静态资源本地化（`static/js` + `static/css`）

## 术语表（Glossary）

- **Operator（人员）**：操机人员；表 `Operators`
- **Machine（设备）**：设备资源；表 `Machines`
- **OperatorMachine（人员-设备关联）**：人员可操作设备的共享关联表；表 `OperatorMachine`
- **OpType（工种）**：工序类型/工种；用于区分 internal/external；表 `OpTypes`
- **Supplier（供应商）**：外协供应商；可绑定工种并提供默认周期；表 `Suppliers`
- **Part（零件/图号）**：图号为主键；表 `Parts`
- **PartOperations（零件工序模板）**：从工艺路线解析生成的模板工序；表 `PartOperations`
- **ExternalGroup（连续外部工序组）**：连续 external 工序聚合；表 `ExternalGroups`
- **Batch（批次）**：排产对象（后续阶段使用）；表 `Batches`
- **BatchOperations（批次工序）**：从零件模板复制到批次；表 `BatchOperations`
- **Schedule（排程结果）**：排产落库结果；表 `Schedule`
- **WorkCalendar（工作日历）**：日历配置（后续阶段）；表 `WorkCalendar`
- **ScheduleConfig（排产配置）**：排序策略/权重等（后续阶段）；表 `ScheduleConfig`
- **OperationLogs（操作日志）**：关键操作留痕（导入/导出/排产…）；表 `OperationLogs`
- **ScheduleHistory（排产历史）**：排产结果摘要与版本（后续阶段）；表 `ScheduleHistory`

## 枚举值（以 DB 字段/`core/models/enums.py` 为准）

### 通用 Yes/No
- `yes` / `no`

### 人员状态（Operators.status）
- `active`：在岗
- `inactive`：停用/休假

### 设备状态（Machines.status）
- `active`：可用
- `maintain`：维修
- `inactive`：停用

### 工序归属（*.source）
- `internal`：内部工序
- `external`：外部工序

### 批次状态（Batches.status，后续阶段）
- `pending` / `scheduled` / `processing` / `completed` / `cancelled`

### 优先级（Batches.priority，后续阶段）
- `normal` / `urgent` / `critical`

### 齐套状态（Batches.ready_status，后续阶段）
- `yes` / `no` / `partial`

### 日历类型（WorkCalendar.day_type，后续阶段）
- `workday` / `weekend` / `holiday`

### 外部组周期模式（ExternalGroups.merge_mode）
- `separate`：分别设置（每道外部工序 `PartOperations.ext_days` 各自存）
- `merged`：合并设置（组表 `ExternalGroups.total_days` 存；组内 `PartOperations.ext_days` 为 NULL）

## 留痕（OperationLogs / ScheduleHistory）

### OperationLogs（DB 表）字段含义
- `module`：模块名（示例：personnel/equipment/process/excel_demo）
- `action`：动作（当前已落地：`import` / `export`；预留：`schedule` / `report`）
- `target_type`：目标类型（示例：operator/machine/op_type/supplier/part_route/operator_machine/part_operation）
- `detail`：JSON 字符串（键名固定，便于审计/报表）

### Excel 导入 detail 固定键（`action=import`）
由 `core/services/common/excel_audit.py::log_excel_import()` 生成：
- `filename`
- `mode`：append/overwrite/replace
- `time_cost_ms`
- `file_hash`（可选，当前未启用）
- `total_rows/new_count/update_count/skip_count/error_count`
- `errors_sample`：最多 10 条，元素格式 `{"row": <Excel行号>, "message": "..."}`（仅示意）

### Excel 导出 detail 固定键（`action=export`）
由 `core/services/common/excel_audit.py::log_excel_export()` 生成：
- `template_or_export_type`
- `filters`（dict）
- `row_count`
- `time_range`（dict）
- `time_cost_ms`

### ScheduleHistory（后续排产阶段使用）
表存在于 `schema.sql`，字段 `result_summary` 预期为 JSON 字符串；V1 后续阶段需包含：strategy/version/批次数/工序数/结果摘要等。

## 排产策略配置（ScheduleConfig，Phase 6 已接入）

> 说明：配置存于表 `ScheduleConfig`（key/value），供 Phase 7 算法读取。

### 配置键（config_key）
- `sort_strategy`：当前排序策略（允许：`priority_first` / `due_date_first` / `weighted` / `fifo`）
- `priority_weight`：权重模式-优先级权重（默认 `0.4`）
- `due_weight`：权重模式-交期权重（默认 `0.5`）
- `ready_weight`：权重模式-齐套权重（默认 `0.1`）

## 数据库表字段字典（SQLite，来自 `schema.sql`）

> 说明：字段类型以 SQLite 声明为准；**状态/布尔一律字符串枚举**；强依赖外键与唯一性约束（请勿随意改表结构）。

### Operators（人员）
- **PK**：`operator_id`
- 字段：`operator_id/name/status/remark/created_at/updated_at`
- 索引：`idx_operators_status`

### Machines（设备）
- **PK**：`machine_id`
- 字段：`machine_id/name/op_type_id/status/remark/created_at/updated_at`
- 外键：`op_type_id -> OpTypes(op_type_id)`
- 索引：`idx_machines_status/idx_machines_op_type`

### MachineDowntimes（设备停机时间段）
- **PK**：`id`（自增）
- 字段：`id/machine_id/start_time/end_time/reason_code/reason_detail/status/created_at/updated_at`
- 外键：`machine_id -> Machines(machine_id)`（级联删除）
- 索引：`idx_machine_downtimes_machine/idx_machine_downtimes_time`

### OperatorMachine（人员-设备关联，共享表）
- **PK**：`id`（自增）
- **UNIQUE**：`(operator_id, machine_id)`
- 字段：`id/operator_id/machine_id/skill_level/is_primary/created_at`
- 外键：`operator_id -> Operators`（级联删除），`machine_id -> Machines`（级联删除）
- 索引：`idx_operator_machine_operator/idx_operator_machine_machine`

### OperatorSkill（预留：人员-工种能力）
- **PK**：`id`（自增）
- **UNIQUE**：`(operator_id, op_type_id)`
- 字段：`id/operator_id/op_type_id/skill_level/is_primary/created_at`

### OpTypes（工种）
- **PK**：`op_type_id`
- **UNIQUE**：`name`
- 字段：`op_type_id/name/category/default_hours/remark/created_at`

### Suppliers（供应商）
- **PK**：`supplier_id`
- 字段：`supplier_id/name/op_type_id/default_days/status/remark/created_at`
- 外键：`op_type_id -> OpTypes`

### Parts（零件/图号）
- **PK**：`part_no`
- 字段：`part_no/part_name/route_raw/route_parsed/remark/created_at/updated_at`

### PartOperations（零件工序模板）
- **PK**：`id`（自增）
- **UNIQUE**：`(part_no, seq)`
- 字段：
  - 基本：`id/part_no/seq/op_type_id/op_type_name/source/status/created_at`
  - 外部：`supplier_id/ext_days/ext_group_id`
  - 内部：`setup_hours/unit_hours`
- 外键：`part_no -> Parts`（级联删除），`op_type_id -> OpTypes`，`supplier_id -> Suppliers`
- 索引：`idx_part_operations_part/idx_part_operations_source`

### ExternalGroups（连续外部工序组）
- **PK**：`group_id`
- 字段：`group_id/part_no/start_seq/end_seq/merge_mode/total_days/supplier_id/remark/created_at`
- 外键：`part_no -> Parts`（级联删除），`supplier_id -> Suppliers`
- 索引：`idx_external_groups_part`

### Batches（后续排产阶段）
- **PK**：`batch_id`
- 字段：`batch_id/part_no/part_name/quantity/due_date/priority/ready_status/status/remark/created_at/updated_at`
- 外键：`part_no -> Parts`
- 索引：`idx_batches_status/idx_batches_priority/idx_batches_due_date`

### BatchOperations（后续排产阶段）
- **PK**：`id`（自增）
- **UNIQUE**：`op_code`，以及 `(batch_id, seq, piece_id)`
- 字段：`id/op_code/batch_id/piece_id/seq/op_type_id/op_type_name/source/machine_id/operator_id/supplier_id/setup_hours/unit_hours/ext_days/status/created_at`
- 外键：`batch_id -> Batches`（级联删除），`machine_id -> Machines`，`operator_id -> Operators`，`supplier_id -> Suppliers`
- 索引：`idx_batch_operations_batch/idx_batch_operations_status`

### Schedule（后续排产阶段）
- **PK**：`id`（自增）
- 字段：`id/op_id/machine_id/operator_id/start_time/end_time/lock_status/version/created_at`
- 外键：`op_id -> BatchOperations(id)`（级联删除）
- 索引：`idx_schedule_op/idx_schedule_machine/idx_schedule_operator/idx_schedule_time`

### WorkCalendar / ScheduleConfig（后续排产阶段）
- WorkCalendar：`date` 主键；字段 `day_type/shift_hours/efficiency/allow_normal/allow_urgent/remark`
- ScheduleConfig：自增 `id`；`config_key` 唯一；字段 `config_key/config_value/description/updated_at`

### OperationLogs / ScheduleHistory（留痕）
- OperationLogs：`id` 自增；字段 `log_time/log_level/module/action/target_type/target_id/operator/detail/error_code/error_message`
- ScheduleHistory：`id` 自增；字段 `schedule_time/version/strategy/batch_count/op_count/result_status/result_summary/created_by`

### Materials / BatchMaterials（预留）
- Materials：`material_id` 主键；字段 `name/spec/unit/stock_qty/status/remark/created_at`
- BatchMaterials：`id` 自增；字段 `batch_id/material_id/required_qty/available_qty/ready_status`
