# ADR-0001: Win7 单机单用户运行环境

- **状态**：已采纳
- **日期**：2026-01-10（项目启动）
- **决策者**：项目负责人

## 背景

本系统服务于工厂现场的排产调度，目标机器运行 Windows 7，无网络连接需求，仅一名排产员使用。

## 决策

- 目标环境锁定 Windows 7 SP1 单机。
- 单用户运行，不考虑多用户并发场景。
- 应用以本地 Web 形式运行（Flask + 浏览器），不部署为在线服务。

## 理由

- 工厂现场机器普遍为 Win7，无法升级。
- 单用户场景无需实现并发控制、会话管理、用户认证等，大幅降低复杂度。
- 本地 Web 形式兼顾了 GUI 开发效率和交互体验。

## 被拒绝的替代方案

- 多用户在线 Web 服务：需要服务器、并发控制、用户管理，复杂度远超需求。
- 原生 Windows 桌面应用（tkinter/PyQt）：开发效率低，UI 灵活性差。

## 后果

- Python 版本锁定 3.8（Win7 支持的最后一个主流版本）。
- 所有依赖必须兼容 Win7 + Python 3.8。
- 交付必须通过 PyInstaller 打包为免安装 exe。
- V1 不实现并发控制（见 ADR-0006）。
