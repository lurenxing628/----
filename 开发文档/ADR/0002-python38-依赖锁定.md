# ADR-0002: Python 3.8 及依赖版本锁定

- **状态**：已采纳
- **日期**：2026-01-10（项目启动）
- **决策者**：项目负责人

## 背景

Win7 兼容性要求限制了 Python 版本选择。Python 3.9+ 已不再官方支持 Win7。

## 决策

- 锁定 Python 3.8.x。
- 核心依赖固定版本：Flask 2.3.3、Werkzeug 2.3.7、openpyxl 3.0.10、python-dateutil 2.8.2。
- 打包工具锁定 PyInstaller 4.10（5.x/6.x 不兼容 Win7）。

## 理由

- Python 3.8 是最后一个官方支持 Win7 的版本。
- 固定依赖版本避免隐式升级导致的兼容性问题。
- PyInstaller 4.10 经验证可在 Win7 环境正常打包运行。

## 被拒绝的替代方案

- Python 3.9+：不支持 Win7。
- 不锁版本：存在隐式升级风险。

## 后果

- 不能使用 Python 3.9+ 的语法特性（如 `dict | dict`、`match-case`）。
- 不能使用 `typing.Union` 的 `X | Y` 简写形式。
- 第三方库升级前必须确认 Python 3.8 兼容性。
