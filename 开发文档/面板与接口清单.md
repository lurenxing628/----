# 面板与接口清单（Win7 无网单机版）

> 目的：给 UI 重写/重构使用——把**用户可见的所有面板（页面）**以及**每个页面对应/触发的接口**、**参数**、**用户可见文案（按钮/提示/确认/成功失败）**统一列清楚，避免重写时漏功能点或漏文案。

---

## 1. 运行环境与约束（必须遵守）

本项目交付与运行环境为 **Windows 7 单机版**，并且为 **无外网/离线环境**：

- **无外网**：不能依赖任何在线资源（CDN、在线字体、在线图表库、在线图标、外部 API、第三方埋点等）。
- **资源必须本地化**：所有 CSS/JS/字体/图片必须来自项目自身（`static/` 或内联），并随安装包/目录一起分发。
- **浏览器也应离线可用**：页面不要引用外部脚本/样式；若需图表/组件必须本地打包或纯 HTML/SVG 实现（项目中“排产优化分析”即为纯 HTML/SVG）。

## 2. 系统结构速览（便于理解接口形态）

### 2.1 技术形态

- **后端**：Flask
- **前端**：Jinja2 模板（服务器渲染 HTML），少量原生 JS
- **模板目录**：`templates/`
- **静态资源**：`static/`

### 2.2 接口形态（非常重要）

- 本系统绝大多数“接口”是：**表单 POST → 后端 `flash()` → redirect 回页面**（不是 JSON API）。
- 明确的 JSON API 目前有两个：**资源排班数据** `GET /scheduler/resource-dispatch/data`（供 `static/js/resource_dispatch.js` 拉取）与 **甘特图数据** `GET /scheduler/gantt/data`（供 `static/js/gantt.js` 拉取）。

### 2.3 Blueprint 与最终 URL 前缀（来自 `app.py`）

- `dashboard`：`/`
- `excel_demo`：`/excel-demo`
- `personnel`：`/personnel`
- `equipment`：`/equipment`
- `process`：`/process`
- `scheduler`：`/scheduler`
- `material`：`/material`
- `reports`：`/reports`
- `system`：`/system`

### 2.4 全站提示信息（flash）展示规则

在 `templates/base.html` 中：

- `flash(message, "error")`：红色提示
- 其他 category：绿色提示

### 4.2 系统使用说明入口与页面内短提示

- **入口宏**：`templates/components/ui_macros.html::floating_manual_button()`
- **页面速览 popover**：由 `floating_manual_button()` 内联渲染当前页面 `help_card`
- **事实源**：`web/viewmodels/page_manuals.py`（`ENDPOINT_TO_MANUAL_ID` + `MANUAL_ENTRY_ENDPOINTS` + `MANUAL_TOPICS`）；模板通过 `web/ui_mode.py` 注入的 `get_manual_url()` / `get_help_card()` 读取
- **事实约束**：页面级说明与 `help_card` 中的可填值、默认值、空值行为、依赖前置条件，必须以 `core/models/enums.py`、`core/services/common/normalization_matrix.py`、`core/services/common/enum_normalizers.py`、`core/services/common/excel_validators.py` 及对应 route/service 的真实接受值为准；`static/docs/scheduler_manual.md` 需要与其保持同口径，但不是独立事实源。
- **显示范围**：由 `ENDPOINT_TO_MANUAL_ID` 统一控制，覆盖 Excel、排产、配置、报表、物料、系统维护及高操作密度详情页；首页等简单页面不显示。
- **统一文案**：悬浮按钮统一显示 `本页说明`
- **跳转方式**：
  - 页面级入口：`GET /scheduler/config/manual?page=<endpoint>&src=...`
  - 完整原文章节入口：`GET /scheduler/config/manual?src=...#<full_manual_anchor>`
  - 无 `page` 参数时回退为整本说明模式
- **回退链路**：
  - 页面内链路：`help_card` 速览 → `查看本页详细说明` → 页面级说明页 → `查看完整原文对应章节`
  - 运行时回退：无 JS 时直接显示服务端 `fallback_text`；JS 配置解析失败时也回退读取服务端注入的 fallback 模板，不再落入空白正文
  - 整本模式：来自非 scheduler 页面时显示 `返回刚才页面`，并隐藏排产子导航
  - 页面级模式：无论来源页是否属于 scheduler，都统一隐藏排产子导航；无 `src` 时提供 `返回首页` / `返回排产首页` 兜底入口
- **页面内短提示策略**：帮助卡数据来自 registry 的 `help_card`；普通左键先展开速览 popover，面板内按钮文案为 `查看本页详细说明`；无 JS、右键/中键时仍按原链接直接进入说明页

#### 本页触发接口：执行排产 / 模拟排产

> 批次选择仅在 `status=pending` 时可用（checkbox）。

1) **执行排产（正式）**
- **URL**：`POST /scheduler/run`（`scheduler.run_schedule`）
- **按钮文案**：`执行排产`
- **form 字段**：
  - `batch_ids`：所选批次（多值）
  - `start_dt`：排产开始时间（默认明天 08:00；placeholder：`YYYY-MM-DD HH:MM`）
  - `end_date`：排产截止日期（可选；date；placeholder：`截止日期`）
  - `enforce_ready`：启用齐套约束（checkbox；未齐套禁止排产；默认勾选来自配置 `enforce_ready_default`）
  - **实现注记（当前）**：Web 路由按 checkbox 三态传值：字段未提交时传 `None`，由服务层回退 `enforce_ready_default`；字段提交且为真值时传 `True`；只有显式提交假值时才会传 `False`。
- **页面提示文案（节选）**：`批次新增/批量编辑请到 批次管理`
- **flash 文案**：
  - 成功主消息：`排产完成（版本 {ver}）：成功 {scheduled_ops}/{total_ops}，失败 {failed_ops}。超期 {N} 个/无超期。`
  - 超期样例：`超期批次（最多展示10个）：{sample}`
  - 状态守门失败：`以下批次状态不允许排产（completed/cancelled）：{sample}`
  - 额外告警：逐条 `flash(str(e), "warning")`
  - 失败：`{e.message}` 或 `排产失败：{e}`
- **返回**：redirect 回 `/scheduler/`

2) **模拟排产（插单模拟）**
- **URL**：`POST /scheduler/simulate`（`scheduler.simulate_schedule`）
- **按钮文案**：`模拟排产`
- **确认弹窗文案**：`将对所选批次执行一次【模拟排产】并生成新版本（不更新批次状态）。确定继续？`
- **form 字段**：同“执行排产”（`batch_ids/start_dt/end_date/enforce_ready`）
- **默认行为补充（当前实现）**：模拟排产路由同样按 checkbox 三态传值：字段未提交时传 `None`，由服务层回退 `enforce_ready_default`；字段提交且为真值时传 `True`；只有显式提交假值时才会传 `False`。
- **flash 文案**：
  - 未选：`请至少选择 1 个批次进行模拟排产。`
  - 成功：`模拟排产完成：生成版本 {ver}（不影响批次状态）。`
  - 状态守门失败：`以下批次状态不允许排产（completed/cancelled）：{sample}`
  - 失败：`{e.message}` 或 `模拟排产失败：{e}`
- **返回**：
  - 成功：redirect 到 `GET /scheduler/gantt?view=machine&week_start={today}&offset=0&version={ver}`（默认跳本周设备视图；`today` 为 `YYYY-MM-DD`）
  - 失败：redirect 回 `/scheduler/`

## 5.8 系统管理（System，前缀 `/system`）

> 说明：`GET /system/` 会 redirect 到备份页（`/system/backup`）。

### 接口：健康检查（启动器/验收脚本）

- **URL**：`GET /system/health`（`system.health`，JSON）
- **路由文件**：`web/routes/system_health.py`
- **用途**：供 `assets/启动_排产系统_Chrome.bat` 与 `validate_dist_exe.py` 判断当前监听实例是否为可复用/可验收的 APS 后端。
- **副作用**：无；该接口不会打开 DB，也不会触发系统维护任务。
- **返回 JSON**：
  - `app`：固定 `aps`
  - `status`：固定 `ok`
  - `contract_version`：当前契约版本（现为 `1`）
  - `ui_mode`：当前入口模式（`default` / `new_ui`）
  - `timestamp`：服务端时间戳
- **失败判定**：若返回非 200，或 `app/status` 与预期不匹配，启动器与验收脚本都应视为“实例不可复用/未就绪”，回退到启动新实例或等待实例完成启动。

### 接口：运行时停机（卸载器内部调用）

- **URL**：`POST /system/runtime/shutdown`（`system.runtime_shutdown`，JSON）
- **路由文件**：`web/routes/system_health.py`
- **用途**：供主程序/legacy 安装器与卸载器内部调用；安装器会先根据 `logs/aps_runtime.json`、`aps_runtime.lock` 和停机 helper 解析结果定位当前实例，再请求 APS 后端优雅退出。
- **访问约束**：
  - 仅允许本机请求（`127.0.0.1` / `::1`）
  - 请求头必须携带 `X-APS-Shutdown-Token`
- **副作用**：触发当前运行中的 Flask server 进入 shutdown；该接口不会打开 DB，也不会触发系统维护任务。
- **成功返回**：`202`，JSON `{"app":"aps","status":"shutting_down"}`
- **拒绝返回**：
  - `403`：来源地址不是本机，或 token 不匹配
  - `503`：当前进程未注册可关闭的 runtime server
- **用户可见性**：否；仅供卸载流程与运维辅助脚本使用，不在页面中暴露入口。

## 6. UI 重写时的“接口兼容”建议（不改后端的前提下）

- **保持所有 URL 不变**：尤其是各 `form action="{{ url_for(...) }}"` 的 endpoint 对应关系。
- **保持所有表单字段名不变**：例如 `batch_ids / start_dt / end_date / mode / raw_rows_json` 等，否则后端无法读取。
- **保留关键用户提示文案**：包括 `flash()`、confirm 弹窗、空数据提示（muted 文案）；这部分是验收与可用性的关键。
- **无外网约束必须在 UI 方案中体现**：不能引入 CDN（如 `cdn.jsdelivr.net`、`unpkg`、`fonts.googleapis.com` 等），不能依赖在线图标库。
