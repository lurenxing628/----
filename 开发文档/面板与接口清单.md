# 面板与接口清单（Win7 无网单机版）

> 目的：给 UI 重写/重构使用——把**用户可见的所有面板（页面）**以及**每个页面对应/触发的接口**、**参数**、**用户可见文案（按钮/提示/确认/成功失败）**统一列清楚，避免重写时漏功能点或漏文案。

---

## 1. 运行环境与约束（必须遵守）

本项目交付与运行环境为 **Windows 7 单机版**，并且为 **无外网/离线环境**：

- **无外网**：不能依赖任何在线资源（CDN、在线字体、在线图表库、在线图标、外部 API、第三方埋点等）。
- **资源必须本地化**：所有 CSS/JS/字体/图片必须来自项目自身（`static/` 或内联），并随安装包/目录一起分发。
- **浏览器也应离线可用**：页面不要引用外部脚本/样式；若需图表/组件必须本地打包或纯 HTML/SVG 实现（项目中“排产优化分析”即为纯 HTML/SVG）。

## 2. 系统结构速览（便于理解接口形态）

### 2.1 技术形态

- **后端**：Flask
- **前端**：Jinja2 模板（服务器渲染 HTML），少量原生 JS
- **模板目录**：`templates/`
- **静态资源**：`static/`

### 2.2 接口形态（非常重要）

- 本系统绝大多数“接口”是：**表单 POST → 后端 `flash()` → redirect 回页面**（不是 JSON API）。
- 明确的 JSON API 只有一个：**甘特图数据** `GET /scheduler/gantt/data`（供 `static/js/gantt.js` 拉取）。

### 2.3 Blueprint 与最终 URL 前缀（来自 `app.py`）

- `dashboard`：`/`
- `excel_demo`：`/excel-demo`
- `personnel`：`/personnel`
- `equipment`：`/equipment`
- `process`：`/process`
- `scheduler`：`/scheduler`
- `material`：`/material`
- `reports`：`/reports`
- `system`：`/system`

### 2.4 全站提示信息（flash）展示规则

在 `templates/base.html` 中：

- `flash(message, "error")`：红色提示
- 其他 category：绿色提示

#### 本页触发接口：执行排产 / 模拟排产

> 批次选择仅在 `status=pending` 时可用（checkbox）。

1) **执行排产（正式）**
- **URL**：`POST /scheduler/run`（`scheduler.run_schedule`）
- **按钮文案**：`执行排产`
- **form 字段**：
  - `batch_ids`：所选批次（多值）
  - `start_dt`：排产开始时间（默认明天 08:00；placeholder：`YYYY-MM-DD HH:MM`）
  - `end_date`：排产截止日期（可选；date；placeholder：`截止日期`）
  - `enforce_ready`：启用齐套约束（checkbox；未齐套禁止排产；默认值来自配置 `enforce_ready_default`）
- **页面提示文案（节选）**：`批次新增/批量编辑请到 批次管理`
- **flash 文案**：
  - 成功主消息：`排产完成（版本 {ver}）：成功 {scheduled_ops}/{total_ops}，失败 {failed_ops}。超期 {N} 个/无超期。`
  - 超期样例：`超期批次（最多展示10个）：{sample}`
  - 额外告警：逐条 `flash(str(e), "warning")`
  - 失败：`{e.message}` 或 `排产失败：{e}`
- **返回**：redirect 回 `/scheduler/`

2) **模拟排产（插单模拟）**
- **URL**：`POST /scheduler/simulate`（`scheduler.simulate_schedule`）
- **按钮文案**：`模拟排产`
- **确认弹窗文案**：`将对所选批次执行一次【模拟排产】并生成新版本（不更新批次状态）。确定继续？`
- **form 字段**：同“执行排产”（`batch_ids/start_dt/end_date/enforce_ready`）
- **flash 文案**：
  - 未选：`请至少选择 1 个批次进行模拟排产。`
  - 成功：`模拟排产完成：生成版本 {ver}（不影响批次状态）。`
  - 失败：`{e.message}` 或 `模拟排产失败：{e}`
- **返回**：
  - 成功：redirect 到 `GET /scheduler/gantt?view=machine&week_start={today}&offset=0&version={ver}`（默认跳本周设备视图；`today` 为 `YYYY-MM-DD`）
  - 失败：redirect 回 `/scheduler/`

- **按钮文案**：`保存配置`
- **form 字段（及页面标签，节选）**：

## 6. UI 重写时的“接口兼容”建议（不改后端的前提下）

- **保持所有 URL 不变**：尤其是各 `form action="{{ url_for(...) }}"` 的 endpoint 对应关系。
- **保持所有表单字段名不变**：例如 `batch_ids / start_dt / end_date / mode / raw_rows_json` 等，否则后端无法读取。
- **保留关键用户提示文案**：包括 `flash()`、confirm 弹窗、空数据提示（muted 文案）；这部分是验收与可用性的关键。
- **无外网约束必须在 UI 方案中体现**：不能引入 CDN（如 `cdn.jsdelivr.net`、`unpkg`、`fonts.googleapis.com` 等），不能依赖在线图标库。
