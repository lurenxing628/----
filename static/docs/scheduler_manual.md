## 1. Excel 导入导出：完整填写指南

系统的很多数据都支持通过 Excel 批量导入导出。这一章手把手教你每个模板怎么填、格式有什么要求、填错了会怎样。

### 1.1 通用操作流程

所有 Excel 导入页面的操作流程都是一样的，就三步：

**第一步：下载模板**
- 点击页面上的 **"下载模板"** 按钮，得到一个 `.xlsx` 文件
- 按模板格式填写数据（表头不要改）
- 下载的模板已为各列预置了格式：编号/ID 列设为文本（`@`）、整数列设为 `0`、小数列设为 `0.00`、日期列设为 `yyyy-mm-dd`、时间列设为 `hh:mm`，枚举列带下拉验证。自己新建 Excel 时建议参照 1.3.1 格式对照表设置
- 系统按模板的首个工作表读取数据（通常就是 `Sheet1`），不要把有效数据挪到第二个工作表

**第二步：上传预览**
- 选择导入模式（默认是"覆盖"，见下方说明）
- 选择文件后点 **"上传并预览"**
- 系统会检查每一行数据，显示预览表格。每行会标一个状态，用不同颜色区分：

| 状态 | 含义 |
|------|------|
| 新增 | 这是新数据，确认后会写入数据库 |
| 更新 | 已存在的数据会被更新 |
| 无变化 | 数据和数据库里一样，不需要改 |
| 跳过 | 追加模式下已存在的数据，不会导入 |
| 错误 | 这行有问题，需要修改后重新上传 |

**第三步：确认导入**
- 检查预览无误后，回到页面顶部点 **"确认导入（写入数据库）"**
- **只要有一行是"错误"，整个文件都不会导入**——必须把所有错误行修好再重新上传
- 导入完成后，建议立即点一次 **"导出 Excel"** 做复核，尤其是覆盖/替换模式

> **小贴士**：单次上传的 Excel 文件大小不能超过 **16MB**。超过限制会提示"上传文件超过 16MB，请缩小文件后重试。"。

> **小贴士**：如果上传失败提示"读取失败/被占用"，请先关闭 Excel 程序后重试。

> **小贴士**：所有采用"上传预览 → 确认导入"流程的 Excel 导入页面都有"预览保护"——如果你预览之后、确认之前有人改了数据库里的数据，系统会拒绝导入，并提示"导入被拒绝：数据已变化，需重新预览后再确认导入。"。

> **小贴士**：部分导入页面（如批次信息导入）会有一个 **"严格模式"** 复选框。开启后，导入过程中遇到兼容性问题（比如需要从工艺路线自动补建模板、未知工种回退等）会直接报错而不是自动兼容。严格模式的开关状态会在预览和确认之间保持一致——如果需要切换，要重新上传并预览。

### 1.2 三种导入模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **覆盖** | 已存在的数据会被更新，不存在的会新增 | 最常用，更新已有数据 |
| **追加** | 已存在的跳过，只导入新数据 | 只补充新数据，不动旧的 |
| **替换** | 先清空数据库中该类数据，再全量导入 | 全量重建，慎用 |

> **注意**："替换"模式的危险程度因模块而异，下面这张表帮你一目了然：

| 模块 | 替换是否可用 | 有没有保护 | 说明 |
|------|------------|----------|------|
| 批次信息 | 可用 | **没有** | 直接删除全部批次和排程记录，不可恢复 |
| 人员 | 可用 | 有 | 被排产数据引用时拒绝 |
| 设备 | 可用 | 有 | 被排产数据引用时拒绝 |
| 供应商 | 可用 | 有 | 被零件工序/批次工序/外部组引用时拒绝 |
| 零件工艺路线 | 可用 | 有 | 存在任意批次时拒绝 |
| 工种 | 可用 | 有 | 被引用时拒绝 |
| 工作日历 | 可用 | 没有 | 直接清空 |
| 人员设备关联 | 可用 | 没有 | 直接清空 |
| 人员专属工作日历 | 可用 | 没有 | 直接清空 |
| 零件工序工时 | **不可用** | — | 页面不提供这个选项 |

### 1.3 日期与时间格式

- **日期**：`2026-04-15`（横杠）和 `2026/04/15`（斜杠）都行。Excel 里的日期单元格也能识别
- **时间**：`08:00` 格式，中文冒号（`：`）也能识别。`HH:MM:SS` 格式也接受（秒会被忽略）。单位数小时如 `8:30` 也可识别（等效于 `08:30`）
- 如果单元格本身是 Excel 日期/时间类型，系统会自动取日期部分；如果是手工输入的文本，请只写日期（如 `2026-04-15` 或 `2026/04/15`），不要带时间
- 不要在文本日期里带 `T` 或空格
- 不要在日期列填纯数字（如 44927），系统不能识别 Excel 序列号

### 1.3.1 Excel 单元格格式注意事项

Excel 单元格的格式（文本/数字/日期/常规）会直接影响系统读到的数据。格式不对，即使你看到的内容是对的，系统也可能读错。

**格式对照表**：

| Excel 格式 | 系统读到的类型 | 适用列 | 注意事项 |
|-----------|-------------|--------|----------|
| **文本** | 字符串（原样保留） | 所有编号/ID/名称类 | 防止前导零丢失、防止被自动转日期 |
| **数字（整数）** | 整数 | 数量、工序号 | 不要有小数点 |
| **数字（小数）** | 小数 | 工时、效率、周期 | 允许小数 |
| **日期** | 日期对象 | 日期、交期、齐套日期 | 也可以用文本格式填 `YYYY-MM-DD` |
| **常规** | 由 Excel 猜测 | **不推荐** | 可能导致 `1-2` 变日期、`001` 变 `1` |
| **布尔值** | 是/否值 | 是/否类字段可用，其它列仍不推荐 | 最稳妥的做法仍然是直接填 `是`/`否` |

**四个常见陷阱**：

**陷阱 1：前导零丢失**

工号 `001` 在常规或数字格式下，Excel 会自动去掉前导零变成 `1`。系统读到的也是 `1`，不是 `001`。

| 在 Excel 中的操作 | 系统读到的值 | 结果 |
|------------------|-------------|------|
| 常规格式输入 `001` | `1` | 前导零丢失，工号变成 "1" |
| 文本格式输入 `001` | `001` | 正确保留 |
| 常规格式输入 `OP001` | `OP001` | 正确（含字母的不会被转数字） |

**陷阱 2：Excel 自动日期**

在常规格式下输入 `1-2`，Excel 会自动变成"1月2日"。上传后系统读到的是一个日期对象，和你想填的文字完全不同。**解决方法**：先把列设成文本格式再填。

**陷阱 3：公式和合并单元格**

- 如果单元格里写了公式（如 `=A2`），但文件是用其他工具生成的、**从未在 Excel 中打开保存过**，系统只能读到空值，会报"XXX不能为空"
- 合并单元格只有左上角有值，其余位置都是空的。**请取消合并，每行独立填写。**

**陷阱 4：布尔值只适用于是/否类字段**

如果是 `允许普通件`、`允许急件`、`主操设备` 这类是/否字段，最稳妥的做法是直接填 `是`/`否`。对于状态、类型、优先级等枚举字段，不要用 Excel 布尔值。另外，`可用工时`、`效率`、`默认周期` 等**数值型字段**也不接受布尔值，填了会直接报错"必须是数字"，请填写正常数字。

**怎么设文本格式**（3 步）：

1. 选中需要填编号/ID 的那一列
2. 右键 → 设置单元格格式 → 选择 **"文本"**
3. **设好格式后再填数据**（如果是已经填好的数据再改格式，前导零不会自动恢复）

> **通用提示**：下载的模板和导出的数据中，状态/类型/优先级等枚举字段优先填写中文值。系统仍兼容历史英文标准值；说明书里出现的英文值都表示“可兼容输入”，不是日常推荐写法。

### 1.4 模板填写顺序——先填哪个后填哪个

如果你是第一次使用系统，建议按下面的顺序来导入数据。从头到尾走一遍，就不会碰到"XXX不存在"之类的报错：

- **第一步：工种配置** — 因为供应商要选工种
- **第二步：供应商配置** — 因为零件工艺路线要用到工种（系统还会自动匹配供应商）
- **第三步：零件工艺路线** — 因为零件工序工时要对应已有的零件和工序
- **第四步：零件工序工时**
- 同时可以做：**人员基本信息**、**设备信息**（互不依赖）
- 等人员和设备都录完了 → **第五步：人员设备关联**（要用到人员和设备）
- 随时可以做：**工作日历**、**人员专属工作日历**
- 最后 → **第六步：批次信息**（图号必须先在工艺管理里存在）

### 1.5 逐模板填写指南

下面按推荐填写顺序，逐个模板说明每一列怎么填。

> **关于"Excel格式"列**：每个表格新增了"Excel格式"列，告诉你这一列在 Excel 中应该设什么格式。如果你不确定，先看 1.3.1 节的格式对照表。下载的模板和导出的数据中，枚举字段建议统一使用中文值。

#### 1.5.1 工种配置

**进入方式**：工艺管理 → 工种配置（Excel导入/导出）

| 列名 | Excel格式 | 必填 | 怎么填 | 填错了会怎样 |
|------|----------|------|--------|------------|
| 工种ID | 文本 | 是 | 自定义编号，如 `OT001`，不能重复 | 重复了预览会报错 |
| 工种名称 | 文本 | 是 | 如 `数控铣`、`钳工`、`表处理` | 空了会报错 |
| 归属 | 文本(枚举) | 否 | 填 `内部` 或 `外部`（`内`/`外` 也行），也兼容英文标准值 `internal`/`external`；不填默认内部 | 填了别的词会报错 |

**填写示例**：

| 工种ID | 工种名称 | 归属 | 说明 |
|--------|---------|------|------|
| OT001 | 数控铣 | 内部 | 常规填法 |
| OT002 | 表处理 | 外部 | 外协工序 |
| OT003 | 热处理 | | 留空默认内部 |

#### 1.5.2 供应商配置

**进入方式**：工艺管理 → 供应商配置（Excel导入/导出）

| 列名 | Excel格式 | 必填 | 怎么填 | 填错了会怎样 |
|------|----------|------|--------|------------|
| 供应商ID | 文本 | 是 | 自定义编号，不能重复 | 重复了报错 |
| 名称 | 文本 | 是 | 供应商名称 | 空了报错 |
| 对应工种 | 文本 | 否 | 填工种ID或工种名称都行（如 `OT002` 或 `表处理`） | 填了不存在的工种会报错 |
| 默认周期 | 小数 | 是 | 正数（天），必须填写；填 `0`、负数、文字、布尔值、非数字或无穷大都会报错 | 填了非法值会报错 |
| 状态 | 文本(枚举) | 否 | `启用`/`停用`，也支持 `在用`/`正常`/`禁用`，并兼容英文标准值 `active`/`inactive`；不填默认启用 | 填了不认识的词会报错 |
| 备注 | 文本 | 否 | 自由填写 | — |

> **小贴士**：下载的模板包含“状态”和“备注”两列。旧模板若只有前 4 列，也可以在 Excel 末尾补上这两列继续导入；状态空白默认启用，备注随便写。

> **补充**：系统导出的供应商配置文件也带有"状态"和"备注"两列，适合导出后修改再导回。

**填写示例**：

| 供应商ID | 名称 | 对应工种 | 默认周期 | 状态 | 备注 | 说明 |
|----------|------|---------|---------|------|------|------|
| S001 | 外协-标印厂 | 表处理 | 3 | 启用 | 常用供应商 | 用工种名称 |
| S002 | 热处理公司 | OT003 | 5 | 启用 | 热处理默认 | 用工种ID也行 |
| S003 | 电镀公司 | | 2 | 停用 | 暂不参与自动匹配 | 停用供应商不会被工艺路线自动匹配 |

> **注意**：工艺路线自动解析外协工序时，只会匹配状态为"启用"的供应商。停用供应商会保留在基础资料里，但不会自动带入外协工序。

> **注意**：替换模式下如果有供应商被零件工序模板、批次工序或外部工序组引用了，替换会被拒绝。改用"覆盖"就行。

#### 1.5.3 零件工艺路线（重点：工艺路线字符串怎么写）

**进入方式**：工艺管理 → 零件工艺路线（Excel导入/导出）

| 列名 | Excel格式 | 必填 | 怎么填 | 填错了会怎样 |
|------|----------|------|--------|------------|
| 图号 | 文本 | 是 | 零件图号，不能重复 | 重复了报错 |
| 名称 | 文本 | 是 | 零件名称 | 空了报错 |
| 工艺路线字符串 | 文本 | 是 | 见下方详细说明 | 格式不对会报错 |

**工艺路线字符串怎么写？**

把每道工序的"工序号"和"工种名"连着写就行了。系统会自动识别哪里是数字（工序号）、哪里是文字（工种名）。

举个例子：这个零件有 6 道工序——

| 工序号 | 工种 |
|--------|------|
| 5 | 数铣 |
| 10 | 钳 |
| 20 | 数车 |
| 35 | 标印 |
| 40 | 总检 |
| 45 | 表处理 |

写成路线字符串就是：`5数铣10钳20数车35标印40总检45表处理`

**写法要点**：
- 必须以数字开头（工序号）
- 最后一项不能只有数字没有工种名
- 工序之间可以用空格、逗号隔开，也可以不隔（系统都认）
- 写成 `5数铣,10钳,20数车` 或 `5数铣 10钳 20数车` 都行
- 顿号 `、`、破折号 `—`、箭头 `→` `>` 等分隔符也会被自动识别并移除
- 全角数字（如 `５`）会自动转为半角

**系统会自动做的事**（不需要你操心）：
- 根据工种名自动判断每道工序是内部还是外部
- 如果是外部工序，自动匹配对应的供应商和默认周期
- 连续的外部工序会自动归为一个"外部组"
- 如果工种名没有在"工种配置"里找到，系统会默认将该工序标记为外部工序并产生告警（不会直接报错拒绝导入）。**建议在导入工艺路线前先维护好工种配置。**

**填写示例**：

| 图号 | 名称 | 工艺路线字符串 |
|------|------|---------------|
| PART-001 | 法兰盘 | 5数铣10钳20数车35标印40总检45表处理 |
| PART-002 | 轴套 | 5数车10磨20总检 |

> **注意**：替换模式下如果系统里已经有批次数据，导入会被拒绝（提示"已存在批次数据，不能执行替换"）。改用"覆盖"或"追加"就行。

#### 1.5.4 零件工序工时

**进入方式**：工艺管理 → 零件工序工时（Excel导入/导出）

| 列名 | Excel格式 | 必填 | 怎么填 | 填错了会怎样 |
|------|----------|------|--------|------------|
| 图号 | 文本 | 是 | 必须是工艺管理里已有的图号 | 不存在会报错 |
| 工序 | 整数 | 是 | 工序号（正整数，如 5、10、20）；不接受布尔值和科学计数法 | 该图号下没有这个工序号会报错；填了布尔值或科学计数法也会报错 |
| 换型时间(h) | 小数 | 否 | 数字（小时），如 `0.5`，不填默认 0 | 填文字或非数字会报错 |
| 单件工时(h) | 小数 | 否 | 数字（小时），如 `0.1`，不填默认 0 | 填文字或非数字会报错 |

**填写示例**：

| 图号 | 工序 | 换型时间(h) | 单件工时(h) | 说明 |
|------|------|-----------|-----------|------|
| PART-001 | 5 | 0.5 | 0.1 | 正常填写 |
| PART-001 | 10 | 0.3 | 0.05 | 同一图号不同工序 |
| PART-001 | 20 | | | 都留空则默认 0 |

> **注意**：这个模板**只提供"覆盖"和"追加"两种模式**，页面上没有"替换"选项。

> **小贴士**：追加模式有个特殊行为——只会补齐那些换型时间和单件工时都是 0 的行。如果某道工序已经填过工时了，追加模式不会动它。**另外，这个模板只适用于内部工序；外部工序不导工时。**

#### 1.5.5 人员基本信息

**进入方式**：人员管理 → 人员基本信息（Excel导入/导出）

| 列名 | Excel格式 | 必填 | 怎么填 | 填错了会怎样 |
|------|----------|------|--------|------------|
| 工号 | 文本 | 是 | 员工编号，不能重复 | 重复了报错 |
| 姓名 | 文本 | 是 | 姓名 | 空了报错 |
| 状态 | 文本(枚举) | 是 | `在岗`/`启用`/`可用`/`正常`，或 `停用`/`休假`/`停用/休假`/`离岗`；也兼容英文标准值 `active`/`inactive` | 空了会报错 |
| 班组 | 文本 | 否 | 班组编号或班组名称，必须是系统里已有的班组。留空=不设班组 | 班组不存在会报错 |

### 1.6 导出功能一览

上面这些导入页面都有一个"导出 Excel"按钮，可以把系统里现有的数据导出来。适合导出一份备份、在 Excel 里批量修改后重新导入、或给同事使用。

现在导出的 Excel 会统一做 3 件事：
- 冻结首行，表头滚动时不丢
- 自动给常见列设置基础列宽，打开后更容易看
- 如果某个文本值刚好以 `=`、`+`、`-`、`@` 开头，系统会自动做安全转义，避免被 Excel 当成公式执行

> **显示口径**：当前表和预览表都会优先显示中文标签，例如状态显示为"启用/停用/维修"，类型显示为"工作日/假期"，优先级显示为"普通/急件/特急"，是/否显示为"是/否"。`active`、`workday`、`yes`、`no` 这类英文标准值只作为历史兼容输入，不作为日常页面展示口径。

另外还有几个只能导出不能导入的：
- 零件工序模板导出（工艺管理 → 导出）
- 周计划导出（排产调度 → 周计划）
- 资源排班导出（排产调度 → 资源排班中心）
- 报表导出：超期清单、资源利用率、停机影响（报表中心）

### 1.7 常见导入错误

| 你碰到了什么 | 系统提示大概长这样 | 怎么解决 |
|-------------|------------------|---------|
| Excel 里有重复编号 | "Excel 中存在重复的'批次号'：B001, B001…" | 去掉重复行 |
| 必填列没填 | "第X行：批次号不能为空" | 补上 |
| 日期格式不对 | "第X行：日期格式不正确" | 改成 2026-04-15 或 2026/04/15 |
| 图号不存在 | "第X行：图号 PART-999 不存在" | 先去工艺管理创建这个零件 |
| 工号/设备编号不存在 | "第X行：工号 OP999 不存在" | 先创建人员或设备 |
| 预览后数据被改了 | "导入被拒绝：数据已变化，需重新预览后再确认导入。" | 重新上传预览一次 |
| 有错误行没修 | "导入被拒绝：Excel 存在 N 行错误" | 修好所有错误行后重新上传 |
| 人员/设备替换被拒 | "已有排程结果引用了设备，不能执行替换" | 改用"覆盖"模式 |
| 供应商替换被拒 | "已有零件工序模板引用了供应商…" | 改用"覆盖"模式 |
| 工艺路线替换被拒 | "已存在批次数据，不能执行替换" | 改用"覆盖"或"追加" |
| 零件工序工时选了替换 | "该页面不支持替换，请使用覆盖或追加" | 换个模式 |
| 数值字段报非数字 | "第X行：数值不合法" | 把那个格子改成正常数字 |
| 工号/设备编号前导零丢失 | 预览中看到编号变成纯数字（如 001 变成 1） | 在 Excel 中先将该列设为"文本"格式，再重新填写数据（见 1.3.1） |
| 公式单元格读出空值 | "第X行：XXX不能为空" | 不要用公式引用，直接填值；或在 Excel 中打开保存一次再上传 |
| 合并单元格导致空行 | "第X行：XXX不能为空" | 取消合并单元格，每行独立填写 |
| 日期列填了纯数字 | "日期格式不合法" | 改用 `YYYY-MM-DD` 格式，或将单元格设为日期格式 |
| 数量列填了带小数的文本 | "数量必须是整数" | 改为整数（不带小数点），如 `100` 而非 `100.0` |
| 枚举列填了布尔值 | "状态/类型不合法" | `是/否` 类字段可以填 `是`/`否`，但状态、类型、优先级这类枚举列必须直接填文字（如 `工作日`/`假期`） |
| 数值列（可用工时/效率/默认周期等）填了布尔值 | "必须是数字"或"数值不合法" | 这些字段是纯数值字段，不是是/否开关，请填写正常数字（如 `8`、`1.0`、`5`） |
| 工序号列填了科学计数法 | "工序号解析失败"或"工序不存在" | 工序号必须填写普通整数（如 5、10、20），不能用科学计数法 |

> **关于错误行号**：上表中"第X行"的行号就是你 Excel 文件中的原始行号（含表头行），可以直接在 Excel 里按行号定位对应的数据。

## 2. 快速上手：5分钟完成第一次排产

> **小贴士**：如果你是第一次使用系统，建议先去"Excel 演示"页面（浏览器直接访问 `/excel-demo/`）玩一遍导入流程，熟悉了再回来跟着下面的步骤走。

**目标**：让系统跑起来，看到排产结果（甘特图）。

### 第一步：导入一个批次

1. 打开 **排产调度 → 批次信息（Excel导入/导出）**
2. 下载模板 `批次信息.xlsx`
3. 按第 1 章的填写指南填好数据
4. 上传文件，勾选 **"导入后自动生成工序"**
5. 确认导入

> **小贴士**：第一次用选"追加"模式就行。详细的填写规则和示例请看第 1 章。

### 第二步：补充工序信息

1. 回到 **排产调度主页**，在批次列表中点击刚才导入的批次号（或点击"查看/编辑"按钮）进入详情页
2. 每道工序需要填写：
   - **设备**：这道工序在哪台机器上做
   - **人员**：谁来操作（下拉框会根据你选的设备自动过滤出可操作的人员）
   - **工时**：换型时间 + 单件时间（单位：小时）
3. 如果有外协工序，填的内容不一样：
   - **供应商**：交给哪家外部供应商做
   - **外协周期**：需要几天完成（按自然日算）
4. 每行填完后点击"保存"

> **小贴士**：如果下拉框里没有可选的人员，说明还没有维护"人员-设备关联"。去人员管理页面，给对应人员添加可操作设备就行。

### 第三步：执行排产

1. 回到 **排产调度主页**
2. 在"待排"列表里勾选你的批次
3. 设置 **排产开始时间**（默认是明天早上8点，可以改）
4. 如果需要，设置 **排产截止日期**（不设就不限）
5. 点击 **"执行排产"**

### 第四步：查看结果

- **甘特图**：直观看到每道工序的时间安排，有设备视角和人员视角两种
- **周计划**：导出 Excel 给车间使用
- **资源排班**：查看每个人或每台设备的任务安排

恭喜！你已经完成了第一次排产。

## 3. 基本概念：像排队买饭一样理解排产

把排产想象成食堂打饭的过程：

### 批次 = 你的餐盘

每个批次就像你端着的餐盘，里面要装几个菜（工序）。餐盘上写着：
- **批次号**：你的取餐号
- **数量**：要打几份菜
- **交期**：什么时候必须吃完（交货日期）
- **优先级**：普通顾客 / VIP / 超级VIP

### 工序 = 要打的每道菜

每道菜就是一道工序，必须按顺序打：
- 先打米饭（第1道工序）
- 再打菜（第2道工序）
- 最后打汤（第3道工序）

工序分两种：
- **内部工序**：食堂自己做的菜（需要占用厨师和灶台）
- **外部工序（外协）**：外面餐厅送来的菜（只需要等配送时间，要指定供应商和送货天数）

### 资源 = 厨师和灶台

- **设备**：灶台（同一时间只能做一道菜）
- **人员**：厨师（同一时间只能操作一个灶台）
- **人员-设备关联**：哪个厨师会用哪个灶台（不是每个厨师都会用每个灶台）

### 齐套 = 食材到齐了吗

做菜之前要先看食材到了没有：
- **齐套**：食材全到了，随时可以做
- **未齐套**：食材还没到，先别排
- **部分齐套**：到了一部分

如果你开启了"齐套约束"，没齐套的餐盘就不能上灶台。

> 齐套的完整设置流程见 5.4 物料管理；齐套约束开关见 7.9 高级设置；批次导入时齐套字段的填写见 1.5.10 批次信息。

### 换型 = 清洗灶台换下一道菜

从做红烧肉切换到做清蒸鱼，灶台需要清洗调整，这段时间就是**换型时间**。如果连续做的是同一道菜，就不用换型了。所以系统会尽量把同类工序排在一起，减少换型。

### 工作日历 = 食堂营业时间

- 哪些天营业（工作日）
- 每天营业几小时（可用工时）
- 效率多少（假期加班可能效率低一点）
- 周末加班怎么算

### 排产策略 = 谁先打饭

- **优先级优先**：VIP先打
- **交期优先**：赶时间的先打
- **综合**：既看身份又看时间，综合排队
- **先进先出**：按排队顺序来

### 排产版本 = 今天的第几次出餐方案

每次执行排产或模拟排产，系统都会生成一个新版本（v1、v2、v3……）。就像食堂主管可以先出一版排菜方案，不满意再出一版，所有方案都留着，随时对比、随时切换。

## 4. 系统总览：模块导航地图

### 首页（仪表盘）

打开系统后看到的第一个页面。

**统计卡片**：页面顶部有 4 张卡片——待排批次、已排批次、超期批次、最近排产版本，让你一眼掌握当前状态。

**最近排产**：显示最近一次排产的时间、策略、结果状态，以及关键指标（拖期小时、总工期、设备利用率）。如果还没排过产，这里会提示"还没有排产记录，排产后这里会显示结果"。

**快捷操作**：一排按钮帮你快速跳转——去排产、批次管理、导入批次、资源排班、甘特图、周计划、报表。不用在导航条里翻来翻去。

> **小贴士**：如果卡片上的数字都是 0 或 "--"，说明还没有导入批次或执行排产。按第 2 章的快速上手走一遍就会有数据了。

### Excel 演示页

**入口**：经典界面的顶部导航没有这个入口，可直接访问 `/excel-demo/`；现代界面可从顶部导航进入。

**用途**：给第一次接触系统的用户练手用。页面以"人员基本信息"模板为例，完整演示"下载模板 → 上传预览 → 确认导入"的流程。

**和正式导入的区别**：操作的数据是真实的（会写入人员表），但页面下方会额外展示当前库中已有的人员数据，方便你对比观察"新增/更新/无变化/跳过/错误"的差异。

> **小贴士**：如果你对 Excel 导入不熟悉，建议先去演示页玩一遍，心里有底了再去正式页面操作。

### 模块导航地图

系统包含以下主要模块，可通过顶部导航栏进入：

- **排产** — 排产调度的核心入口
  - 执行排产（主页）：选批次、设参数、执行/模拟排产
  - 批次管理：新增、复制、删除、批量修改批次
  - 高级设置/模板：调整排产参数、管理常用方案
  - 甘特图：设备视图 / 人员视图，查看排程结果
  - 资源排班中心：从人员/设备/班组角度查看排班
  - 排产优化分析：对比各版本指标和优化曲线
  - 周计划：预览和导出周计划表
  - 工作日历配置：设置工作日/假期
  - 批次信息（Excel导入/导出）
  - 工作日历（Excel导入/导出）
- **工艺** — 零件工艺模板管理
  - 零件列表：新增/编辑/删除零件
  - 工序模板：查看和编辑每个零件的工序
  - 工种配置：管理内部/外部工种
  - 供应商配置：管理外协供应商
  - Excel导入/导出：工艺路线、工种、供应商、工序工时
- **人员** — 操作人员管理
  - 人员列表：新增/编辑/状态切换/删除
  - 班组管理：创建和维护班组
  - 人员-设备关联（含Excel）
  - 个人工作日历
- **设备** — 设备资源管理
  - 设备列表：新增/编辑/状态切换/删除
  - 设备-人员关联（含Excel）
  - 停机计划（含批量停机）
- **物料** — 物料与齐套管理
  - 物料主数据：新增/编辑/删除
  - 批次物料需求：管理齐套状态
- **报表** — 统计分析
  - 超期清单、资源利用率、停机影响统计
- **系统** — 系统维护
  - 备份/恢复、操作日志、排产历史、插件管理

### 界面模式

系统有两种界面风格：**经典**和**现代**。可以在页面右上角的下拉框中切换。功能完全一样，只是外观不同。

### 页面上的提示消息

操作成功或失败后，页面顶部会出现提示条：
- **绿色**：操作成功（比如"排产完成"、"已创建批次"）
- **红色**：操作失败或有错误（比如"排产失败"、"导入被拒绝"）

看到红色提示时，仔细阅读提示内容，通常会告诉你问题出在哪里。

### 贴心小设计

- **绝大多数删除操作都有二次确认**：点"删除"后通常会弹窗问你"确定要删吗？"，不会一点就没了（少数操作如取消停机计划等除外）
- **按钮防误触**：提交表单后按钮会自动变灰，防止你手快点了两次

### 系统使用说明入口

在 Excel 导入、排产、配置、报表、物料、系统维护等复杂操作页面，都有一个 **"本页说明"** 入口。现代界面里它在左侧导航栏，经典界面里它在页面内容后面，不会盖住保存、删除、执行排产这类业务按钮。普通点击会先弹出这一页的速览提示；想看完整说明时，再点面板里的 **"查看本页详细说明"**。如果脚本没有加载成功，按钮仍会直接打开当前页面对应的详细说明。

## 5. 数据准备：排产前必须做的事

在第一次排产之前，需要先把基础数据准备好。以下按推荐顺序说明。

> **小贴士**：如果你更喜欢用 Excel 批量导入这些数据，请看第 1 章的逐模板填写指南。

### 5.1 工艺管理

**进入方式**：顶部导航 → 工艺

#### 零件列表

这是工艺管理的主页面，管理所有零件的工艺模板。

**主要操作**：
- **添加零件**：填写图号和名称，系统会自动解析工艺路线生成工序模板
- **查看/编辑**：点击零件图号或"查看/编辑"按钮进入详情
- **删除**：单条删除或勾选多个后批量删除

> **小贴士**：添加零件时旁边有一个 **"严格模式"** 复选框。开启后，工艺路线字符串里的未知工种、缺供应商映射、默认天数无效等情况会直接报错拒绝创建，而不是按兼容逻辑自动回退。日常不需要勾选，但如果你需要确保工艺数据的准确性，建议开启。如果创建过程中产生了告警（比如工种自动标为外部），页面会直接展示告警信息。

> **注意**：如果零件已经被批次引用了（即已经有批次使用了这个图号），删除会失败。系统会提示你"已被批次引用"。

#### 零件详情（工序模板）

进入零件详情后，可以看到该零件的所有工序模板：

- **重新解析**：点击"重新解析（覆盖工序模板）"按钮，系统会根据工艺路线字符串重新生成工序。会弹窗确认，因为这会覆盖已有的工序设置
- **保存工时**：内部工序可以设置换型时间和单件工时（单位：小时）
- **外部工序组**：连续的外部工序会被自动归为一组。可以选择"分别设置"（每道外协工序单独设天数）或"合并设置"（整组共用一个总周期）

> **小贴士**：重新解析和外部工序组设置旁边也有 **"严格模式"** 复选框。重新解析的严格模式含义同添加零件；外部工序组"分别设置"时的严格模式含义是：每道外协工序的天数如果为空、非数字或 ≤ 0，会直接报错而不是回退到默认 1.0 天。

#### 工种配置

**进入方式**：工艺管理页面内的子导航 → 工种配置

工种用来区分"内部工序"和"外部工序"。

- **创建工种**：填写工种ID、名称、归属（内部/外部）
- **编辑/删除**：点击工种进入详情页修改或删除

#### 供应商配置

**进入方式**：工艺管理页面内的子导航 → 供应商配置

管理外协工序的供应商。

- **创建供应商**：填写供应商ID、名称、对应工种、默认周期（天）
- **编辑/删除**：点击供应商进入详情页修改或删除
- 外协工序创建时，系统会根据工种自动匹配启用供应商和默认周期；停用供应商只保留历史资料，不参与自动匹配

> **注意**：默认周期必须填写大于 0 的数字，**不能留空**。如果留空，系统会拒绝保存并提示"默认天数不能为空"。这和以前不同——以前留空会被静默当成 0 处理，现在必须明确填写。

### 5.2 人员管理

**进入方式**：顶部导航 → 人员

#### 人员列表

- **添加人员**：填写工号和姓名
- **批量操作**：勾选多个人员后，可以批量设为"在岗"或"停用/休假"，也可以批量删除
- **班组筛选**：如果有多个班组，可以用下拉框筛选某个班组的人员
- **单条删除**：点击"删除"按钮，会弹窗确认

> **注意**：如果人员已经被排产数据引用（比如某道工序指定了这个人），删除会失败。建议改为"停用"。

#### 人员详情

点击人员工号或"查看/编辑"进入详情页：

- **保存信息**：修改姓名、状态、班组、备注后点"保存"
- **切换状态**：在岗 / 停用
- **可操作设备**：添加或解除这个人可以操作的设备关联。可选字段包括"技能等级"（初级/普通/熟练）和"主操设备"（同一人仅允许1台主操设备）

> **小贴士**：技能等级不填的话默认是"普通"。如果你给某人设了一台新的主操设备，系统会自动把他之前的主操取消掉——不用你自己去清。

> **小贴士**：如果可操作设备列表中某些行旁边出现灰色的"兼容归一"提示（比如"技能等级：旧值 xxx 已归一为 普通"），说明这些关联数据来自历史导入时的非标准值，系统已自动转换为标准值。数据本身可以正常使用，但建议在方便时核对并重新保存一次。

#### 班组管理

**进入方式**：人员管理页面内的子导航 → 班组管理

- **添加班组**：填写班组编号和名称
- **编辑/删除**：修改班组信息或删除（如果仍有人员/设备归属该班组，删除会失败）

#### 个人工作日历

**进入方式**：人员详情页 → 个人工作日历

可以为每个人单独设置工作安排（请假、加班、轮班等）。个人日历会**覆盖**全局工作日历。比如全局设为工作日，但某人在个人日历里设为假期，排产时这个人当天就不会被安排任务。

也可以通过 Excel 批量导入个人工作日历（人员管理子导航 → 人员专属工作日历（Excel））。

### 5.3 设备管理

**进入方式**：顶部导航 → 设备

#### 设备列表

- **添加设备**：填写设备编号和名称
- **批量操作**：批量设为"可用"、"维修"或"停用"，也可以批量删除
- **班组筛选**：下拉框筛选某个班组的设备
- **单条删除**：弹窗确认后删除

> **注意**：如果设备已经被排产数据引用，删除会失败。建议改为"停用"或"维修"。

> **小贴士**：停用、维修的设备在批次工序的下拉框里会灰显（不能选），防止你误选了一台正在修的设备。同样，停用的人员在下拉框里也会灰显。

#### 设备详情

点击设备编号或"查看/编辑"进入详情页：

- **保存信息**：修改名称、工种、类别、班组、状态、备注
- **切换状态**：可用 / 维修 / 停用
- **可操作人员**：添加或解除可以操作这台设备的人员关联
- **停机计划**：为设备安排停机时间段（比如设备保养、维修），排产时系统会自动避开停机时段

#### 批量停机计划

**进入方式**：设备管理页面内的子导航 → 批量停机计划

可以一次性为多台设备创建停机计划：

- **按单台**：选择一台设备
- **按类别**：选择一个设备类别，该类别下所有设备都会创建停机
- **按全部**：所有设备都创建停机

> **小贴士**：如果某台设备在同一时间段已经有停机计划了，系统会自动跳过（不会重复创建）。创建后可以到设备详情页查看和管理停机记录。

### 5.4 物料管理

**进入方式**：顶部导航 → 物料

#### 物料主数据

管理物料基本信息：

- **创建物料**：填写物料编号、名称、规格、单位、库存数量、状态（可用/停用）、备注；库存数量可以为 0，但不能为负数
- **编辑/删除**：修改或删除物料
- **状态**：物料有"可用"和"停用"两种状态。停用的物料不影响已有需求，但不会出现在新增需求的下拉选择列表中
- **分页**：列表默认每页显示 100 条，超过可翻页

#### 批次物料需求（齐套判定）

**进入方式**：物料管理页面内的子导航 → 批次物料需求

在这里为每个批次设置所需物料和到货情况：

**齐套概览**：未选择批次时，页面顶部会显示三张统计卡片——齐套、部分齐套、未齐套各有多少个批次，帮你快速掌握整体情况。选择某个批次后，页面会切换为该批次的物料需求明细。

**操作步骤**：
1. 先看概览卡片判断整体齐套情况
2. 在上方选择一个批次号
3. 点"新增"添加物料需求（选物料、填写需求数量和已到数量）
4. 系统会自动判定齐套状态并回写到批次信息中

> **小贴士**：如果某批次没有任何物料需求行，系统不会自动接管它的齐套状态——保留你在批次信息里手工填的值（或Excel导入的值）。只有添加了物料需求后，系统才会根据到料情况自动更新齐套。

> **补充**：批次物料需求里的"需求数量"必须大于 0；"已到数量"可以为 0，但不能为负数。

### 5.5 工作日历配置

**进入方式**：排产调度 → 工作日历配置

工作日历告诉系统哪些天可以排产、每天工作多长时间。

**操作步骤**：
1. 在日期选择器中选择一个日期
2. 选择类型：工作日 / 假期
3. 如果是工作日，填写：
   - 班次开始时间（默认 08:00）
   - 班次结束时间（可选）
   - 可用工时（比如 8 小时）
   - 效率（默认 1.0，假期加班可以设低一点）
   - 是否允许排普通件（是/否）
   - 是否允许排急件/特急（是/否）
4. 点"保存"

> **小贴士**：未配置的日期按默认规则处理——工作日默认8小时，周末和节假日默认不排产。所以只需要配置那些和默认规则不一样的日期就行（比如周末加班、节假日调休）。

#### 日历页面的贴心设计

- **点击已配置的日期会自动填表**：在日期选择器里点一个已经配置过的日期，下面的表单会自动把那天的类型、工时、效率都填好，方便你修改
- **周末自动识别**：新选一个周末日期时，类型会自动设为"假期"
- **已配置日期有高亮**：日期选择器里配过的日期有不一样的颜色，一眼就能看出哪些天已经配了
- **鼠标悬停看摘要**：把鼠标放在已配置的日期上，会浮出一个小提示，告诉你那天设的是什么（类型、工时、效率等）

> **小贴士**：如果页面顶部出现黄色警告横幅，说明高级设置中的"假期默认效率"读取出了问题（比如值被设成了 0 或非法值）。系统会自动回退到默认值 0.8 继续工作，但建议去高级设置页检查并重新保存一下假期效率配置。个人工作日历页面也有同样的提示。

### 5.6 个人工作日历

除了全局工作日历，还可以为每个人设置专属工作日历。

**进入方式**：人员管理 → 点击某人进入详情 → 个人工作日历

操作方式和全局日历一样。个人日历的优先级高于全局日历——如果某天全局设为工作日，但某人的个人日历设为假期，那这个人当天不会被安排任务。

**适用场景**：请假、个人加班、轮班差异。

也可以通过 Excel 批量导入个人工作日历（人员管理子导航 → 人员专属工作日历（Excel））。

## 6. 排产操作：完整指南

### 6.1 新增批次

**方法一：Excel批量导入**（适合一次导入多个批次）
1. 排产调度 → 批次信息（Excel导入/导出）
2. 下载模板、按第 1 章指南填写、上传
3. 选择导入模式：
   - **覆盖**：批次号相同的会被新数据更新，新批次号的会新增
   - **追加**：批次号相同的跳过，只导入新的
   - **替换**：先删掉所有旧批次，再全量导入
4. 勾选"导入后自动生成工序"（系统会根据图号对应的工艺模板自动创建工序）
5. 在预览页面检查数据是否正确，确认导入

> **注意**："替换"模式会删除数据库中所有已有批次，并且会一并删除已有的排程记录（不可恢复）。如果你已经有排产结果，请谨慎使用。建议先手动备份（系统管理 → 备份）。

> **小贴士**：导入完成后，已有的批次数据也可以在同一个页面导出为 Excel，方便备份或整理。

**方法二：页面手动添加**（适合零星添加）
1. 排产调度 → 批次管理 → 填写新增批次表单
2. 填写批次号、图号、数量、交期、优先级、齐套状态等
3. 点击"创建批次"，系统会自动根据图号生成工序

> **小贴士**：手动创建批次时，齐套状态默认就是"齐套"。

### 6.2 补充工序信息

1. 排产调度主页或批次管理页 → 点击批次号（或"查看/编辑"按钮）进入详情页
2. 找到要修改的工序
3. 根据工序类型不同，能改的内容也不一样：

**内部工序**：
- 设备：选哪台机器
- 人员：选谁来操作（会根据设备自动筛选可操作人员）
- 换型时间(h)：设备准备时间
- 单件时间(h)：做一个零件要多久

**外部工序**：
- 供应商：交给谁做
- 外协周期(天)：需要几天

4. 每行改完后点"保存"

> **小贴士**：如果工序信息填错了或者工艺路线变了，可以点"从零件模板重建工序（会覆盖补充信息）"，系统会按图号对应的最新工艺模板重新生成全部工序。会弹窗让你确认，因为这会清空已有的设备、人员、工时设置。

> **小贴士**：重建工序时旁边有一个 **"严格工艺解析"** 复选框。开启后，如果零件缺少模板且需要根据工艺路线字符串自动补建模板，未知工种、缺供应商映射或默认天数无效会直接拒绝重建，而不是按兼容逻辑自动回退。

#### 工序补充页面的贴心设计

- **设备和人员双向联动**：选了设备后，人员下拉会自动只显示能操作这台设备的人；反过来选了人员后，设备下拉也会自动过滤
- **组合不匹配会提示**：如果你选了一个不匹配的设备+人员组合，当前行下方会显示一行提示，告诉你"当前设备/人员组合不匹配"以及具体是哪道工序。保存时后端也会做校验
- **已删除资源不会"消失"**：如果你之前选的设备或人员后来被删了，系统不会把这个选择丢掉，而是显示成"XXX（已删除）"——提醒你该换一个了
- **不可用的资源不能选**：停用/维修的设备、停用的人员在下拉里会显示"（不可用：停用）"并且灰掉，点不了
- **合并周期组提示**：如果某道外协工序属于"合并周期"的外部组，外协周期的输入框会被锁住，旁边提示"该工序周期由外部组合并周期控制"——要改得去工艺管理里改
- **列宽可调**：工序表格的列可以拖动边框调宽度，方便看长文本
- **大批次不卡顿**：工序特别多的时候（超过 30 道），系统会自动启用"懒加载"——下拉框第一次点开才加载完整选项，不会一上来就全部渲染出来。如果你想关闭懒加载（比如想一次看到所有选项），在地址栏加 `?lazy_select=0` 即可

### 6.3 执行排产与模拟排产

**执行排产**

1. 排产调度主页 → 在"待排"列表里勾选要排产的批次（只有"待排"状态的批次才会出现复选框）
2. 设置排产参数：
   - **排产开始时间**：默认是明天早上8点，可以改成其他时间
   - **排产截止日期**（可选）：排产结果不能超过这个日期。如果工期太长排不完，系统会报错
   - **启用齐套约束**（勾选框）：勾选后，齐套状态为"未齐套"的批次不会参与排产。这个勾选框的默认值取自"高级设置"里的齐套约束配置，你也可以在这里临时勾选或取消，只影响这一次排产，不会改掉高级设置里的默认值
   - **严格调度参数校验**（勾选框）：开启后，如果高级设置中的排产方案、智能派工策略、自动分配设备人员等参数值非法，系统会直接拒绝排产，而不是按兼容逻辑自动回退。日常不需要开，排查问题时可以临时开启
3. 点击 **"执行排产"**
4. 排产完成后页面会显示结果摘要（成功/失败工序数、是否有超期等）。如果有告警（比如资源池降级、配置兼容回退等），页面会显示告警条数和前几条预览；完整告警列表可以在系统历史页查看

> **主页参数和高级设置的关系**：主页上的三个参数（开始时间、截止日期、齐套约束）只控制"这一次排产"的时间窗口和齐套开关。算法参数（排产策略、计算模式、优化目标、权重等）全在"高级设置"页面里，不需要每次排产都重新设。

**模拟排产（插单模拟）**

操作流程和执行排产一样，但点的是 **"模拟排产"** 按钮。会弹窗确认："将对所选批次执行一次【模拟排产】并生成新版本（不更新批次状态）。确定继续？"

> **模拟排产 vs 执行排产**：
> - **模拟**：只是看看效果，不改变任何批次的状态，生成一个新版本号。你可以反复模拟，反复对比。模拟成功后会自动跳转到甘特图查看结果。
> - **执行**：正式生效，批次状态会从"待排"变为"已排"。

#### 方案快捷切换

在排产主页上方有一个"方案"下拉框，可以快速切换预设的参数组合（比如"默认-稳定"、"交期优先"等）。选完就自动应用了，不用再点"保存"。

> **小贴士**：方案下拉框下方有时会出现一行灰色说明文字（比如"自定义方案，部分参数与任何内置预设不完全匹配"），帮助你理解当前方案的来源。高级设置页面也有同样的说明。

#### 留痕快照

每次排产（不管是执行还是模拟）后，主页底部会显示"最近一次排产"的快照信息，包括版本号、排产时间、策略、结果摘要等。

### 6.4 甘特图

**进入方式**：排产调度 → 甘特图-设备 或 甘特图-人员

甘特图用条形图展示排程结果。横轴是时间，纵轴是资源（设备或人员）。

#### 视图切换

页面上方有两个按钮：**"设备视图"** 和 **"人员视图"**，点击即可切换。

#### 时间和版本控制

- 用 **"上周 / 回到本周 / 下周"** 按钮快速切换周
- 用 **"指定周"** 输入一个日期跳到那一周
- 用 **"排产区间起/止"** 设置自定义时间范围
- 用 **"版本"** 下拉框切换不同排产版本的结果
- 设好参数后点 **"加载"**

**版本选择规则**：
- 不指定版本或选择 `latest` 时，系统会取当前最新排产历史。
- 如果当前没有任何排产历史，页面会显示空态，不会假装存在 v1 或 v0。
- 如果手动输入了不存在的版本号（例如 `version=999`），页面会明确提示"排产版本不存在"或"v999 无对应排产历史"，不会自动切回最新版本。
- 报表和导出也遵守同一规则：未指定版本可取最新；显式指定不存在的版本会失败并提示选择已有版本。

#### 筛选与配色

点击 **"筛选与配色"** 展开筛选面板，里面包含：

**时间粒度**：日视图 / 周视图 / 月视图

**配色模式**（默认按批次）：
- **按批次**：同一批次的工序同色，方便追踪一个批次的所有工序
- **按优先级**：普通/急件/特急各有颜色，一眼看出哪些是急单
- **按来源**：内制和外协用不同颜色，区分自己做和外面做
- **按工序状态**：未开始、进行中、已完成各有颜色

**筛选**：
- 按 **批次号** 筛选：只看某几个批次
- 按 **设备/人员** 筛选：只看某台设备或某个人的任务
- 勾选 **仅超期**：只显示超过交期的任务（快速定位问题）
- 勾选 **仅外协**：只显示外协工序

**关键链高亮**：默认勾选。系统会用外框标出决定总工期的那条最长路径，帮你发现瓶颈——优化关键链上的工序，总工期才能缩短。关键链是基于**整个排产版本**计算的，不按当前周或自定义区间截断；页面只高亮当前时间窗口内可见的关键链任务，图例会区分"全版本关键链数"和"本窗口可见数"。

**依赖箭头**：可以选择显示"仅关键链箭头"、"完整工艺依赖箭头"、或"不显示箭头"。

#### 聚焦与详情

- **聚焦**：点击甘特图上的某个任务条，同一批次的所有工序会高亮，其他工序变淡。再点一次取消聚焦
- **详情弹窗**：点击任务条后会弹出详情框，显示这道工序的完整信息（时间、批次、图号、设备、人员、优先级、交期等）

#### 甘特图的贴心设计

- **超期的任务有红色边框**，一眼就能看到哪些批次超期了
- **外协工序用虚线边框**，和内部工序一眼就能区分
- **周末和假期有底色标记**：时间轴上没配置日历的周末会自动按假期显示，有不一样的底色
- **日历加载失败提示**：如果工作日历加载失败（比如日历数据损坏或为空），甘特图上方会出现黄色提示"工作日历加载失败，当前不显示假期/停工背景标注"。排产结果本身不受影响，只是底色标记暂时不可用
- **数据异常提示**：如果加载过程中有部分记录因为时间格式损坏而无法展示，页面会显示"已过滤 N 条时间不合法的排程记录"，帮你判断当前看到的数据是否完整
- **超期标记降级提示**：如果系统在标记超期任务时遇到数据异常（比如超期摘要格式损坏），甘特图上方会出现黄色提示"超期标记可能不完整"，同时"仅超期"筛选框会被暂时禁用。数据本身不丢失，修复数据源后刷新即可恢复
- **关键链不可用提示**：如果关键链计算失败或当前版本没有可计算数据，页面会显示简短中文原因，并清空关键链任务、依赖边和边类型统计；这表示"关键链高亮不可用"，不是排程数据被删除
- **退化与超期提示**：接口会返回公开的退化事件和超期标记状态。页面只展示中文提示，不展示内部键名、异常路径或原始堆栈
- **网址可以分享**：甘特图的所有筛选条件（版本、时间范围、配色、关键链等）都记在网址里。你可以把网址发给同事，他打开后看到和你一模一样的画面。如果链接里带了某个批次号或设备编号筛选，但对方的下拉框里没有这个选项，系统会自动补上一项并选中
- **重置**：点"重置"按钮可以一键清除所有筛选条件
- 甘特图上有个 **"怎么看？怎么交互？"** 的说明面板，点击可以展开查看操作提示

### 6.5 周计划导出

**进入方式**：排产调度 → 周计划

1. 设置排产区间的起止日期（或用"指定周"跳到某一周）
2. 选择排产版本
3. 页面上会预览数据
4. 点击 **"导出周计划表.xlsx"** 下载Excel

导出的 Excel 包含这些列：日期、批次号、图号、工序、设备、人员、时段。跨天的工序会被自动拆成多行。外协工序在"设备/人员"列会显示外协提示（如"外协 XXX供应商"）。

> **小贴士**：如果页面上出现"已过滤 N 条时间不合法的排程记录"的提示，说明有少量工序因开始时间或结束时间异常未能展示。可以回到甘特图、周计划或排产历史页交叉核对完整数据。

### 6.6 资源排班中心

**进入方式**：排产调度 → 资源排班中心

**资源排班中心**可以让你从不同角度查看任务安排：

1. 选择查看视角：
   - **人员**：选一个人，看他的所有任务
   - **设备**：选一台设备，看它的排程
   - **班组**：选一个班组，看整组的任务分布（还需要选择**班组轴**：按人员展开还是按设备展开）
2. 选择时间范围（按周、按月、或自定义起止日期）
3. 选择排产版本
4. 点击 **"查询"**

查询结果有三种展示方式，可以点击标签页切换：
- **任务明细**：表格列出每条排程的详细信息（开始/结束时间、工序、设备、人员等）
- **日历矩阵**：按日期格子展示任务分布，跨天任务会按自然日拆分
- **甘特图**：用时间条形图展示，支持调整粒度（日/周/月）

**统计卡片**：页面上方会显示 5 个汇总数据——任务数 / 总工时（小时） / 超期任务 / 跨班组&外协未分配 / 跨天任务。

> **小贴士**：班组视角下，任务明细会分成"班组人员任务""班组设备任务""跨班组借调"三个分区，并且每个任务都标注了"同班组"还是"跨班组借调"。

> **小贴士**：排班数据支持导出 Excel，点击 **"导出资源排班.xlsx"** 按钮即可。

> **小贴士**：资源排班中心加载数据时，如果有部分行因时间格式损坏被跳过，页面会提示跳过数量。导出 Excel 时如果发生数据退化，导出文件的页面提示会包含退化原因说明。

> **小贴士**：如果超期统计数据格式异常，页面上方会出现黄色提示"超期统计和标记可能不完整"。这不影响任务明细的正常查看，修复数据源后刷新即可恢复。

### 6.7 排产优化分析

**进入方式**：排产调度 → 排产优化分析

1. 选择一个版本号，点击 **"加载"**。如果手动输入了不存在的版本，页面会保留该版本号并显示"不存在"占位，不会默认选中最新版本
2. 页面上可以看到：

**版本概览**：这次排产用了什么方式、什么时间跑的、是否成功。如果是精细计算，还会展示"评分顺序"（系统按什么先后来比较方案好坏）和"本次配置快照"（排产那一刻实际生效的参数），方便你事后复盘。

**约束降级提示**：如果"停机避让"或"锁定近期排程（冻结窗口）"在排产时遇到异常（比如停机数据加载失败、上一版本排程缺失等），页面会用黄色横幅提示你该约束已降级，并说明原因。没有降级时不会出现这个提示。

**排产摘要退化提示**：如果排产过程中发生了可见的退化事件（如冻结窗口降级、停机避让降级、资源池构建降级、交期数据异常、历史外协周期兼容回退等），页面会在版本概览下方用黄色横幅逐条列出退化原因和次数，帮你快速判断本次排产是否存在数据或约束层面的异常。

**数据裁剪提示**：当排产数据量特别大（摘要超过 512KB）时，系统会自动裁剪方案列表、优化曲线和超期明细，确保页面可以正常加载。如果发生了裁剪，页面顶部会用黄色横幅提示你"当前展示为裁剪后摘要"，此时显示的方案列表和超期明细可能不完整。
**指标卡片**（共 10 项）：

| 指标 | 说明 |
|------|------|
| 数据异常批次数 | 当前主要统计交期数据异常（如格式错误、空白、非法日期等）的批次数；这些批次不参与超期计算但仍然会被排产。为 0 说明没有发现这类异常 |
| 超期批次数 | 有多少个批次超过了交期（越少越好） |
| 总拖期（小时） | 所有超期批次加起来拖了多少小时（越少越好） |
| 加权拖期（小时） | 按优先级加权的拖期，特急的拖1小时比普通的严重 |
| 总工期（小时） | 从排产开始到最后一道工序结束的总时长 |
| 内制工期（小时） | 仅统计内部工序的工期，不含外协 |
| 换型次数 | 设备切换产品类型的次数（越少越好） |
| 设备平均利用率 | 设备被使用的时间占比（越高越好） |
| 人员平均利用率 | 人员被使用的时间占比（越高越好） |
| 未排批次数 | 有多少个批次未能被成功排入计划（比如全部工序被冻结、没有可用资源等）。为 0 说明全部批次都排上了 |

每个指标旁边还会显示和上个版本的差值，一眼看出变好了还是变差了。

> **小贴士**：如果摘要中出现"冻结窗口"区域，说明本次排产启用了"锁定近期排程"功能，部分工序被保持了上一版的位置不动。页面会详细展示：冻结窗口的启用状态、当前状态（已生效/已降级/未启用）、锁定天数、是否实际生效、冻结工序数、冻结批次数。如果冻结了多个批次，还会列出最多 5 个示例批次号；如果冻结窗口发生了降级（比如上一版本排程缺失），页面还会显示降级原因。

**方案列表**：精细计算模式会列出系统尝试过的各种方案，包括排序策略（优先级优先/交期优先/综合等）和派工模式（按批次顺序/智能派工）。每行会保留可追溯的方案来源或标签；非智能派工时不会显示空的 `/-`。你可以看到每个方案的失败工序数、核心指标和综合评分，了解系统是怎么一步步找到最优方案的。

**优化曲线**：如果用了"精细计算"模式，这里会画出系统寻找更优方案的过程——一条下降的曲线说明系统在持续改进。快速计算模式不会有优化曲线。

**版本趋势**：最近几个版本的各项指标变化折线图，看看是不是一版比一版好。至少需要 2 个版本才会显示趋势图。

选中版本后，页面上还有 **"查看甘特图-设备"** 和 **"查看甘特图-人员"** 的快捷链接，方便直接跳转查看当前版本的甘特图。

> **场景示例**：你先用"快速计算"跑了 v1，又用"精细计算"跑了 v2。打开分析页面，切换 v1 和 v2 对比：v2 超期减少了2个、换型少了3次，但多花了40秒计算时间——值不值得？你说了算。

### 6.8 批次管理

**进入方式**：排产调度 → 批次管理

批次管理页提供新增、复制、删除、批量修改等功能。

#### 复制批次

1. 勾选要复制的批次
2. 点击 **"复制所选批次"**
3. 弹窗确认后，系统会自动给批次号末尾数字加1（如 B001 → B002）。如果 B002 已经存在了，会继续+1 直到找到一个没被占用的编号

> **注意**：如果原批次号末尾没有数字（比如"TEST-A"），复制会失败，需要手动改批次号。

#### 删除批次

1. 勾选要删除的批次
2. 点击 **"删除所选批次"**
3. 弹窗确认："确定删除所选批次吗？将级联删除批次工序与排程记录（若存在）。"

也可以在每行末尾单独删除一个批次。

#### 批量修改

多个批次需要改同一个属性（比如统一改优先级或交期）：

1. 勾选要修改的批次
2. 点击 **"批量修改"**
3. 选择要改的内容：优先级、交期、或备注。只有你填写了的字段才会被修改，留空的不会动
4. 弹窗确认："确定批量修改所选批次吗？仅会修改你填写的字段。"

#### 批次管理页的贴心设计

- **状态筛选**：页面上方是状态下拉框，可选 待排 / 已排 / 加工中 / 已完成 / 已取消 / 全部，方便只看某一类状态的批次
- **齐套筛选**：状态筛选旁边还有齐套下拉框，可按"齐套/未齐套/部分齐套"筛选，方便在执行排产前检查哪些批次还没齐套
- **全选/取消全选**：点表头的复选框可以一键选中/取消当前页所有批次
- **已选计数**：页面上会显示"已选 X 个批次"，勾选变化时自动更新
- **前端搜索**：在搜索框里输入关键词，表格会即时过滤（不用刷新页面）
- **表头点击排序**：点列头可以按那一列排序（批次号、图号、数量、交期等）

## 7. 高级设置：每个开关的作用

**进入方式**：排产调度 → 高级设置/模板

高级设置页面有很多开关，下面逐个解释。日常大多数情况不需要改，先选个"常用方案"就够了。

#### 页面总览

页面分两个区域：

- **上方——常用方案**：一个卡片区域，包含方案切换下拉框、"保存为新方案"和"删除方案"的操作区
- **下方——排产策略配置**：分两部分
  - **常用参数**（直接显示）：排产策略、计算模式、优化目标、优先级权重、交期权重
  - **更多设置**（折叠区域，点击展开）：派工方式、智能派工策略、计算时间上限、假期工作效率、以及一组功能开关（锁定近期排程、自动分配、齐套约束、优先推荐主操、深度优化）

页面顶部还会展示"当前运行配置"：包括当前配置的基线方案、是否与方案一致、是否发生兼容修复，以及"自动回写资源"的当前状态。这里展示的是排产真正读取到的配置，不只是页面表单当前值。

#### 基本操作

- **修改并保存**：改好参数 → 点 **"保存当前设置"** → 页面提示"排产策略配置已保存"
- **恢复默认**：点 **"恢复默认"** 按钮 → 弹窗确认"确定恢复默认设置吗？当前页面的所有参数都会重置。" → 确认后所有参数回到出厂值，方案也会切回"默认-稳定"

> **隐藏修复提示**：配置页有少量内部字段不会直接出现在表单里，例如 `auto_assign_persist`（自动分配结果回写，默认 `yes`）。如果旧数据缺少这些隐藏字段，系统会用中文字段名提示"涉及字段"，并在允许的情况下修复为当前默认值；严格保存或方案应用时遇到非法隐藏字段，会直接阻止保存而不是静默吞掉。

### 7.1 常用方案

**作用**：一键切换预设的参数组合，省得一个一个调

**系统自带 4 个方案，具体参数如下**：

| 方案名 | 排产策略 | 计算模式 | 优化目标 | 优先级权重 | 交期权重 | 计算时间 | 派工方式 |
|--------|---------|---------|---------|-----------|---------|---------|---------|
| 默认-稳定 | 优先级优先 | 快速计算 | 最少超期 | 0.4 | 0.5 | 20秒 | 按批次顺序 |
| 交期优先 | 交期优先 | 快速计算 | 最少超期 | 0.2 | 0.7 | 20秒 | 按批次顺序 |
| 换型最少 | 优先级优先 | 精细计算 | 最少换型次数 | 0.4 | 0.5 | 30秒 | 按批次顺序 |
| 改进-更优(慢) | 优先级优先 | 精细计算 | 最少拖期小时 | 0.4 | 0.5 | 120秒 | 按批次顺序 |

> **建议**：日常用"默认-稳定"就够了。切换方案后，下面的参数会自动变成方案对应的值。

#### 方案操作步骤

- **切换方案**：在下拉框里选一个方案 → 系统自动保存并刷新页面 → 参数全部变成方案对应的值
- **保存为新方案**：先调好参数并点"保存当前设置" → 在"保存为新方案"输入框里起个名字（不超过 50 字） → 点 **"保存"** → 弹窗确认 → 下次在下拉框里就能看到了
- **删除方案**：在"删除方案"下拉框里选一个自定义方案 → 点 **"删除"** → 弹窗确认
- **主页快捷切换**：排产主页上方也有方案下拉框，选完立刻生效，和高级设置页是同一套方案

#### 容易误会的几个点

- 选了 **"当前手动设置"** 只是把方案名标记回"自定义"，**不会改变任何参数**——你之前设的值都还在
- 你**手动改了任意参数并保存**后，当前方案会自动变成"自定义"（因为参数已经和原来的方案不一样了）
- 系统自带的 4 个方案**不能删除，也不能覆盖**
- 方案名**不能为空**，不能和系统自带方案同名
- **"恢复默认"不等于"切换到默认-稳定方案"**——恢复默认会把所有参数（包括功能开关）都重置为出厂值，相当于"全部从头来"
- "基线方案"只是说明当前运行配置最初来自哪个方案；如果页面提示当前运行配置和基线不一致，应以页面展示的"当前运行配置"为准

### 7.2 排产策略

**作用**：决定先排哪些批次

**默认**：优先级优先

**在页面哪里**：常用参数区域，直接显示

| 策略 | 说明 | 适合场景 |
|------|------|----------|
| 优先级优先 | 特急 > 急件 > 普通，同优先级按交期排 | 有VIP客户、需要保急件时 |
| 交期优先 | 交期近的先排，没交期的排最后 | 交货压力大、赶交付时 |
| 综合 | 综合考虑优先级和交期，按权重打分排序 | 需要平衡时 |
| 先进先出 | 按批次创建时间排 | 公平对待所有订单 |

#### 综合策略的权重怎么调

选择"综合"时，页面上会出现两个输入框——"优先级重要程度"和"交期重要程度"。

**出厂默认**：优先级 0.4、交期 0.5

**怎么理解这两个数**：比例关系决定排序。比如优先级填 40、交期填 50，意思是交期稍微比优先级重要一点。如果填 80 和 20，那基本只看优先级。

**推荐值参考**：

| 场景 | 优先级权重 | 交期权重 | 效果 |
|------|-----------|---------|------|
| 交付压力大 | 20 | 80 | 交期几乎决定一切，但特急件仍有微弱加分 |
| 日常平衡 | 50 | 50 | 优先级和交期各占一半 |
| VIP客户优先 | 70 | 30 | 主要看优先级，交期作参考 |

**填法**：可以填小数（0.4 / 0.5）也可以填整数（40 / 50），系统会自动按比例处理。

> **只有选了"综合"策略才需要调权重**——选其他三种策略时权重不生效。

> **防报错**：两个权重之和不能超过 1（或 100%），否则保存时会报错。不能填负数，不能填文字。

### 7.3 计算模式

**默认**：快速计算

**在页面哪里**：常用参数区域，直接显示

| 模式 | 速度 | 效果 | 说明 |
|------|------|------|------|
| 快速计算 | 快 | 够用 | 按规则一次排完，几秒出结果 |
| 精细计算 | 慢 | 通常更好 | 先快速做一版，然后反复调整优化，在时间预算内找更好的方案 |

> **条件关系**：选了精细计算后，"计算时间上限"和"优化目标"才会真正参与择优。快速计算时这两项虽然可以设，但不影响结果（因为快速计算只排一版，没有比较和择优的过程）。

> **场景示例**：平时用"快速计算"，月度大排产或者对结果不满意时切到"精细计算"试试。精细计算每次运行结果可能略有不同（因为它在探索不同可能性），但一般都比快速计算好。

### 7.4 派工方式

**默认**：按批次顺序

**在页面哪里**：更多设置 → 展开后可见

| 方式 | 说明 |
|------|------|
| 按批次顺序 | 一个批次排完再排下一个（简单稳定，推荐日常用） |
| 智能派工 | 不按批次顺序，而是看哪道工序当前准备好了就先排（更灵活，适合复杂场景） |

选择"智能派工"时，还可以进一步选择挑选策略：

| 策略 | 说明 | 适合场景 |
|------|------|----------|
| 时间余量少的先做 | 离交期近、余量最少的工序优先 | 多数场景适用 |
| 交期更紧的先做 | 看交期和剩余加工时间的比值，越吃紧越优先 | 工序多、交期密集时 |
| 综合判断更紧急的先做 | 系统综合多个因素判断紧急程度 | 想让系统自己决定时 |

> **注意**：智能派工策略只有在选了"智能派工"时才生效（页面上也标了"仅智能派工时生效"）。除非"按批次顺序"的排产结果让你不满意（比如超期太多），否则不用改。

### 7.5 优化目标

**作用**：告诉系统"什么叫好的排产结果"

**默认**：最少超期

**在页面哪里**：常用参数区域，直接显示

> **条件关系**：只在精细计算模式下真正参与"择优"。快速计算模式下设了也不影响结果。

| 目标 | 优先考虑 | 适合场景 |
|------|----------|----------|
| 最少超期 | 尽量减少超期的批次数量 | 交期刚性、不允许任何批次超期时 |
| 最少拖期小时 | 尽量减少总的延迟时间 | 允许个别批次稍微超期，但总延迟要最小 |
| 最少加权拖期小时 | 尽量减少按优先级加权后的延迟时间 | 特急件拖一小时比普通件严重得多，想让系统区别对待时 |
| 最少换型次数 | 尽量减少设备调整次数 | 换型成本高、想减少设备频繁切换时 |

> "最少拖期小时"和"最少加权拖期小时"有什么区别？
> - "最少拖期小时"只看总延迟，不管哪个批次拖的——10个普通件各拖1小时 = 1个特急件拖10小时，系统觉得一样严重。
> - "最少加权拖期小时"会按优先级区分：特急件拖1小时的"惩罚"是普通件的3倍，急件是2倍。所以系统会更卖力地保住特急件的交期。

> **场景示例**：
> - 客户合同有逾期罚款？选"最少超期"，一个都不能超。
> - 超期难免，但想把影响降到最低？选"最少拖期小时"。
> - 特急件绝对不能拖，普通件可以稍微让让？选"最少加权拖期小时"。
> - 换一次型要停机2小时调模具？选"最少换型次数"。

### 7.6 计算时间上限

**作用**：精细计算模式下，系统最多花多长时间来寻找更好的方案

**默认**：20 秒

**在页面哪里**：更多设置 → 展开后可见

> **条件关系**：只在精细计算模式下生效。快速计算模式一般几秒就出结果，不受这个限制。

**推荐值**：

| 批次数量 | 建议时间 |
|---------|---------|
| 20 个以下 | 20 秒 |
| 20~50 个 | 30~60 秒 |
| 50 个以上 | 60~180 秒 |

时间越长不一定结果越好，但给系统更多时间探索，通常不会更差。

> **防报错**：至少填 1 秒，不能为空，不能填文字。

### 7.7 锁定近期排程

**作用**：保持近期排程不变，只重新排后面的

**默认**：关闭，天数 0

**在页面哪里**：更多设置 → 功能开关

**操作**：勾选 → 填入天数 → 点"保存当前设置"。取消勾选即关闭。

> **场景示例**：今天是周一，已经开始按排程生产了。你希望未来3天的安排不要变（因为物料已经准备好了、人员已经通知了），只重新排3天以后的。这时勾选"锁定近期排程"，填入天数"3"。

**锁定后会怎样**：
- 锁定窗口内的工序会保留上次排产的时间安排，直接复用到新版本中，不会被重新排
- 只有窗口外的工序会参与重新排产
- 如果上次排产结果中没有这些工序的数据，系统会跳过并给出提示

> **注意**：锁定可能让整体结果变差（因为优化空间小了），但能保证现场稳定、不用来回改计划。天数不能小于 0。

### 7.8 自动分配

**作用**：工序没有选设备/人员时，系统自动选一个

**默认**：关闭

**在页面哪里**：更多设置 → 功能开关

**操作**：勾选 → 点"保存当前设置"。取消勾选即关闭。

**系统怎么选**：
- 优先选 **完工时间最早** 的设备和人员（让工序尽快做完）
- 尽量选 **同类工序** 用过的设备（减少换型）
- 在以上条件相同时，选 **更空闲** 的设备和人员（负荷均衡）
- 在以上条件都相同时，会微弱倾向主操和高技能人员（这是自动分配的内置逻辑，无需额外开启其他开关）

> **场景示例**：你只想快速看个粗略排产结果，还没来得及给所有工序选设备和人员。开启自动分配，系统帮你选，先出个排程看看。

> **注意**：自动选的不一定完全符合实际情况（比如某台设备正在维修、某个人请假了），建议后续手动确认重要工序的分配。

### 7.9 齐套约束

**作用**：物料没齐的批次不允许排产

**默认**：关闭

**在页面哪里**：更多设置 → 功能开关

**操作**：勾选 → 点"保存当前设置"。

- **开启**：齐套状态为"未齐套"的批次会被拦截，不参与排产
- **关闭**：不管齐不齐套，所有勾选的批次都能排

> **和执行排产页面的关系**：这里设的是"默认值"——执行排产时页面上的"启用齐套约束"勾选框会默认按这个来。你也可以在执行排产时临时勾选或取消，只影响那一次排产，不会改掉这里的默认值。

> **小贴士**：齐套状态在批次信息里设置。如果批次填了"齐套日期"，系统会在该日期之前不安排这个批次开工。

### 7.10 假期默认效率

**作用**：周末/假期加班但没有单独设置效率时，按多少算

**默认**：0.8（即 80%）

**在页面哪里**：更多设置 → 展开后可见

意思是假期加班效率比正常工作日低一点。如果你的车间假期效率和平时一样，改成 1.0 就行。

> **防报错**：必须大于 0，不能填 0 或负数。

> **小贴士**：如果某天在工作日历里单独设置了效率，那天就用单独设置的值，不用这个默认值。

### 7.11 优先推荐主操/高技能人员

**作用**：开启后，在批次详情页编辑工序选人时，人员下拉框会把主操、高技能人员排在最前面，方便快速选择

**默认**：关闭

**在页面哪里**：更多设置 → 功能开关

**操作**：勾选 → 点"保存当前设置"。

> **场景示例**：你们车间每台设备都有主操和副手。开启这个开关后，选人员时主操会排在最前面，不用翻找。

> **注意**：这个开关只影响下拉框的排列顺序，不影响排产算法本身。排产算法有自己的选人逻辑。

### 7.12 深度优化

**作用**：在精细计算模式下，用额外的计算组件为排产寻找一个更优的起点，通常能让最终结果更好

**默认**：关闭，时间 5 秒

**在页面哪里**：更多设置 → 功能开关

**操作**：勾选 → 填入时间上限 → 点"保存当前设置"。

**前提条件**：必须同时把计算模式设为"精细计算"，否则本开关不生效。精细计算会在这个起点的基础上继续优化。

> **防报错**：秒数至少填 1，建议 3~15 秒。过大的时间上限会让排产变慢，建议不要超过 15 秒。

> **注意**：这个功能需要系统支持额外的计算组件（OR-Tools）。如果你的环境里没有安装，开启后排产不会报错，系统会自动跳过这一步继续排。

> **深度优化和精细计算是什么关系？** 深度优化是精细计算的"加速器"——精细计算（7.3 节）会反复尝试不同顺序找更优方案，深度优化帮它找到一个更好的起点，让精细计算更快收敛到好结果。单独开启深度优化但不开精细计算，深度优化不会生效。

### 7.13 怎么填才不容易报错

| 设置项 | 怎么填 | 常见报错 |
|--------|--------|---------|
| 优先级/交期权重 | 填 0~1 的小数或 0~100 的整数；两者之和不超过 1（或 100%）；不能填负数、不能填文字 | "优先级权重 + 交期权重 之和不能超过 1" |
| 计算时间上限 | 正整数，至少 1，建议不超过 180 | "计算时间上限不能为空" |
| 假期工作效率 | 大于 0 的数字 | "假期工作效率必须大于 0" |
| 锁定天数 | 0 或正整数 | — |
| 深度优化秒数 | 正整数，至少 1，建议 3~15 | — |
| 方案名称 | 不能为空，不超过 50 字，不能和系统自带方案同名 | "方案名称不能为空" / "系统自带方案不能覆盖" |

> **注意**：以上数值字段在严格模式下，**空白值也会被拒绝**（而不是静默补 0）。如果你看到"xxx不能为空"的提示，说明对应字段需要填入一个有效的数字。这比以前更严格，但能避免"系统帮你补了个 0 你却不知道"的隐患。

## 8. 算法选择：快餐还是私厨

### 快餐模式（快速计算）

像快餐店一样快速出餐：
- **速度**：几秒钟出结果
- **原理**：按你设定的策略（优先级优先/交期优先等）排好顺序，然后依次把每道工序放到最早能做的时间点上，排完就是最终结果
- **优点**：快、稳定、每次跑结果一样
- **缺点**：不一定是最优解，因为排完不回头调整

**适合场景**：
- 日常排产，数据经常变
- 先排一版看看大致情况
- 对排产结果没有特别高的要求

### 私厨模式（精细计算）

像私人厨师一样精心烹制：
- **速度**：几十秒到几分钟（取决于你给的"计算时间上限"）
- **原理**：先用快速计算出一版方案，然后在时间预算内不断尝试——交换工序顺序、换设备、调时间——看看能不能找到更好的排法
- **优点**：通常比快速计算好（超期更少、或换型更少）
- **缺点**：慢；每次结果可能略有不同（因为每次探索的路径不一样）

**适合场景**：
- 月度/周度重要排产
- 对交期或换型有严格要求
- 愿意多等一会儿换更好的结果
- 用快速计算跑出来不满意，想试试能不能更好

### 该怎么选？

| 情况 | 推荐组合 | 说明 |
|------|---------|------|
| 每天日常排产 | 快速计算 + 按批次顺序 | 几秒出结果，够用 |
| 交期很紧张 | 精细计算 + 交期优先 + 60秒 | 让系统多花时间找更好的方案 |
| 换型成本高 | 精细计算 + 最少换型次数 + 60秒 | 尽量减少换型 |
| 特急件多、不能拖 | 精细计算 + 最少加权拖期小时 + 60秒 | 优先保住高优先级批次 |
| 快速看个大概 | 快速计算 + 自动分配 | 不用填设备人员也能排 |
| 月度大排产 | 精细计算 + 最少加权拖期小时 + 120秒 | 给充分时间优化，优先保重要批次 |

> **小贴士**：你可以先用"快速计算"跑一版，再用"精细计算"跑一版，然后去"排产优化分析"页面对比两个版本的指标，选效果更好的那个。

## 9. 报表中心

**进入方式**：顶部导航 → 报表

报表中心提供三种统计报表，都基于排产版本的数据生成，支持导出 Excel。

> **注意**：当导出数据量特别大（超过系统预设阈值）时，系统会拒绝导出并提示"导出范围过大，请缩小范围后重试"。此时请缩小日期范围或选择数据量较少的版本后重试。

> **导出失败口径**：报表导出只显示公开中文错误，例如版本不存在、暂无排产历史或导出范围过大；不会把内部路径、原始异常或 JSON 内部键名直接展示给用户。

### 报表首页

打开报表中心后先看到的是首页。

- 页面顶部显示**最新排产版本号**和**超期批次数量**两张统计卡片
- 如果还没有排产历史，页面会提示"暂无排产历史，请先执行排产后再查看报表"
- 下方有三张导航卡片，分别进入超期清单、资源负荷与利用率、停机影响统计

> **小贴士**：所有报表都基于排产版本的数据生成。如果你刚执行了新一轮排产，记得在报表里选最新版本。

### 9.1 超期清单

**进入方式**：报表中心 → 超期清单

**操作步骤**：
1. 在"排产版本"下拉框中选择要查看的版本（不选默认取最新版本）。每个选项会显示版本号、排产时间和结果状态，方便你找到目标版本
2. 点击 **"查询"**
3. 页面显示结果统计：总超期数、已排程逾期数、未排程逾期数

**两种超期的区别**：

| 类别 | 含义 |
|------|------|
| 已排程逾期 | 这个批次已经排了产，排出来的完工时间晚于交期 |
| 未排程逾期 | 这个批次还没排产（或排了但没生成排程），截至当前时间已经过了交期 |

**表格列说明**：类别（已排程/未排程）、批次号、图号、名称、数量、交期、完工/截至时间、超期(天)、超期(小时)。

**导出**：有超期数据时，"导出 Excel"按钮可用，点击下载；没有数据时按钮会灰掉。

> **小贴士**：如果超期清单显示 0 个，恭喜——说明当前排产方案没有超期批次。

> **小贴士**：超期天数 = 超期小时 / 24 并向上取整。所以超期 1 小时也算超期 1 天。

### 9.2 资源负荷与利用率

**进入方式**：报表中心 → 资源负荷与利用率

**操作步骤**：
1. 选择排产版本
2. 设置日期范围（开始日期和结束日期）
3. 点击 **"查询"**

**日期范围的自动带入逻辑**：
- 如果你没填日期：系统会自动按所选版本的排程时间范围来填。比如排产结果从4月1日排到4月30日，日期就自动设为4月1日到4月30日
- 如果版本没有排程数据：默认显示最近7天
- 页面上会有一行提示告诉你日期是怎么来的（"已按所选版本的排程范围自动带入日期"或"已回退到最近 7 天"）

**表格分两部分**：

**设备负荷**：

| 列 | 说明 |
|------|------|
| 设备编号 | 设备编号 |
| 设备名称 | 设备名称 |
| 排程负荷(小时) | 这段时间内该设备被排了多少小时任务 |
| 任务数 | 排了多少条工序 |
| 可用工时(小时) | 根据工作日历算出的可用时间 |
| 利用率(%) | 排程负荷 / 可用工时 |

**人员负荷**：

| 列 | 说明 |
|------|------|
| 工号 | 人员工号 |
| 姓名 | 人员姓名 |
| 排程负荷(小时) | 这段时间内该人被排了多少小时任务 |
| 任务数 | 排了多少条工序 |
| 可用工时(小时) | 根据工作日历算出的可用时间 |
| 利用率(%) | 排程负荷 / 可用工时 |

**怎么看利用率**：利用率 = 排程负荷 / 可用工时。比如某设备在这段时间内可用40小时、排了30小时任务，利用率就是75%。

**"单资源可用工时"**：页面筛选区下方会显示"单资源可用工时：XX小时"——这是根据工作日历计算的、一台设备/一个人在你选的日期范围内总共可用多少小时。

**导出**：Excel 包含两个工作表：machines（设备）和 operators（人员）。页面和 Excel 都按百分比显示利用率，表头统一为"利用率(%)"，例如 0.75 会显示为 75。

> **小贴士**：利用率超过100%说明排程超出了可用工时——可能是因为工作日历没配对（比如排了周末但日历没设加班），或者工序太多设备不够用。

### 9.3 停机影响统计

**进入方式**：报表中心 → 停机影响统计

**操作步骤**：同利用率页面——选版本、选日期范围、点 **"查询"**。

**日期范围自动带入逻辑**：同上。

**表格列说明**：

| 列 | 说明 |
|------|------|
| 设备编号 | 有停机计划的设备 |
| 设备名称 | 设备名称 |
| 停机时长(小时) | 该设备在日期范围内的总停机时间 |
| 停机次数 | 有几段停机计划 |
| 与排程重叠(小时) | 停机计划和排产任务时间上撞车的总时长 |
| 重叠次数 | 有几段时间撞车了 |

**"与排程重叠"是什么意思**：就是停机计划和排产任务在时间上撞车了。比如你设了某台设备周三下午2点到4点停机保养，但排产系统把一个任务排在了这台设备的下午1点到5点——那就有2小时重叠。重叠意味着实际执行时这个任务可能做不了或要延误。

**怎么用这个报表**：
- 停机次数多但重叠为0 → 停机安排合理，没有和排产冲突，不用担心
- 有重叠 → 需要调整停机时间，或者重新排产让系统自动避开

**导出**：导出 Excel 包含完整的停机影响明细。

> **小贴士**：如果你在排产之前就把停机计划配好了，排产算法会自动避开停机时段，正常情况下不会出现重叠。重叠通常出现在"先排了产，后来又加了停机计划"的场景——这时候要么调整停机时间，要么重新排一次产。

## 10. 系统维护

**进入方式**：顶部导航 → 系统

### 10.1 备份与恢复

**进入方式**：系统管理 → 备份/恢复

#### 手动备份

点击 **"手动备份"** 按钮，系统会立即创建一个数据库备份文件（文件名包含日期时间）。

#### 恢复数据库

在备份列表中找到要恢复的备份，点击 **"恢复"**。会弹窗确认："确认从该备份恢复数据库？恢复会覆盖当前数据。"

> **小贴士**：恢复前系统会自动创建一份"恢复前自动备份"；如果恢复过程失败，系统会尽最大努力回滚到这份备份。重要操作前仍建议手动再留一份备份。当前页面通常会显示两种结果状态：
> - **已验证**：页面会提示"已从备份恢复并完成结构校验……"，可以放心使用。
> - **失败**：如果恢复过程出错，或恢复后的结构校验没有通过，系统会提示失败，并尽最大努力回滚到恢复前自动备份。

恢复成功的门槛不只是文件复制完成，还包括恢复后的数据库结构校验。若页面提示"数据库文件已恢复，但后续结构检查失败，请查看日志后再继续使用"，表示文件已经复制，但不能视为完整恢复成功。

> **注意**：恢复完成后，建议刷新页面或重新打开浏览器，确保看到最新数据。

#### 清理过期备份

点击 **"清理过期备份"**，系统会删除超过保留天数的旧备份。保留天数可以在页面上设置。

#### 删除备份

- **单条删除**：每行末尾有删除按钮
- **批量删除**：勾选多个备份文件，点 **"批量删除选中"** 一次删掉。删除前有弹窗确认"确认批量删除选中的备份文件？删除后不可恢复。"。表头有"全选"复选框

> **小贴士**：自动备份产生的文件会越来越多，建议定期用"清理过期备份"功能（而不是手动逐个删），设好保留天数就行。

#### 自动任务设置

页面上方有"自动任务设置"区域，可以开启：
- 自动备份：有人访问系统时按间隔自动备份，正常关闭系统时也会自动备份一次
- 自动清理：按间隔自动清理过期备份

每个自动任务旁边会显示上次执行时间。如果上次执行时间的记录损坏（比如数据库迁移或意外中断导致），页面会显示 **"时间损坏（原始值：xxx）"** 而不是空白，帮助你判断任务是否正常运行。

> **小贴士**：如果页面顶部出现黄色提示框显示"检测到系统配置包含历史兼容读取结果"，说明某些系统配置字段的值是通过历史兼容规则读取的（比如旧版本留下的非标准值被自动钳制）。提示框会列出哪些字段受影响及原因，这不影响正常使用，但建议在方便时手动复核并重新保存一次。

### 10.2 操作日志

**进入方式**：系统管理 → 操作日志

记录系统的关键操作（导入、导出、排产、备份等），方便追溯。

**筛选字段**：

| 筛选项 | 说明 |
|--------|------|
| 时间范围 | 选择开始和结束日期/时间，只看这段时间内的日志 |
| 模块 | 可按中文标签或英文模块值筛选，如 系统 / 排产调度 / 工艺管理，或 `system`、`scheduler`、`process`、`personnel`、`equipment` |
| 动作 | 可按中文标签或英文动作值筛选，如 导入 / 导出 / 排产 / 创建 / 删除，或 `import`、`export`、`schedule`、`create`、`delete` |
| 级别 | 按日志级别筛选：信息 / 警告 / 错误 |
| 条数 | 限制显示多少条日志（默认50条） |

- **删除**：单条删除或勾选多条后批量删除（删除不可恢复；批量删除前先确认筛选条件、已选条数和本次删除范围）
- **自动清理**：可设置保留天数和清理间隔

> **小贴士**：操作日志建议先缩小范围再查——先选模块和时间范围，不然一下子加载太多条可能会比较慢。页面列表会优先显示中文标签，同时保留原始 module/action code 作为次要信息，方便和开发文档或日志文件交叉核对。

### 10.3 排产历史

**进入方式**：系统管理 → 排产历史

查看所有排产版本的策略和结果摘要。

**操作步骤**：
1. 在版本下拉框中选择某次排产查看详情（每个选项显示版本号、排产时间、策略、结果状态）
2. 或者在"最近记录条数"输入框填一个数字（如 20），点 **"查询"** 看最近 N 条

每条记录包含：排产时间、版本号、策略（优先级优先 / 交期优先 / 综合等）、批次数、结果状态（成功/失败/部分成功等）、结果摘要

**空态和缺失版本**：
- 当前没有排产历史时，页面显示暂无历史，不会构造不存在的版本号。
- 如果地址里指定了不存在的版本，例如 `/system/history?version=999`，页面会显示"v999 无对应排产历史"，不会自动跳到最新版本。
- 页面摘要只展示公开口径：结果状态、完成状态、核心指标、告警/错误预览。隐藏的内部字段、原始异常和路径只用于后台诊断。

> **小贴士**：这里主要看版本概览和排产结果概况。想看具体排程时间安排，请去甘特图、周计划或资源排班页面。

### 10.4 插件管理

**进入方式**：系统管理 → 备份/恢复页面底部的"可选插件/可选依赖状态"区域

可以开启或关闭系统插件。修改后需要重启系统才能生效。插件区域会显示以下信息：

- **配置来源**：说明当前插件开关的配置来自哪里。正常情况下显示"全部来自系统配置"；如果数据库连接失败或配置读取异常，会提示"当前按默认开关运行"并说明具体原因（如数据库连接失败、配置读取器初始化失败等）
- **依赖路径**：列出插件所需的第三方依赖安装位置
- **每个插件的当前来源**：在每个插件的启用开关旁边，会标注该插件的开关值来自"系统配置"还是"默认开关"，帮你判断哪些插件的配置是真正生效的、哪些是因异常而回退到默认值的

### 10.5 启动与端口

系统启动后，实际监听的端口会写入 `logs/aps_port.txt` 文件。如果默认端口被占用，系统会自动切换到其他可用端口。

如果浏览器打不开系统页面，可以查看这个文件确认实际端口号。同样，监听地址会写入 `logs/aps_host.txt`。

## 11. 常见问题：遇到报错怎么办

### "内部工序未补全资源，无法排产"

**原因**：某道工序没有选择设备或人员

**解决**：
1. 去批次详情页
2. 找到报错的工序（通常会提示是哪个批次哪道工序）
3. 选择设备和人员
4. 保存后重新排产

**或者**：开启"自动分配"（高级设置 → 未指定设备或人员时，系统自动分配）

---

### "人员未被配置为可操作设备"

**原因**：你选的这个人没有被设置为可以操作这台设备

**解决**：
1. 去人员管理
2. 找到这个人，点击进入详情
3. 在"可操作设备"里添加这台设备
4. 保存后回来重试

---

### "未齐套，禁止排产"

**原因**：批次的齐套状态是"未齐套"，但开启了齐套约束

**解决**（三选一）：
1. 把批次的齐套状态改成"齐套"
2. 在执行排产时取消勾选"启用齐套约束"
3. 在高级设置里关闭"排产前检查物料是否齐套"

---

### "预计完工超出排产窗口"

**原因**：你设置了截止日期，但工期太长排不完

**解决**（三选一）：
1. 放宽截止日期，或者干脆不设截止日期
2. 减少本次排产的批次数量
3. 增加可用资源或工时

---

### Excel导入失败，提示"存在错误行"

**原因**：Excel里某些行的数据有问题

**解决**：
1. 看预览页面的错误提示，会标红告诉你哪一行哪一列出了问题。错误提示中的行号对应的是你 Excel 文件中的原始行号（含表头行），可以直接在 Excel 里定位
2. 按提示修改Excel
3. 重新上传

**常见错误**：
- 日期格式错误（支持 `YYYY-MM-DD` 和 `YYYY/MM/DD`，不能带时间）
- 必填字段为空（批次号、图号、数量不能空）
- 图号不存在（需要先在工艺管理里录入这个零件）

详细的错误清单和解决方法请看第 1 章的"常见导入错误"表格。

---

### 甘特图上工序时间重叠了

**原因**：这通常是正常的，因为不同设备/不同人员可以同时工作

**如果确实有问题**：
1. 检查重叠的工序是否在同一台设备上、同一个人操作——如果是，说明数据有问题
2. 检查工序依赖（同一批次的工序是否按顺序排了）

---

### 外协工序排不进去

**原因**：外协工序需要指定供应商和外协周期（天数），缺少任一信息都无法排

**解决**：
1. 去批次详情页
2. 找到外协工序（来源标记为"外部"）
3. 选择供应商，填写外协周期（天数）
4. 保存后重新排产

> **小贴士**：外协工序不占用你的内部设备和人员，按自然日计算周期。比如外协5天，从周五开始，周三结束（中间包含周末）。

---

### 版本号是什么意思？

每次执行排产或模拟排产，系统都会生成一个新版本（如 v1、v2、v3……）。

- **执行排产**：会更新批次状态（待排 → 已排），同时生成新版本
- **模拟排产**：不改变批次状态，只生成新版本供你查看

你可以在甘特图、周计划、资源排班等页面的"版本"下拉框里选择不同版本来对比效果。版本号只会增加不会删除，不用担心覆盖。

---

### 甘特图上没有数据

**原因**（按可能性排列）——甘特图会根据具体情况显示不同的空白提示：
1. 还没有执行过排产
2. 选的时间范围和排产结果不在同一个区间（比如排产排到了下周，但甘特图显示的是本周）——此时提示"暂无排程数据（该周/该版本）"
3. 选的版本号不对——同上
4. 当前筛选条件过滤掉了所有任务（比如选了某个设备但该设备在当前周没有任务）——此时提示"当前筛选条件下暂无可显示任务"
5. 排程数据存在时间格式异常，全部被系统过滤——此时提示"当前区间存在时间非法的排程数据，已全部过滤，请检查排产结果"

**解决**：
1. 先确认已经执行过排产（排产调度主页会显示最近排产的信息）
2. 在甘特图页面，点"回到本周"或调整时间范围
3. 检查"版本"下拉框，选择最新的版本
4. 清除筛选条件试试；如果提示时间非法，说明排产结果中有工序的开始/结束时间数据损坏，建议重新执行一次排产

---

### 排产很慢怎么办

**可能原因**：
- 批次太多，计算量大
- 用了"精细计算"模式，计算时间上限设得很大
- 开启了"深度优化"

**解决**：
1. 先试试"快速计算"模式，看看结果够不够用
2. 如果用"精细计算"，把计算时间上限调低（比如20~30秒）
3. 关闭"深度优化"
4. 分批排产：先排紧急的，再排普通的

---

### 设备/人员组合不匹配（行内提示）

**原因**：你选的设备和人员之间没有建立"可操作"关联

**解决**：
1. 去人员管理 → 找到这个人 → 在"可操作设备"里添加这台设备
2. 或者去设备管理 → 找到这台设备 → 在"可操作人员"里添加这个人
3. 保存后回到批次详情页重新选择

---

### Excel 文件被占用，无法上传

**原因**：Excel 文件正在被 Microsoft Excel 或其他程序打开

**解决**：
1. 关闭正在打开该文件的程序
2. 回到系统页面重新上传

---

### 导入用"替换"模式但被拒绝

**原因**：人员或设备导入的"替换"模式有保护机制——如果仍有排产数据引用了这些人员/设备，系统会拒绝清空

**解决**：
1. 改用"覆盖"模式（只更新已有数据，不删除）
2. 或者先在排产历史中确认不需要旧排程数据，手动清理后再替换

---

### 批次导入用"替换"后排程记录丢了

**原因**：批次导入的"替换"模式没有引用保护。它会删除所有批次，排程记录因为外键级联也一起被删了

**解决**：
- 如果有备份，去系统管理 → 备份/恢复，恢复到之前的备份
- 以后使用"替换"前，先手动备份

---

### 页面某个链接变灰了

**原因**：系统的某个功能模块没有加载成功，通常是因为：
- 运行的是旧版本程序，未包含新功能
- 程序启动后没有重启

**解决**：重启系统，或者使用最新版本重新打包

---

### 备份恢复后数据没有更新

**原因**：浏览器可能缓存了旧页面

**解决**：刷新页面（按 F5），或者关闭浏览器重新打开

---

### 系统打不开 / 端口被占用

**原因**：默认端口可能被其他程序占用了

**解决**：
1. 查看 `logs/aps_port.txt` 文件，里面记录了系统实际使用的端口号
2. 在浏览器中输入正确的地址和端口访问
3. 如果仍然打不开，检查是否有其他程序占用了端口，或者尝试重启系统

---

### 删除人员/设备失败

**原因**：该人员或设备已经被批次工序或排程结果引用了

**解决**：
- 不要删除，改为"停用"（人员）或"停用/维修"（设备）
- 停用后排产时不会选择该资源，但历史数据不受影响

---

### 删除零件失败

**原因**：该零件的图号已经被某个批次引用了

**解决**：
1. 先删除引用了这个图号的所有批次
2. 然后再删除零件

---

### 批次详情页面卡顿（Win7 用户）

**原因**：工序较多或设备/人员候选较多时，页面渲染下拉框需要较长时间

**解决**：
- 系统在工序较多时（超过 30 道）会自动启用懒加载（下拉框第一次点开才加载完整选项）
- 如果你想关掉懒加载（方便一次看到所有选项），在地址栏加 `?lazy_select=0`
- 或者分批处理：先处理重要的工序，再处理其他的

---

### 工作日历没生效

**原因**（按可能性排列）：
1. 配置的日期不在排产的时间范围内
2. 个人工作日历覆盖了全局工作日历（个人日历优先级更高）
3. 没有点"保存"

**解决**：
1. 检查日历配置的日期是否在排产开始时间之后
2. 如果有个人日历，检查是否和全局设置冲突
3. 确认每次修改后都点了"保存"

---

### 排产结果和上次完全一样

**原因**：快速计算模式是确定性的——相同的输入数据 + 相同的参数 = 相同的结果

**解决**：
- 如果想看看有没有更好的方案，切换到"精细计算"模式（每次结果可能略有不同，因为有随机探索）
- 或者调整参数（比如换一种排产策略、改变优先级权重）

---

### 外协工序周期怎么算？

外协工序按**自然日**计算周期（包含周末和假期），不受工作日历影响。

- 周期从上一道工序完成时开始计时
- 比如外协5天，从周五完工开始算，周三结束
- 外协工序不占用任何内部设备和人员

---

### 齐套日期的作用是什么？

齐套日期是物料预计齐全的日期。设置后，系统在排产时**不会在齐套日期之前安排这个批次开工**。

比如某批次的齐套日期是3月20日，排产时即使有空闲设备，系统也不会把这个批次安排在3月20日之前开始。

---

### 换型时间和单件时间怎么填？

两个时间的单位都是**小时**：

- **换型时间**：设备从上一个产品切换到当前产品需要的准备时间。如果连续做同类产品，系统会自动跳过换型
- **单件时间**：加工一个零件需要多长时间

总加工时长 = 换型时间 + 单件时间 × 批次数量

> **场景示例**：换型需要0.5小时，做一个零件需要0.1小时，批次数量100个。则总工时 = 0.5 + 0.1 × 100 = 10.5小时。

---

### 数值字段报错（非数字或无穷大）

**原因**：某个数值字段填入了非法值（比如文字、特殊符号、公式错误导致的非数字结果等），或者必填数值字段留空了

**解决**：
1. 找到报错提示中提到的字段
2. 改为正常的数字（比如 0.5、8、1.0 等）。如果提示"不能为空"，说明这个字段是必填的，不能留空
3. 保存后重试

> **补充**：现在系统会更严格地检查必填数值字段。像空白、非数字、无穷大这类异常值，系统会直接报错，不再悄悄替你补默认值。这样能避免表面上保存成功、实际数值却被系统自动改写的情况。
>
> 布尔值在数值字段中也会被拒绝（报错"必须是数字"），因为系统不允许将逻辑值当数字使用。这和是/否类字段不同，后者应直接填写 `是` 或 `否`。

---

### 下拉框里没有可选的设备或人员

**原因**：还没有维护"人员-设备关联"

**解决**：
1. 去人员管理 → 点击某个人 → 在"可操作设备"里添加设备
2. 或者去设备管理 → 点击某台设备 → 在"可操作人员"里添加人员
3. 保存后回到批次详情页，刷新页面即可看到选项

也可以通过 Excel 批量导入关联关系（人员管理 → 人员设备关联（Excel） 或 设备管理 → 设备人员关联（Excel）），详见第 1 章。

---

### 首页上数字都是0怎么办

**原因**：正常现象——说明还没有导入批次或执行排产

**解决**：按第 2 章的快速上手走一遍就会有数据了。导入批次、执行排产后，首页的统计卡片会自动更新。

---

### 报表页面提示"暂无排产历史"

**原因**：需要先执行排产才能查看报表

**解决**：去排产调度主页执行一次排产就好。报表中心的数据全部基于排产版本生成。

---

### 利用率超过100%正常吗

**原因**：说明排程工时超过了可用工时

**解决**：
1. 检查工作日历配置是否正确（比如是不是漏配了加班日）
2. 考虑增加设备或人员来分摊负荷
3. 或者减少同时段排产的批次数量

---

### 停机计划和排程重叠了怎么办

**原因**：停机计划是在排产之后才加上去的，排产算法没机会避开

**解决**（二选一）：
1. 调整停机时间，避开已排产的任务
2. 重新排产，系统会自动避开停机时段

> **小贴士**：建议在排产之前就把停机计划配好，这样排产算法会自动绕开。

---

### 页面上出现黄色提示是什么意思？

**原因**：系统在读取或展示数据时发现了部分异常，但已经自动做了兼容处理，所以页面还能继续使用。

**常见提示**：
- "工作日历加载失败，当前不显示假期/停工背景标注"：甘特图的假期底色标记暂时不可用，但排产结果本身不受影响
- "已过滤 N 条时间不合法的排程记录"：部分工序的开始时间或结束时间异常，暂时无法展示
- "检测到系统配置包含历史兼容读取结果"：某些系统配置值来自旧数据的兼容读取或范围修正
- "当前展示包含 N 条历史兼容归一结果"：旧数据里的技能等级或主操标记写法不标准，系统已自动转换为标准写法
- "关键链不可用"：当前版本的关键链无法可靠计算，甘特图会停止高亮关键链，但任务条仍按原排程展示
- "超期标记可能不完整"：超期摘要或标记数据存在异常，页面暂时禁用仅超期筛选，修复数据后刷新即可恢复

**处理建议**：这类提示一般不影响继续使用，但建议在方便时检查原始数据，确认无误后重新保存一次。

---

### 备份恢复成功后还需要做什么？

**说明**：正常情况下，系统会在恢复后继续完成结构校验。页面成功提示一般会显示"已从备份恢复并完成结构校验……"。如果恢复失败或恢复后的结构校验没有通过，系统会提示失败，并尽最大努力回滚到恢复前自动备份。

**建议操作**：
1. 刷新页面或重新打开浏览器
2. 抽查关键数据（批次、排程、工艺模板）是否正常
3. 如果页面提示恢复失败或自动回滚，请先查看日志，再决定是否重新恢复

## 12. 术语速查表

按拼音首字母排列，方便查找。

### B

**班组 (Resource Team)**
人员和设备的分组单位。可以在人员管理和设备管理中设置归属班组，并在资源排班中心按班组视角查看排班。

**版本 (Version)**
每次执行或模拟排产生成的一个快照编号（v1、v2、v3……）。可以在甘特图、周计划等页面选择不同版本对比效果。

**批次 (Batch)**
一个生产任务单元。包含批次号、图号、数量、交期、优先级等信息。一个批次有多道工序。

### C

**超期 (Overdue/Tardiness)**
实际完工时间（或当前时间）晚于交期。报表中心的超期清单会列出所有超期批次，分为"已排程逾期"和"未排程逾期"两类。

**齐套 (Ready Status)**
物料是否齐全，分为：齐套、未齐套、部分齐套。开启齐套约束后，未齐套的批次不能排产。

**齐套日期 (Ready Date)**
物料齐套的预计日期。在此日期之前批次不会开工。

### D

**锁定近期排程 (Freeze Window)**
保持一段时间内的排程不变，只重新排后面的。比如"锁定3天"表示未来3天的安排不会被重新排。

### G

**甘特图 (Gantt Chart)**
用条形图展示排程结果。横轴是时间，纵轴是资源（设备或人员）。支持筛选、配色切换、聚焦、关键链高亮等功能。

**关键链 (Critical Chain)**
从排产开始到最后完工的最长路径。关键链上的工序决定了整体工期——只有优化关键链上的工序，总工期才能缩短。

**工序 (Operation)**
批次内的一道加工步骤。分为内部工序（自己做，需要设备和人员）和外部工序（外协，需要供应商和外协周期）。

**工种 (Operation Type)**
工序的类型分类。用来区分"内部工序"和"外部工序"，也用来匹配设备和供应商。

**工作日历 (Work Calendar)**
定义哪些天工作、每天工作几小时、效率多少。支持逐天设置和 Excel 批量导入。个人工作日历优先级高于全局日历。

**供应商 (Supplier)**
外协工序的承接方。外协工序需要指定供应商和外协周期（天数）。

### H

**合并周期 / 分别设置**
外部工序组的两种周期模式。"合并设置"表示一组连续的外协工序共用一个总天数；"分别设置"表示每道外协工序单独设天数。

**换型 (Changeover)**
设备从做上一个产品切换到做下一个产品需要的准备时间。系统会尽量把同类工序排在一起，减少换型。

### J

**界面模式**
系统有两种界面风格：经典（v1）和现代（v2）。可以在页面右上角切换。功能完全一样，只是外观不同。

**计算模式**
排产算法的运行方式，分为快速计算和精细计算两种。快速计算几秒出结果；精细计算多花时间找更优方案。

**精细计算**
排产的高级计算模式。先快速做一版，然后在时间预算内不断调整寻找更优方案。速度较慢，结果通常更好，每次结果可能略有不同。

**交期 (Due Date)**
客户要求的交货日期。排产时会尽量保证在交期前完成。

### K

**快速计算**
排产的默认计算模式。按设定的策略和规则一次排完，速度快、结果稳定、每次相同。

### L

**利用率 (Utilization)**
排程负荷占可用工时的比例（利用率 = 排程负荷 / 可用工时），越高说明资源用得越充分。在报表中心的"资源负荷与利用率"页面查看。

**留痕 / 操作日志**
系统会记录关键操作（导入、导出、排产、备份等）到数据库中，方便追溯。在系统管理 → 操作日志页面查看。

### M

**模拟排产**
只看排产效果，不真正更新批次状态。每次模拟也会生成新版本号，适合评估插单影响或反复对比方案。

### P

**派工方式 (Dispatch Mode)**
决定按什么顺序把工序放进时间表。"按批次顺序"逐个批次排完再排下一个，"智能派工"则动态挑选当前最紧急的工序。

### S

**深度优化**
在快速计算基础上额外进行的一轮自动优化。需要系统支持额外计算组件，开启后可设置时间上限（建议3~15秒）。

### T

**停机计划 (Machine Downtime)**
为设备安排的计划性停机时间段（如保养、维修）。排产时系统会自动避开停机时段。支持单台或批量创建。

**拖期 (Tardiness)**
实际完工时间超过交期的小时数。"总拖期"是所有超期批次的拖期时间加在一起。

### W

**外部工序组 (External Group)**
连续的外协工序会被自动归为一组。可以选择"合并设置"（整组共用总周期）或"分别设置"（每道分别设天数）。

**外协 (External/Outsourcing)**
交给外部供应商完成的工序。按自然日计算周期，不占用内部设备和人员。

### Y

**仪表盘/首页 (Dashboard)**
打开系统后看到的第一个页面，展示待排/已排/超期批次统计卡片、最近排产留痕和快捷操作入口。

**优化目标 (Objective)**
告诉系统"什么算好的排产结果"。可选：最少超期（尽量不超期）、最少拖期小时（总延迟最小）、最少加权拖期小时（按优先级区分，特急件拖期惩罚更重）、最少换型次数（减少设备调整）。

**优先级 (Priority)**
批次的紧急程度，分为：普通、急件、特急。优先级高的批次会被优先排产。

### Z

**执行排产**
正式生成排产结果，会更新批次和工序的状态（待排 → 已排）。和"模拟排产"的区别是：执行会改状态，模拟不会。

**智能派工**
一种派工方式，系统不按批次顺序，而是自动挑选当前准备好的、最紧急的工序优先排入。相比"按批次顺序"更灵活。有三种策略可选：时间余量少的先做、交期更紧的先做、综合判断更紧急的先做。

**资源排班 (Resource Dispatch)**
从人员、设备或班组的视角查看排产后的任务安排。支持任务明细表格、日历矩阵、甘特图三种展示方式，可导出 Excel。
