## 1. 快速上手：5分钟完成第一次排产

**目标**：让系统跑起来，看到排产结果（甘特图）。

### 第一步：导入一个批次

1. 打开 **排产调度 → 批次信息（Excel导入）**
2. 下载模板 `批次信息.xlsx`
3. 填写一行数据：
   - 批次号：例如 `B001`
   - 图号：例如 `PART-001`
   - 数量：例如 `10`
4. 上传文件，勾选 **"导入后自动生成工序"**
5. 确认导入

### 第二步：补充工序信息

1. 打开 **排产调度 → 批次详情**（点击刚才导入的批次）
2. 每道工序需要填写：
   - **设备**：这道工序在哪台机器上做
   - **人员**：谁来操作
   - **工时**：换型时间 + 单件时间
3. 点击保存

> **小贴士**：如果下拉框里没有可选的人员，说明还没有维护"人员-设备关联"。去人员管理页面设置一下就行。

### 第三步：执行排产

1. 回到 **排产调度主页**
2. 在"待排"列表里勾选你的批次
3. 点击 **执行排产**

### 第四步：查看结果

- **甘特图**：直观看到每道工序的时间安排
- **周计划**：导出 Excel 给车间使用

恭喜！你已经完成了第一次排产。

## 2. 基本概念：像排队买饭一样理解排产

把排产想象成食堂打饭的过程：

### 批次 = 你的餐盘

每个批次就像你端着的餐盘，里面要装几个菜（工序）。餐盘上写着：
- **批次号**：你的取餐号
- **数量**：要打几份菜
- **交期**：什么时候必须吃完（交货日期）
- **优先级**：普通顾客 / VIP / 超级VIP

### 工序 = 要打的每道菜

每道菜就是一道工序，必须按顺序打：
- 先打米饭（第1道工序）
- 再打菜（第2道工序）
- 最后打汤（第3道工序）

工序分两种：
- **内部工序**：食堂自己做的菜（需要占用厨师和灶台）
- **外部工序（外协）**：外面餐厅送来的菜（只需要等配送时间）

### 资源 = 厨师和灶台

- **设备**：灶台（同一时间只能做一道菜）
- **人员**：厨师（同一时间只能操作一个灶台）
- **人员-设备关联**：哪个厨师会用哪个灶台

### 工作日历 = 食堂营业时间

- 哪些天营业（工作日）
- 每天营业几小时
- 周末加班怎么算

### 排产策略 = 谁先打饭

- **优先级优先**：VIP先打
- **交期优先**：赶时间的先打
- **先来先打**：按排队顺序

## 3. 日常操作：按任务找方法

### 3.1 我要新增批次

**方法一：Excel批量导入**
1. 排产调度 → 批次信息（Excel导入）
2. 下载模板、填写、上传
3. 勾选"导入后自动生成工序"

**方法二：页面手动添加**
1. 排产调度 → 批次管理 → 新增批次
2. 填写批次号、图号、数量等

### 3.2 我要修改工序信息

1. 排产调度 → 点击批次号进入详情页
2. 找到要修改的工序
3. 修改设备、人员、工时等
4. 保存

### 3.3 我要安排周末加班

1. 排产调度 → 工作日历配置
2. 找到周末的日期
3. 把类型改成"工作日"或"调休补班"
4. 填写可用工时（比如8小时）

### 3.4 我要看排产结果

- **甘特图-设备视角**：看每台设备的工作安排
- **甘特图-人员视角**：看每个人的工作安排
- **周计划导出**：生成Excel给车间

### 3.5 我要模拟插单影响

1. 在主页勾选要插入的新批次
2. 点击 **模拟排产**（不是"执行排产"）
3. 查看甘特图，对比变化
4. 满意后再正式执行排产

> **模拟排产 vs 执行排产**：模拟只是看看效果，不会真的改变批次状态。

### 3.6 我要复制批次

1. 批次管理 → 勾选要复制的批次
2. 点击"批量复制"
3. 系统会自动给批次号加1（如 B001 → B002）

## 4. 高级设置：每个开关的作用

高级设置页面有很多开关，下面逐个解释。

### 4.1 模板/方案

**作用**：一键切换预设的参数组合

| 模板名 | 适合场景 |
|--------|----------|
| 默认-稳定 | 日常使用，快速稳定 |
| 交期优先 | 交货压力大时 |
| 换型最少 | 减少设备调整次数 |
| 改进-更优(慢) | 愿意多等一会儿换更好的结果 |

> **建议**：日常用"默认-稳定"就够了。

### 4.2 排产策略

**作用**：决定先排哪些批次

| 策略 | 说明 | 适合场景 |
|------|------|----------|
| 优先级优先 | 特急 > 急件 > 普通 | 有VIP客户时 |
| 交期优先 | 交期近的先排 | 交货压力大时 |
| 权重混合 | 综合考虑优先级和交期 | 需要平衡时 |
| 先进先出 | 按创建时间排 | 公平对待所有订单 |

### 4.3 算法模式

| 模式 | 速度 | 效果 | 说明 |
|------|------|------|------|
| 贪心（快速） | 快 | 够用 | 几秒出结果 |
| 改进（多尝试） | 慢 | 可能更好 | 多花时间找更优解 |

> **场景示例**：平时用"贪心"，月度大排产用"改进"。

### 4.4 派工方式

| 方式 | 说明 |
|------|------|
| 批次顺序 | 一个批次排完再排下一个（简单稳定） |
| SGS动态派工 | 哪道工序准备好就先排哪道（更灵活） |

> **建议**：除非对结果不满意，否则用"批次顺序"就行。

### 4.5 目标函数

**作用**：告诉系统"什么叫好的排产结果"

| 目标 | 优先考虑 |
|------|----------|
| 最少超期 | 尽量不让订单超过交期 |
| 最少拖期 | 尽量减少总的延迟时间 |
| 最少换型 | 尽量减少设备调整次数 |

### 4.6 时间预算

**作用**：改进模式下愿意等多久

- 20秒：日常够用
- 60秒：重要排产
- 120秒以上：月度大排产

### 4.7 冻结窗口

**作用**：保持近期排程不变

**场景示例**：今天是周一，已经开始按排程生产了。你希望未来3天的安排不要变，只重新排3天以后的。这时开启"冻结窗口=3天"。

**注意**：冻结可能让整体结果变差，但能保证现场稳定。

### 4.8 自动分配资源

**作用**：工序没有选设备/人员时，系统自动选

**场景示例**：你只想快速看个粗略排产结果，还没来得及填所有工序的设备人员。

**注意**：自动选的不一定符合实际情况，建议后续手动确认。

### 4.9 齐套约束

**作用**：物料没齐的批次不允许排产

- 开启：未齐套的批次会被拦截
- 关闭：所有批次都能排

### 4.10 假期默认效率

**作用**：周末/假期加班时，效率按多少算

默认0.8（即80%），意思是假期加班效率比平时低一点。

## 5. 算法选择：快餐还是私厨

### 快餐模式（贪心算法）

像快餐店一样快速出餐：
- **速度**：几秒钟出结果
- **原理**：按固定规则排，不回头调整
- **优点**：快、稳定、容易理解
- **缺点**：不一定是最优解

**适合场景**：
- 日常排产
- 数据变化频繁
- 对结果要求不高

### 私厨模式（改进算法）

像私人厨师一样精心烹制：
- **速度**：几十秒到几分钟
- **原理**：先快速做一版，然后不断调整找更好的
- **优点**：结果可能更好
- **缺点**：慢、结果可能每次不一样

**适合场景**：
- 月度/周度重要排产
- 对交期或换型有严格要求
- 愿意多等一会儿

### 该怎么选？

| 情况 | 推荐 |
|------|------|
| 每天日常排产 | 贪心 + 批次顺序 |
| 交期很紧张 | 改进 + 交期优先 + 60秒 |
| 换型成本高 | 改进 + 最少换型 + 60秒 |
| 快速看个大概 | 贪心 + 自动分配 |

## 6. 常见问题：遇到报错怎么办

### "内部工序未补全资源，无法排产"

**原因**：某道工序没有选择设备或人员

**解决**：
1. 去批次详情页
2. 找到报错的工序
3. 选择设备和人员
4. 保存后重新排产

**或者**：开启"自动分配资源"（高级设置）

---

### "人员未被配置为可操作设备"

**原因**：你选的这个人没有被设置为可以操作这台设备

**解决**：
1. 去人员管理
2. 找到这个人
3. 在"可操作设备"里添加这台设备
4. 保存后回来重试

---

### "未齐套，禁止排产"

**原因**：批次的齐套状态是"未齐套"，但开启了齐套约束

**解决**（三选一）：
1. 把批次的齐套状态改成"齐套"
2. 取消勾选"启用齐套约束"
3. 在高级设置里关闭"默认启用齐套约束"

---

### "预计完工超出排产窗口"

**原因**：你设置了截止日期，但工期太长排不完

**解决**（三选一）：
1. 放宽截止日期
2. 减少本次排产的批次数量
3. 增加可用资源或工时

---

### Excel导入失败，提示"存在错误行"

**原因**：Excel里某些行的数据有问题

**解决**：
1. 看预览页面的错误提示
2. 按提示修改Excel
3. 重新上传

**常见错误**：
- 日期格式错误（应该是 `2026-01-25`，不能带时间）
- 必填字段为空
- 图号不存在

---

### 甘特图上工序时间重叠了

**原因**：这通常是正常的，因为不同设备/人员可以同时工作

**如果确实有问题**：
1. 检查是否同一设备同一人员
2. 检查工序依赖是否正确

## 7. 术语速查表

按字母顺序排列，方便查找。

### B

**批次 (Batch)**
一个生产任务单元。包含批次号、图号、数量、交期、优先级等信息。

### C

**齐套 (Ready Status)**
物料是否齐全。分为：齐套、未齐套、部分齐套。

### D

**冻结窗口 (Freeze Window)**
保持一段时间内的排程不变。比如"冻结3天"表示未来3天的安排不会被重新排。

### G

**甘特图 (Gantt Chart)**
用条形图展示排程结果。横轴是时间，纵轴是资源（设备或人员）。

**工序 (Operation)**
批次内的一道加工步骤。分为内部工序（自己做）和外部工序（外协）。

**工作日历 (Work Calendar)**
定义哪些天工作、每天工作几小时、效率多少。

### H

**换型 (Changeover)**
设备从做上一个产品切换到做下一个产品需要的准备时间。

### J

**交期 (Due Date)**
客户要求的交货日期。

### M

**模拟排产**
只看排产效果，不真正更新批次状态。适合评估插单影响。

### P

**派工方式 (Dispatch Mode)**
决定按什么顺序把工序放进时间表。

**优先级 (Priority)**
批次的紧急程度。分为：普通、急件、特急。

### Q

**齐套日期 (Ready Date)**
物料齐套的预计日期。在此日期之前批次不会开工。

### S

**SGS (动态派工)**
一种派工方式，哪道工序准备好就先排哪道，更灵活。

### T

**拖期 (Tardiness)**
实际完工时间超过交期的小时数。

### W

**外协 (External/Outsourcing)**
交给外部供应商完成的工序。按自然日计算周期，不占用内部资源。

### Z

**执行排产**
正式生成排产结果，会更新批次和工序的状态。
